## 1. 현재 AI 아키텍처의 구조적 한계

| 한계 | 현재 상태 | R&D로 해결 가능한 방향 |
|---|---|---|
| **범용 LLM 의존** | 모든 에이전트가 GPT-4o API 호출 (temperature 0.1~0.7) | 도메인 파인튜닝 모델로 비용↓ 정확도↑ |
| **사진 분류가 텍스트 기반** | VisualClassificationAgent가 이미지 URL만 전달 | 자체 Vision 모델로 실제 픽셀 분석 |
| **VibeFit이 주관적** | VAD 좌표를 LLM이 텍스트로 추정 | 학습된 감성 임베딩 모델로 정량화 |
| **가격 예측 없음** | 가격대(price_band)만 신호 수준 | 실거래가 기반 예측 모델 |
| **패턴 학습 미구현** | 딜 결과가 다음 딜에 영향 없음 | 피드백 루프 + 강화학습 |

## 2. MECE R&D 아이템 체계

### 축 1: AI 기술 유형 (How)
- A. **파인튜닝/특화 모델** — 도메인 데이터로 기존 LLM 최적화
- B. **컴퓨터 비전** — 이미지/영상 직접 분석
- C. **예측/추천 알고리즘** — 시계열·그래프·매칭 엔진
- D. **NLP/문서 이해** — 비정형 문서 자동 파싱
- E. **강화학습/에이전트 오케스트레이션** — 자율 의사결정 체인

### 축 2: 비즈니스 도메인 (What)
- ① 매물 인텔리전스 (딜카드)
- ② 투자 분석 (Full IM)
- ③ 임대 마케팅 (Space AI)
- ④ 크로스시스템 통합

## 3. R&D 아이템 상세 (15개)

### 영역 A: 파인튜닝/특화 모델

#### A-1. 한국어 CRE 메모 파싱 SLM (Small Language Model)
- **현재**: GPT-4o로 카카오 메모 파싱 (호출당 ~$0.03, 레이턴시 3-5초)
- **R&D**: 한국어 부동산 메모 1만건으로 Llama 3 8B 파인튜닝
- **목표**: 비용 1/50 (자체 GPU), 레이턴시 500ms 이하, 오프라인 가능
- **데이터 소스**: `activity_events` + `building_ssot_lite` 누적 데이터
- **학술 가치**: 한국어 도메인 특화 SLM 벤치마크 논문 (CRE-KoNLU)
- **사업 가치**: API 비용 연간 ~₩5,000만 절감 (MAU 1만 기준)

#### A-2. IM 섹션 작성 특화 모델
- **현재**: 18개 섹션 각각 GPT-4o 호출 (프로젝트당 18회 × ~$0.10)
- **R&D**: 전문가 패치 이력(수정 전/후 diff)을 학습 데이터로 RLHF
- **목표**: 전문가 패치 빈도 50% 감소, 초안 품질 기관투자자 수준 도달
- **데이터 소스**: `ai_runs` 테이블의 input_ref/output_ref + 전문가 수정 이력
- **독점성**: 한국 CRE IM 작성 데이터는 이 시스템에서만 축적 가능

#### A-3. 가드레일 특화 분류기 (Risk Boundary Classifier)
- **현재**: 정규표현식 8개 패턴 매칭 (`FORBIDDEN_PATTERNS`)
- **R&D**: BERT 기반 한국어 CRE 위험 표현 분류기 파인튜닝
- **목표**: 패턴 매칭으로 못 잡는 우회 표현 탐지 (예: "사실상 안전한")
- **학술 가치**: 금융 규제 언어 자동 탐지 모델 (RegTech AI 분야)
- **사업 가치**: 법적 리스크 방지 → 보험/컴플라이언스 비용 절감

---

### 영역 B: 컴퓨터 비전

#### B-1. 상업공간 사진 자동 분류 모델 (CRE-Vision)
- **현재**: `VisualClassificationAgent`가 텍스트 프롬프트로 분류 (실제 픽셀 미분석)
- **R&D**: ResNet/EfficientNet 기반 상업공간 사진 분류기 학습
  - 클래스: 외관/로비/계단/엘리베이터/화장실/주방설비/전기판넬/주차장/옥상/간판위치 등 20+
  - 품질 자동 평가: 블러/노출/구도 점수
- **학습 데이터**: `visual_assets` 테이블 + `tags`/`facility_tags` 레이블
- **독점성**: 한국 상업 부동산 사진 분류 데이터셋 자체가 존재하지 않음
- **사업 가치**: 사진 분류 정확도 85%→95%, 분류 시간 5초→0.3초

#### B-2. 공간 감성 임베딩 모델 (SpaceVibe-Net)
- **현재**: `VibeFitAgent`가 VAD를 텍스트로 추정 (주관적)
- **R&D**: CLIP 계열 모델로 공간 사진 → 감성 벡터 매핑 학습
  - 감성 축: 따뜻함/차가움, 고급/캐주얼, 활기/차분, 개방/아늑
  - 업종 임베딩과 코사인 유사도로 객관적 핏 점수 산출
- **학술 가치**: Spatial Affective Computing (공간 감성 컴퓨팅) — 신규 연구 분야
- **사업 가치**: 감성 기반 임차인 매칭 정확도 → 공실 해소 기간 단축
- **특허 가능성**: "상업공간 이미지의 감성 벡터와 업종 프로파일 매칭 방법" ★

#### B-3. 건물 외관/노후도 자동 평가 모델
- **현재**: 물리적 상태를 텍스트 메모에 의존
- **R&D**: 외관 사진 → 노후도 5단계 자동 분류 + 리모델링 잠재력 점수
- **사업 가치**: Full IM의 `building_condition_physical_review` 섹션 자동화

### 영역 C: 예측/추천 알고리즘

#### C-1. 실거래가 기반 가격 예측 모델 (CRE-Price)
- **현재**: `price_band`만 신호 수준 (예: "80억대")
- **R&D**: 국토부 실거래가 API + 공시지가 + 건물 속성 → 가격 범위 예측
  - 모델: XGBoost/LightGBM 앙상블 + 불확실성 구간 (PI 90%)
  - 핵심: **절대 단일 가격 제시 금지** → 항상 구간 + 면책문구
- **학술 가치**: 한국 소형 상업부동산 가격 예측 (기존 연구 거의 없음)
- **사업 가치**: 딜 큐리오시티 점수의 정확도 비약적 향상
- **가드레일 통합**: `runRiskBoundaryCheck()`에 자동 연결

#### C-2. 임차인-공간 매칭 추천 엔진 (TenantMatch)
- **현재**: `TenantFitAgent`가 LLM으로 매번 새로 평가
- **R&D**: 과거 임대 성사/실패 데이터 기반 협업 필터링 추천
  - 공간 피처: 면적/층/설비/감성벡터/권역
  - 임차인 피처: 업종/예산/선호 조건
  - 매칭 성사율 예측 모델
- **데이터 소스**: `leasing_inquiries` + `inquiry_qualifications` + `tenant_fit_results`
- **사업 가치**: "이 공간에 가장 잘 맞는 업종 Top 3" → 공실 해소율 향상
- **네트워크 효과**: 데이터가 쌓일수록 추천 정확도 향상 → 플라이휠 가속

#### C-3. 딜 전환율 예측 모델 (DealConversion)
- **현재**: 크로스시스템 퍼널이 카운트만 표시
- **R&D**: 딜카드 → Full IM → 거래 성사의 전환 확률 예측
  - 피처: 딜큐리오시티 점수, SSoT 완성도, 문의 수, 전문가 패치 수
- **사업 가치**: 중개인에게 "이 딜은 성사 확률 72%입니다" → 시간 배분 최적화

### 영역 D: NLP/문서 이해

#### D-1. 등기부등본/임대차계약서 자동 파싱 (DocParser-CRE)
- **현재**: `legal_registry` 레이어가 수동 입력 의존
- **R&D**: OCR + Layout-LM 기반 한국어 부동산 문서 자동 파싱
  - 등기부등본 → 소유자/근저당/가압류/지상권 자동 추출
  - 임대차계약서 → 임차인/면적/임대료/만기일/특약 자동 추출
- **사업 가치**: Full IM 준비도 점수를 자동으로 30-40점 올려줌
- **학술 가치**: 한국어 법률 문서 구조화 NLP (Legal Document AI)
- **특허 가능성**: "부동산 등기부등본의 권리관계 자동 추출 및 리스크 태깅 방법" ★

#### D-2. 건물주/매수자 의도 분석 모델 (IntentNLP)
- **현재**: `BuyerIntentNormalizer`가 매수자 메모만 정규화
- **R&D**: 건물주 대화(카카오 메시지) → 매각 긴급도/가격 유연성/숨은 동기 분류
- **사업 가치**: 중개인의 협상 전략 수립 지원 → 딜 클로징 확률 향상

---

### 영역 E: 강화학습/에이전트 오케스트레이션

#### E-1. 멀티에이전트 자율 딜 파이프라인 (AutoDeal)
- **현재**: 24개 에이전트가 각각 독립 호출 (수동 트리거)
- **R&D**: 에이전트 간 자율 조율 — 데이터 상태를 보고 다음 에이전트 자동 결정
  - 예: SSoT 완성도 70% → "사진 부족" 감지 → 사진 요청 자동 푸시 → 분류 → 앨범 → LP 자동 발행
- **학술 가치**: CRE 도메인 멀티에이전트 시스템 (Multi-Agent Orchestration)
- **사업 가치**: 중개인 개입 최소화 → "완전 자율 임대 마케팅" 실현

#### E-2. 프롬프트 자동 최적화 (PromptEvolution)
- **현재**: `prompt_version` 수동 관리 (v1.0 고정)
- **R&D**: A/B 테스트 자동화 + 전문가 패치율 기반 프롬프트 자동 개선
  - `ai_runs` 테이블에서 prompt_version별 성공률 자동 집계
  - 성공률 높은 프롬프트 자동 승격
- **사업 가치**: 프롬프트 엔지니어링 인건비 절감 + 품질 지속 향상

## 7. 결론: 독점 솔루션의 핵심 공식

```
[독점 솔루션] = [축적된 딜 데이터] × [도메인 특화 모델] × [법적 가드레일]
```

- **축적된 딜 데이터**: 3개 시스템을 관통한 딜 이력 → 경쟁자가 시간으로도 살 수 없는 자산
- **도메인 특화 모델**: 범용 LLM으로는 도달 불가능한 정확도 + 속도 + 비용 효율
- **법적 가드레일**: P0 금지 패턴 + 면책 자동화 → 규제 산업에서의 신뢰 해자

> **이 세 가지가 결합되면, "한국 상업부동산 AI"라는 카테고리 자체를 정의하는 플레이어가 됩니다.**
