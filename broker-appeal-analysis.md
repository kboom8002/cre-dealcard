> 분석 기준일: 2026-05-11  
> 대상 시스템: cre-dealcard (MVP) · cre-aipage (Space AI Page) · cre-fullim (Full IM Studio)

## 2. 연계 포인트 정밀 분석

### 2-A. cre-dealcard → cre-aipage (매물 메모 → 임대 마케팅 자동화)

**트리거**: 딜카드 결과 화면의 "Space AI 마케팅 시작" 버튼 클릭

**데이터 흐름**:
1. 브로커의 카카오 수준 메모 (`"성수동 2층 38평 보증금5천 월세420"`) 전달
2. `SpaceStructuringAgent`가 AI로 파싱 → `SpaceSSoT` 구조 생성
   - 층수, 전용면적, 보증금, 월세, 관리비, 권리금 자동 추출
   - 추천 임차인 업종 자동 분류 (카페/클리닉/쇼룸/학원 등)
   - 확인 불가 항목 → `missing_data` 목록화 (예: 급배수 확인 필요)
3. Space AI Page에 Space SSoT 초안 자동 생성

**결과물**:
- 즉시 편집 가능한 Space SSoT 페이지 (브로커가 사진만 올리면 됨)
- 임대 조건 공개/비공개 구분 자동 적용

**차별점**: 브로커가 별도 입력 없이 딜카드 메모가 마케팅 자료로 변환됨

---

### 2-B. cre-aipage 자체 AI 에이전트 체인

**핵심 에이전트 13종**:

| 에이전트 | 역할 |
|---|---|
| `SpaceStructuringAgent` | 메모 → 구조화된 SpaceSSoT 변환 |
| `VisualClassificationAgent` | 업로드 사진 AI 분류 (공간유형/시설/분위기) |
| `VisualAlbumAgent` | 임차인 업종별 사진 앨범 자동 구성 |
| `TenantFitAgent` | 업종별 임차인 적합도 평가 (0-100점) |
| `VibeFitAgent` | 공간 분위기·감성 분석 (F&B/쇼룸/클리닉 등) |
| `LeasingPageWriterAgent` | 임대 LP 섹션 자동 작성 |
| `CampaignCopyAgent` | 카카오/네이버/인스타 홍보 문구 자동 생성 |
| `InquiryQualifierAgent` | 문의 임차인 자동 사전 심사 |
| `OwnerReportAgent` | 건물주 대상 임대 현황 자동 보고서 |
| `SafeLanguageAgent` | 과도한 확정 표현 필터링 (법적 리스크 방지) |
| `DisclosureGuardAgent` | 민감 정보 유출 차단 |

**임차인 Fit 평가 보안 가드레일**:
- "가능합니다", "문제없습니다" 등 확정 표현 자동 탐지 및 교체
- 클리닉/학원/F&B 업종 허가 가능성을 단정하는 표현 금지
- 실제 가능성은 법규·설비 확인 후 달라질 수 있음을 항상 부기

---

### 2-C. cre-aipage → cre-dealcard (임대 결과 역반환)

**트리거**: Space AI Page에서 임대 페이지 발행 후 결과 전송

**반환 데이터**:
- 임대 페이지 URL (공개 공유 가능)
- 카카오 공유 문구 (원터치 복사)
- 문의 집계 (총 문의 수, 관심 업종, 우수 문의 수)

**단일사용(Single-Use) 보안**:
- 반환 토큰은 소비 즉시 `consumed_at` 기록 → 재사용 불가
- 문의자 실명/연락처 등 PII는 반환하지 않음 (`protected_fields_removed` 명시)

---

### 2-D. cre-dealcard → cre-fullim (투자설명서 제작 위임)

**트리거**: 딜카드에서 "Full IM 제작 요청" (Handoff Token 생성)

**Handoff 데이터**:
- Building SSoT Lite (건물 기본 정보 스냅샷)
- 공개 허용 범위 (`allowed_import_scope`)
- 요청 유형 (AI 셀프 제작/전문가 검토/전문가 풀빌드)

**Full IM Studio 처리 흐름**:
1. Handoff Token 검증 (만료/취소/중복 체크)
2. `handoff_source_snapshots` 테이블에 불변 스냅샷 저장
3. `building_ssot_full` 생성 (14개 레이어: 자산정체성/물리적사실/법적등기/임대소득/시장입지/가치상승가설/리스크/바이어핏 등)
4. IM 프로젝트 생성 → 7개 챕터 AI 초안 자동 작성
5. 전문가 패치 요청 → DealRoom 게시

**Mobile IM (50-150억 매물 특화)**:
- 7섹션 모바일 최적화 투자설명서 자동 생성
- 원터치 카카오 공유
- Full IM 업그레이드 CTA 내장

---

### 2-E. cre-fullim → cre-aipage (공실 능동 마케팅 트리거)

**트리거**: Full IM 분석 중 "공실/리포지셔닝 가능 공간" 감지 시 버튼

**흐름**:
1. Full IM 프로젝트의 Physical Fact에서 공실 신호 추출
2. Space AI Page에 `space_draft` 자동 생성 요청
3. 브로커가 사진만 추가하면 즉시 임대 마케팅 가능

---

### 2-F. 크로스시스템 통합 분석 대시보드

**위치**: `/admin/cross-system` (MVP 어드민)

**시각화 내용**:
- 3개 시스템의 `activity_events` 통합 퍼널
- "건물 주소 입력 → 딜카드 생성 → Full IM 요청 → IM 완성 → Space AI 마케팅" 전환율
- Handoff 상태 현황 (생성/임포트/만료/취소)

## 3. 60대 꼬마빌딩 브로커 대상 소구 분석

### 대상 프로파일

| 항목 | 특성 |
|---|---|
| 연령대 | 55-70세 |
| 주력 매물 | 꼬마빌딩 (50억-200억, 5-10층 이하) |
| 주요 고객 | 자산가 건물주, 법인 매수자 |
| 페인포인트 | 손으로 메모 → 워드 정리 → 카카오 공유 과정이 너무 오래 걸림 |
| 핵심 니즈 | 빠른 딜 클로징, 건물주 신뢰 유지, 매수자 확보 |
| 기술 수준 | 카카오톡·사진 앱 정도 익숙, 복잡한 입력 거부 |

---

### 소구 포인트 1: "메모 한 줄이면 딜카드 완성"

**기능**: MVP 주소 입력 → 1분 딜카드 + 블라인드 티저 자동 생성  
**AI 역할**: 공시가/실거래가/층별 임대 현황을 AI가 조회해 딜 큐리오시티 리포트 자동 완성

**소구 메시지**:
> "선생님, 주소만 치시면 매물 개요서가 바로 나와요. 이걸 건물주한테 카카오로 바로 보낼 수 있어요."

**기존 업무 대비 효율**:
- 기존: 주소 확인 → 공시가 조회 → 엑셀 정리 → 워드 작성 → 출력 (2-4시간)
- 신규: 주소 입력 → 1분 딜카드 + 공유 링크 (3분)

---

### 소구 포인트 2: "공실 나면 AI가 임차인 찾아줘요"

**기능**: Full IM에서 공실 감지 → Space AI Page 자동 마케팅 트리거  
**AI 역할**: 사진 업로드 → 업종별 적합도 분석 → 카페/클리닉/쇼룸 등 추천 업종 자동 도출

**소구 메시지**:
> "1층 공실 사진만 올리면 AI가 어떤 업종이 들어올 수 있는지 분석해줘요. 그 결과로 임대 홍보 페이지가 자동으로 만들어지고, 문의가 들어오면 자동 정리도 돼요."

**60대 브로커 공감 포인트**:
- 공실 관리는 건물주가 가장 불안해하는 문제
- "내가 대신 해드렸어요"를 AI 산출물로 증명 가능
- 건물주 신뢰 유지의 강력한 도구

---

### 소구 포인트 3: "투자설명서를 AI가 써줘요 — 전문가 검토도 연결돼요"

**기능**: Handoff Token → Full IM Studio 7챕터 투자설명서 AI 초안  
**Mobile IM 특화**: 50-150억 꼬마빌딩에 딱 맞는 모바일 투자설명서

**소구 메시지**:
> "50억짜리 건물 매수자한테 보내는 설명서, AI가 초안 잡아줘요. 선생님이 손으로 고치는 부분만 코멘트 남기시면 돼요. 그 다음 전문가 검토도 연결해 드려요."

**60대 브로커 공감 포인트**:
- 투자설명서 작성이 가장 큰 진입 장벽
- "Full IM 해줄게"라는 말을 실제로 이행할 수 있는 도구
- 경쟁 브로커와의 전문성 차별화

---

### 소구 포인트 4: "건물주한테 카카오로 바로 보내요"

**기능**: Mobile IM 카카오 공유 / 임대 LP URL / 카카오 홍보 문구 자동 생성

**소구 메시지**:
> "임대 페이지 링크 하나를 건물주나 예비 임차인한테 카카오로 보내면 끝이에요. 페이지 안에 사진, 임대 조건, 문의 버튼이 다 들어가 있어요."

**60대 브로커 공감 포인트**:
- 브로커의 주요 소통 채널이 카카오톡
- URL 하나로 전문 마케팅 자료 공유
- 문의가 들어오면 자동 집계 → 건물주에게 "이번 주 문의 3건"으로 보고 가능

---

### 소구 포인트 5: "오래된 매물도 다시 살려요"

**기능**: 공실 신호 감지 → Space AI Page 트리거 → 새 임대 LP 자동 재생성

**소구 메시지**:
> "6개월 넘게 안 나간 매물도 AI가 사진 다시 분석해서 다른 업종으로 타겟 바꿔 홍보 페이지 새로 만들어드려요."

## 4. 60대 브로커를 위한 UX 권장사항

> 현재 코드 기반으로 도출한 UI/UX 개선 권고

| 현재 문제 | 권장 개선 |
|---|---|
| 영어 용어 혼재 ("Handoff", "SSoT") | 모든 브로커 화면에 한국어만 노출 (SSoT → "매물 기본정보") |
| 다단계 입력 프로세스 | 카카오 메시지 수준의 1줄 입력 UI (스텝 최소화) |
| 텍스트 위주 결과 | 딜카드 결과를 카드 형태 1장으로 요약 (숫자+한줄 코멘트) |
| 기술 용어 (AI 에이전트, 신뢰도) | "AI가 확인했어요" / "직접 확인 필요" 2단계로 단순화 |
| 복잡한 Full IM 진입 | "투자설명서 만들기" 버튼 하나만 노출 (내부 흐름 숨김) |

## 5. 브로커 하루 업무 시나리오 (60대 전문 브로커 기준)

```
[오전 9시] 건물주로부터 카카오로 매각 의뢰 연락
      ↓
[9시 5분] cre-dealcard에 주소 입력 → 딜카드 + 블라인드 티저 생성
      ↓
[9시 8분] 건물주에게 딜카드 카카오 전송 → "이렇게 정리해드렸어요"
      ↓
[오전 10시] 매수 후보자에게 블라인드 티저 공유 → 관심 매수자 확인
      ↓
[오후 2시] 매수 의향 확인 → Full IM 제작 요청 버튼 클릭
           → AI가 7챕터 투자설명서 초안 자동 작성 (30분 이내)
      ↓
[오후 3시] 초안 검토 후 전문가 검토 요청 (클릭 한 번)
      ↓
[오후 4시] 1층 공실 확인 → "공실 AI 임대 마케팅 시작" 버튼 클릭
           → 임대 LP 자동 생성 + 카카오 공유 문구 복사
      ↓
[퇴근 전] 건물주에게 "오늘 임대 문의 2건 들어왔어요" 보고 가능
```

**기존 동일 업무**: 2-3일 소요 (서류 작성, 사진 정리, 이메일 발송)  
**신규**: 당일 완결 가능

## 6. 핵심 소구 메시지 요약

> **"선생님 현장 경험에 AI가 속도를 더해드려요. 메모 하나로 매물 개요서, 투자설명서, 임대 홍보 페이지까지 — 선생님은 건물주 만나고, 나머지는 저희가 합니다."**
