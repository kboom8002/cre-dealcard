## 1. 시스템 개요 및 첫 접속

### 1.1 시스템이란?

CRE DealCard는 **상업용 부동산(CRE) 전문 중개사**를 위한 올인원 영업 플랫폼입니다.
매물 등록부터 AI 투자설명서 자동 생성, 고객 매칭, 딜 파이프라인 추적,
주간 매거진 발행, Vibe AI 명함까지 — 중개사의 전체 영업 사이클을 하나의 시스템에서 관리합니다.

### 1.2 첫 접속 및 로그인

| 단계 | 작업 |
|------|------|
| ① | `https://credeal.net/login` 접속 |
| ② | 이메일/비밀번호 또는 소셜 로그인 |
| ③ | 최초 접속 시 `/onboarding` 페이지에서 프로필 사진 업로드 및 기본 정보 입력 |
| ④ | 온보딩 완료 후 `/broker` 대시보드로 자동 이동 |

### 1.3 모바일 하단 네비게이션 (5개 탭)

| 아이콘 | 탭 이름 | 경로 | 용도 |
|--------|---------|------|------|
| 🏠 | **홈** | `/broker` | 대시보드 코크핏 |
| 🏢 | **매물** | `/broker/buildings` | 보유 매물 목록 및 관리 |
| ⚡ | **매칭** | `/broker/matching` | AI 자동 매칭 결과 확인 |
| 👥 | **고객** | `/broker/clients` | CRM 고객 관리 |
| 👤 | **프로필** | `/broker/profile` | 프로필 편집 및 Vibe 명함 |

## 2. 브로커 코크핏 (대시보드)

> **경로**: `/broker`

대시보드는 **3개 탭**으로 구성되어 있으며, 로그인 직후 가장 먼저 보이는 핵심 허브입니다.

### 2.1 개요(Overview) 탭

#### KPI 그리드 (핵심 성과 지표)

| 지표 | 의미 | 예시 |
|------|------|------|
| 보유 매물 | 등록된 빌딩 수 | 12건 |
| 임대 공간 | 관리 중인 임대 스페이스 | 8건 |
| 매수 고객 | 등록된 매수 의향 고객 | 25명 |
| 임차 고객 | 등록된 임차 의향 고객 | 15명 |
| CRM 고객 | 전체 관리 고객 수 | 40명 |
| S급 매칭 | AI 최상위 등급 매칭 수 | 3건 |
| A급 매칭 | AI 우수 등급 매칭 수 | 7건 |
| 평균 보유일 | 매물 평균 보유 기간 | 39일 |

#### 오늘의 일정
- 금일 확정/보류 상태의 임장 예약이 시간순으로 표시됩니다.
- 매물 주소, 방문자 정보, 상태 뱃지(확정/보류)가 함께 나타납니다.

#### ROI 카드
- 월간 투자 수익률(ROI)이 시각적으로 표시됩니다.

#### 알림 피드
최근 5건의 주요 이벤트가 실시간으로 표시됩니다:

| 아이콘 | 이벤트 | 설명 |
|--------|--------|------|
| 🔒 | Gate 주소 요청 | 신규 상세 주소 열람 요청 접수 |
| 📅 | Gate 승인 완료 | 주소 열람 승인 → 즉시 임장 예약 가능 |
| 🎯 | 매수 고객 등록 | 신규 매수 의향서 자동 등록 |
| ✨ | S/A급 매칭 발생 | AI가 새로운 고품질 매칭을 발견 |
| 📈 | 파이프라인 변경 | 딜 단계가 전환됨 |
| 🔒 | IM 정밀 열람 감지 | 고객이 투자설명서를 60초 이상 열람 |
| 🔥 | **Hot Lead 감지** | 교차 채널 스코어 80점 이상 고객 발견 |
| 🔔 | 매거진 구독자 | 새로운 매거진 구독 신청 접수 |

### 2.2 인텔리전스(Intelligence) 탭

**모닝 인텔리전스** AI가 매일 아침 제공하는 시장 브리핑입니다.

- 권역별 시장 동향 요약
- 주목할 거래 사례
- 커스텀 인텔리전스 질의 기능 (자유 질문 → AI 답변)
- 복수 인텔리전스 소스 통합 분석

### 2.3 전략(Strategy) 탭

- **시장 대응 전략 모드**: 현재 시장 방향(상승/보합/하락)에 따른 대응 전략 제시
- **주간 리포트**: 주간 실적 요약
- **월간 리포트**: 월간 성과 분석

### 2.4 유니버셜 메모 FAB

대시보드 우측 하단에 항상 떠 있는 **플로팅 액션 버튼(FAB)**입니다.
어떤 페이지에 있든 즉시 메모를 작성할 수 있습니다.

## 3. 매물 등록 및 관리

> **경로**: `/broker/buildings`

### 3.1 매물 목록 페이지

4개 탭으로 구성된 종합 매물 관리 화면입니다.

| 탭 | 용도 |
|----|------|
| **전체 매물** | 모든 등록 매물의 그리드/리스트 뷰 + 검색/필터 |
| **활성 딜** | 현재 진행 중인 딜만 필터링 |
| **IM 관리** | 생성된 모바일 투자설명서 목록 및 상태 관리 |
| **임대** | 임대 스페이스 관리 |

### 3.2 매물 등록 절차

1. **주소 입력**: 건물 주소 입력 → 자동 좌표 변환
2. **기본 정보**: 건물명, 자산유형(꼬마빌딩/물류센터/리테일 등), 연면적, 대지면적
3. **재무 정보**: 호가, NOI, 캡레이트, 임대차 정보
4. **사진 업로드**: 건물 외관 및 내부 사진
5. **추가 정보**: 브로커 메모, 특이사항

### 3.3 매물 카드 정보

각 매물 카드에는 다음이 표시됩니다:
- 건물명 및 주소
- 자산유형 뱃지
- 호가(억 원) / 캡레이트(%)
- 딜 상태 뱃지 (활성/보류/종료)
- 빠른 액션: IM 보기, 편집, 스튜디오 이동

## 5. 모바일 투자설명서(IM) 생성

> **생성**: `/broker/buildings/[id]/im-lite`
> **열람(공개)**: `/im-lite/[buildingId]`

### 5.1 IM이란?

**Investment Memorandum(투자설명서)**을 모바일 최적화 7섹션으로 AI가 자동 생성합니다.
브로커가 매물 정보만 입력하면, AI가 전문 투자보고서 수준의 IM을 생성합니다.

### 5.2 IM 7개 섹션

| # | 섹션 | 내용 |
|---|------|------|
| ① | **매물 개요** | 건물 요약, 위치, 규모, 자산유형 |
| ② | **투자 논거** | 핵심 투자 포인트, 차별화 요인 |
| ③ | **건물 분석** | 구조, 설비, 상태, 준공연도 |
| ④ | **재무 분석** | NOI, 캡레이트, 수익률 시뮬레이션 |
| ⑤ | **입지 분석** | 교통, 상권, 인구, 개발호재 |
| ⑥ | **리스크 평가** | 잠재 리스크 및 완화 방안 |
| ⑦ | **실행 계획** | 거래 절차, 추천 일정, 다음 단계 |

### 5.3 물류센터 특화 기능

매물 유형이 **물류센터 또는 창고**인 경우, AI가 자동으로 다음 17개 특화 지표를 분석합니다:

| 카테고리 | 특화 지표 |
|----------|----------|
| **구조** | 유효 천장고(m), 바닥 하중(톤/㎡), 램프 구조 |
| **접안** | 도크 수, 도크 레벨러 유무, 접안 규격 |
| **온도** | 냉동 전용 면적(평), 냉장 전용 면적(평) |
| **접근성** | 최근접 IC 거리(km), 대형 차량 진입 가능 톤수 |
| **안전** | 소방 등급 |

### 5.4 IM 승인 워크플로우

```
IM 생성 요청 → AI 초안 생성 → 브로커 검토/편집 → 승인 → 공개 발행
```

- **초안(Draft)**: AI가 생성한 첫 버전
- **승인 대기**: 브로커가 검토 중
- **발행(Published)**: 공개 URL로 고객에게 공유 가능

### 5.5 IM 공유

발행된 IM은 다음 URL로 누구에게나 공유할 수 있습니다:
```
https://credeal.net/im-lite/[buildingId]
```

공유 받은 고객이 60초 이상 열람하면 **자동으로 카카오 알림톡**이 발송됩니다.

## 6. 딜 파이프라인 관리

> **경로**: `/broker/pipeline`

### 6.1 파이프라인 6단계

칸반(Kanban) 보드 형태로 딜의 진행 상태를 시각적으로 관리합니다.

```
발굴 → 분석 → 제안 → 협상 → 계약 → 완료
```

| 단계 | 의미 | 주요 활동 |
|------|------|----------|
| **발굴** | 신규 매물/리드 발견 | 매물 등록, 초기 조사 |
| **분석** | 투자 분석 진행 | IM 생성, 재무 분석 |
| **제안** | 매수 고객에게 제안 | 매칭 결과 활용, IM 공유 |
| **협상** | 조건 협의 중 | 가격 협상, 조건 조율 |
| **계약** | 계약 진행 중 | 계약서 작성, 법률 검토 |
| **완료** | 거래 종결 | 잔금, 등기 이전 |

### 6.2 주요 기능

- **드래그 앤 드롭**: 딜 카드를 단계 간 이동
- **딜 카드 정보**: 매물명, 호가, 캡레이트, 해당 단계 체류일수
- **자산유형/권역/가격 필터링**
- **단계별 통계**: 각 단계의 딜 수와 총액
- **전환 이력 자동 기록**: 모든 단계 변경이 `pipeline_stage_transitions`에 기록

## 7. AI 자동 매칭 엔진

> **경로**: `/broker/matching`

### 7.1 매칭 등급 체계

AI가 매수 고객의 투자 기준과 보유 매물을 자동으로 교차 분석합니다.

| 등급 | 적합도 | 의미 |
|------|--------|------|
| **S** | 90%+ | 완벽한 매칭 — 즉시 연결 추천 |
| **A** | 75~90% | 우수한 매칭 — 적극 제안 가능 |
| **B** | 60~75% | 보통 — 조건 조율 시 가능 |
| **C** | 60% 미만 | 약한 매칭 — 참고용 |

### 7.2 매칭 유형

- **매매 매칭**: 매수 고객 ↔ 매물 (match_results)
- **임대 매칭**: 임차 고객 ↔ 임대 공간 (lease_match_results)

### 7.3 매칭 카드 정보

각 매칭 카드에는 다음이 표시됩니다:
- 고객 정보 (이름, 투자 기준)
- 매물 정보 (주소, 가격, 수익률)
- 매칭 등급 뱃지
- **AI 매칭 사유**: 왜 이 조합이 적합한지 AI가 서술
- 빠른 액션: 고객 연락, 매물 상세, 딜 생성

## 8. 고객 CRM 관리

### 8.1 고객 목록 (`/broker/clients`)

CRM 스타일의 고객 관리 허브입니다.

- **고객 유형**: 매수자, 매도자, 임차인, 투자자
- **검색 및 필터**: 이름, 유형, 최근 연락일 기준
- **고객 카드**: 이름, 회사, 유형, 연락처, 최근 연락일, 진행 중 딜 수

### 8.2 매수 고객 의향서 (`/broker/buyer-intents`)

| 입력 항목 | 설명 |
|----------|------|
| 고객 정보 | 이름, 회사, 연락처 |
| 희망 자산유형 | 꼬마빌딩, 물류센터, 리테일 등 (복수 선택) |
| 투자 예산 | 최소~최대 (억 원) |
| 희망 권역 | 강남, 성수, 인천 등 (복수 선택) |
| 기대 수익률 | 희망 캡레이트(%) |
| 투자 시기 | 즉시/3개월/6개월/1년 |
| 특이사항 | 자유 메모 |

### 8.3 임차 고객 의향서 (`/broker/tenant-intents`)

| 입력 항목 | 설명 |
|----------|------|
| 업체 정보 | 업체명, 업종 |
| 희망 면적 | 최소~최대 (평) |
| 월 임대료 예산 | (만 원) |
| 희망 위치 | 선호 권역 |
| 임대 기간 | 희망 계약 기간 |
| 특수 요건 | 도크, 천장고, 냉동시설 등 |

### 8.4 컨택 이력

각 고객별로 연락 이력이 타임라인으로 기록됩니다:
- 전화, 문자, 미팅 기록
- 자동 생성 이벤트 (매칭 알림, IM 공유 등)

## 9. 음성/텍스트 메모 시스템

> **경로**: `/broker/memos`

### 9.1 메모 유형

| 유형 | 입력 방식 |
|------|----------|
| **텍스트 메모** | 직접 타이핑 |
| **음성 메모** | 마이크 녹음 → AI 자동 전사 |

### 9.2 태그 분류

메모를 카테고리별로 태그합니다: `임장`, `고객`, `시장`, `매물`

### 9.3 AI 자동 라우팅 ⚡

메모의 **핵심 혁신 기능**입니다. 하나의 메모를 AI가 분석하여 자동으로 적절한 레코드를 생성합니다:

```
"성수동 꼬마빌딩 60억 캡 4.5% 오늘 임장했는데 상태 좋음.
 김 사장이 45~55억 성수동 빌딩 찾고 있다고 연락 옴."
```

위 메모 하나로 AI가 자동 생성:
- ✅ **딜카드**: 성수동 꼬마빌딩 매물 정보
- ✅ **매수 의향서**: 김 사장 45~55억 성수동 빌딩 투자 의향

### 9.4 사용 팁

> **임장 직후** 차 안에서 음성 메모를 녹음하면,
> AI가 매물 정보와 고객 요청을 자동으로 분리·정리·등록합니다.
> **1분 음성 녹음 = 10분 수작업 절감** 효과를 기대할 수 있습니다.

## 11. 프로필 및 Vibe 명함 관리

### 11.1 프로필 편집 (`/broker/profile`)

11개 섹션으로 구성된 종합 프로필 관리 페이지입니다.

| # | 섹션 | 내용 |
|---|------|------|
| ① | 기본 정보 | 이름, 회사, 전화, 이메일, 한줄 소개 |
| ② | 프로필 사진 | 드래그 앤 드롭 사진 업로드 |
| ③ | 전문 분야 | 꼬마빌딩, 물류센터, 리테일, 오피스 등 칩 선택 |
| ④ | 자기 소개 | 장문 소개 (AI 자동 생성 가능) |
| ⑤ | 경력 사항 | 회사명, 직책, 기간 입력 |
| ⑥ | 학력 | 학교, 전공, 학위 |
| ⑦ | 자격증 | 전문 자격 사항 |
| ⑧ | 수상/인증 | 수상 및 인증 이력 |
| ⑨ | 미디어 출연 | TV, 언론 출연 이력 |
| ⑩ | SNS 링크 | 네이버 블로그, 유튜브, 링크드인, 카카오 채널 |
| ⑪ | Vibe 명함 | 명함 생성 버튼 및 미리보기 링크 |

### 11.2 Vibe AI 명함 생성 (`/broker/my-card/new`)

3단계 위저드로 AI 명함을 생성합니다:

| 단계 | 작업 | 소요 시간 |
|------|------|----------|
| **Step 1** | 프로필 사진 업로드 → AI 인상 분석 | ~30초 |
| **Step 2** | 이름, 회사, 전문분야 확인 | ~1분 |
| **Step 3** | AI가 Vibe 명함 자동 생성 | ~10초 |

### 11.3 공개 Vibe 명함 (`/vibe-card/[slug]`)

생성된 명함은 공개 URL로 누구에게나 공유됩니다.
명함에 표시되는 정보:

- 프로필 사진 (글로우 링 효과)
- 이름, 회사, 한줄 소개
- AI 분석 성격 유형 (예: 🎯 ENTJ - 전략형 딜메이커)
- 활성 매물 수, 평균 캡레이트, 매칭 수
- **보유 매물 IM 리스트** (최대 3건, 딥링크 포함)
- **최신 주간 매거진** (발행된 경우)
- 전문 분야 태그
- 경력/학력/자격증/수상 이력
- 전화 상담 / 카카오톡 CTA 버튼
- 공유 기능 (카카오톡, 링크 복사, QR코드)
