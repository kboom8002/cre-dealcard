## 1. 무마찰 영업 퍼널 — 전체 그림

CRE DealCard의 핵심 철학은 **"IM 하나 만들면, 나머지는 시스템이 알아서"** 입니다.

### 1.1 콘텐츠 자동 순환 흐름

```
 ┌──────────────────────────────────────────────────────────────────┐
 │                    브로커의 단 하나의 입력                        │
 │              매물 임장 → 음성 메모 → IM 생성 요청                  │
 └───────────────────────┬──────────────────────────────────────────┘
                         ▼
              ┌─────────────────────┐
              │   AI 모바일 IM 생성   │  ← 7섹션 자동 작성
              │   (투자설명서)        │  ← 물류센터면 특화 분석 추가
              └─────────┬───────────┘
                        │
          ┌─────────────┼─────────────────┐
          ▼             ▼                 ▼
   ┌─────────────┐ ┌─────────────┐ ┌──────────────┐
   │ Vibe 명함에  │ │ 주간 매거진에 │ │ 딜카드에     │
   │ IM 링크 자동 │ │ 소재 자동    │ │ IM 링크 자동 │
   │ 노출 (최대3건)│ │ 추출·적재    │ │ 연결         │
   └──────┬──────┘ └──────┬──────┘ └──────┬───────┘
          │               │               │
          └───────┬───────┘               │
                  ▼                       ▼
          ┌──────────────┐        ┌──────────────┐
          │ 고객 열람 시   │        │ 매수 고객     │
          │ 교차 채널     │        │ 자동 매칭     │
          │ 스코어 계산   │        └──────────────┘
          └──────┬───────┘
                 ▼
          ┌──────────────┐
          │ Hot Lead!    │
          │ 카카오 알림톡 │  ← 스코어 80점+ 달성 시
          │ 즉시 발송    │
          └──────────────┘
```

### 1.2 브로커 1분 워크플로우 요약

| 단계 | 브로커 행동 | 시스템 자동 처리 |
|------|-----------|----------------|
| ① | 임장 후 음성 메모 녹음 | AI가 매물 정보 + 고객 요청 자동 분리·등록 |
| ② | IM 생성 버튼 클릭 | 7섹션 전문 IM 자동 생성 |
| ③ | IM 승인 | Vibe 명함에 매물 자동 노출 + 매거진 소재 자동 적재 |
| ④ | 매거진 필드노트 입력 (3분) | AI 주간 매거진 자동 생성 + 구독자 배포 |
| ⑤ | — (대기) | 고객 열람 분석 → Hot Lead 알림 자동 수신 |

## 2. 주간 매거진 발행 가이드

> **에디터 경로**: `/broker/magazine-editor`
> **공개 URL**: `/magazine/[brokerId]/[date]`

### 2.1 매거진 구성 (6개 MVP 섹션)

| # | 섹션 | 콘텐츠 소스 | 브로커 입력 필요 여부 |
|---|------|-----------|---------------------|
| ① | **커버** | 시장온도 + 3키워드 + 브로커 코멘트 | ✅ 필수 (약 30초) |
| ② | **AI 브리핑** | AI 자동 생성 | ❌ 자동 |
| ③ | **필드노트** | 브로커 5필드 구조화 입력 | ✅ 필수 (약 2분) |
| ④ | **금주의 테마** | AI 테마 + 연관 매물 자동 선택 | ❌ 자동 (수정 가능) |
| ⑤ | **추천 매물** | 등록된 IM에서 자동 추출 | ❌ 자동 |
| ⑥ | **브로커 프로필** | Vibe 명함 미니카드 | ❌ 자동 |

### 2.2 매거진 에디터 5탭 사용법

#### 탭 1: 커버 설정

| 항목 | 설명 | 예시 |
|------|------|------|
| 시장 온도 | 5단계 선택 | 🔥 적극 매수 / 📈 선별 매수 / 📊 관망 / ❄️ 조정 대기 / 🚨 위기 경계 |
| 3대 키워드 | 이번 주 핵심 키워드 3개 | "성수 리노베이션", "물류 수요 증가", "공실 감소" |
| 커버 이미지 | 대표 이미지 URL | (선택사항) |
| 브로커 코멘트 | 독자에게 보내는 한마디 | "이번 주 성수동 물류센터 2건 신규 진입했습니다" |

#### 탭 2: 필드노트 (핵심 ⭐)

브로커의 현장 감각을 전달하는 **가장 중요한 입력**입니다.

| 필드 | 질문 | 입력 시간 | 예시 |
|------|------|----------|------|
| 주간 시장 한 문장 | 이번 주 시장 분위기를 한 문장으로? | ~20초 | "수요자 문의 증가, 가격 눈치보기 중" |
| 매수 반응 | 매수자들의 반응은? | ~20초 | "50억 이하 물류센터 적극 관심" |
| 매도 반응 | 매도자들의 반응은? | ~20초 | "호가 조정 의향 소폭 증가" |
| 시장 판단 | 전문가로서의 판단은? | ~40초 | "하반기 금리 인하 기대감에 거래 재개 신호" |
| 독자에게 한마디 | 커버에 노출될 메시지 | ~20초 | "좋은 물건 나오면 빠르게 연락드리겠습니다" |

> 💡 **총 예상 입력 시간: 약 2분** — 나머지(AI 브리핑, 테마, 추천 매물)는 AI가 자동 생성합니다.

#### 탭 3: 테마 & 매물

- 이번 주의 투자 테마를 설정합니다.
- 등록된 활성 매물(building_ssot_lite)에서 관련 매물을 선택합니다.
- 선택된 매물은 IM 딥링크(`/im-lite/[buildingId]`)와 함께 표시됩니다.

#### 탭 4: 뉴스 큐레이션

- 시스템이 수집한 외부 뉴스(중요도순 상위 20건)에서 이번 주에 소개할 뉴스를 선택합니다.
- 토픽 필터: 전체, 시장동향, 거래, 정책, 경매, 임대

#### 탭 5: 발행 설정

- **발행 상태**: Draft → Editing → Review → Scheduled → **Published** → Archived
- 발행 시 구독자에게 자동으로 카카오 알림톡이 배포됩니다.
- 카카오 공유 및 링크 복사 기능

### 2.3 매거진 구독자 관리

세 가지 채널(Vibe 명함, IM, 매거진)의 하단에 배치된 **구독 카드**를 통해
방문자가 이름과 전화번호만 입력하면 자동으로 구독 신청이 처리됩니다.

```
구독 신청 → magazine_subscribers 등록
         → 신규 발행 시 카카오 알림톡 자동 배포
         → 대시보드에 "🔔 매거진 신규 구독자" 알림 표시
```

### 2.4 타겟 세그먼트 렌더링

매거진 URL에 쿼리 파라미터를 추가하면 독자 유형별 최적화된 레이아웃이 표시됩니다:

| URL | 대상 | 레이아웃 차이 |
|-----|------|-------------|
| `/magazine/slug/date` | 일반 | 표준 MVP 순서 |
| `/magazine/slug/date?target=buyer` | 매수자 | 매물 + 테마 상단 배치, 시장데이터 접기 |
| `/magazine/slug/date?target=seller` | 매도자 | 시장데이터 + 실거래 상단 펼침 |

## 3. Vibe Card × IM × Magazine 교차 활용

### 3.1 각 채널의 역할

| 채널 | 공개 URL | 핵심 목적 | 주요 독자 |
|------|---------|----------|---------|
| **Vibe 명함** | `/vibe-card/[slug]` | 브로커 브랜딩 + 신뢰 구축 | 신규 고객, 네트워킹 |
| **모바일 IM** | `/im-lite/[buildingId]` | 매물별 전문 투자 분석 | 관심 매수자 |
| **주간 매거진** | `/magazine/[id]/[date]` | 시장 인사이트 + 매물 큐레이션 | 구독자, 잠재 고객 |

### 3.2 교차 링크 구조

```
 ┌─────────────────────────────────────────────────────────────┐
 │                                                             │
 │   Vibe 명함                                                 │
 │   ┌──────────────────────────────────────────────────────┐  │
 │   │ 프로필 사진 + AI 성격 분석                             │  │
 │   │ 경력/자격증/수상                                      │  │
 │   │                                                      │  │
 │   │ 📋 보유 매물 IM (최대 3건)  ─────────→  /im-lite/xxx  │  │
 │   │ 📰 최신 매거진 리포트     ─────────→  /magazine/xxx   │  │
 │   │ 🔔 주간 매거진 구독 CTA                               │  │
 │   └──────────────────────────────────────────────────────┘  │
 │                                                             │
 │   모바일 IM (투자설명서)                                      │
 │   ┌──────────────────────────────────────────────────────┐  │
 │   │ 7섹션 투자 분석                                       │  │
 │   │ Vibe 명함 브랜드 색상이 자동 반영됨                     │  │
 │   │                                                      │  │
 │   │ 👤 브로커 프로필 카드   ─────────→  /vibe-card/slug   │  │
 │   │ 🔔 주간 매거진 구독 CTA                               │  │
 │   └──────────────────────────────────────────────────────┘  │
 │                                                             │
 │   주간 매거진                                                │
 │   ┌──────────────────────────────────────────────────────┐  │
 │   │ AI 시장 분석 + 필드노트                               │  │
 │   │                                                      │  │
 │   │ 📄 주목 매물 "투자설명서"  ─────────→  /im-lite/xxx   │  │
 │   │ 👤 브로커 미니명함       ─────────→  /vibe-card/slug  │  │
 │   │ 🔔 주간 매거진 구독 CTA                               │  │
 │   └──────────────────────────────────────────────────────┘  │
 │                                                             │
 └─────────────────────────────────────────────────────────────┘
```

### 3.3 통합 브랜딩

Vibe 명함에서 설정된 브랜드 테마(배경 그라디언트, 강조 색상, 링 글로우 효과)가
**IM 뷰어의 히어로 영역과 브로커 프로필 카드**에 자동으로 반영됩니다.

이를 통해 고객은 어떤 채널에서 브로커를 접하든 **일관된 브랜드 경험**을 얻습니다.

### 3.4 콘텐츠 자동 추출 (IM → 매거진 브릿지)

IM이 생성되면 `im-to-magazine-bridge`가 자동으로 다음을 추출합니다:

- 한 줄 투자 포인트 요약
- 블라인드 매물명
- 자산유형 및 가격대
- 대표 사진

이 정보는 매거진의 **"추천 매물"** 섹션에 즉시 활용 가능합니다.

## 4. 교차 채널 리드 스코어링 & Hot Lead 알림

### 4.1 리드 스코어 산정 방식

시스템은 방문자의 행동을 14일간 추적하여 가중 점수를 산정합니다.

| 행동 | 점수 | 의미 |
|------|------|------|
| Vibe 명함 조회 | +5 | 기본 관심 |
| 명함 → IM 클릭 | +15 | 매물 관심 확인 |
| 명함 → 매거진 클릭 | +10 | 시장 관심 |
| 매거진 조회 | +10 | 콘텐츠 소비 |
| 매거진 구독 | +20 | 높은 관심 표명 |
| 매거진 → IM 클릭 | +25 | 구매 의향 신호 |
| IM 열람 | +25 | 투자 검토 중 |
| IM → 명함 클릭 | +10 | 브로커 신뢰 확인 |
| **멀티채널 보너스** | **+30** | 3채널 모두 접촉 시 |

### 4.2 Hot Lead 임계점

```
총 스코어 ≥ 80점  →  🔥 Hot Lead 감지!
                  →  카카오 알림톡 즉시 발송
                  →  대시보드 알림 피드에 표시
```

### 4.3 Hot Lead 알림 메시지 예시

```
[CRE Deal] 🔥 Hot Lead 감지!

리드 스코어: 95점
접촉 채널: Vibe 명함, 투자설명서(IM), 위클리 매거진
조회 매물수: 2건

대시보드에서 매칭 현황을 확인하세요.
→ https://credeal.net/broker/funnel
```

### 4.4 IM 정밀 열람 알림

Hot Lead와 별도로, 고객이 특정 IM을 **60초 이상** 정밀 열람하면
브로커에게 즉시 카카오 알림톡이 발송됩니다.

```
[CRE Deal] 📄 투자설명서 정밀 열람 감지

매물: 성수동 꼬마빌딩
열람 시간: 약 3분 12초

즉시 대시보드에서 확인하세요.
```

### 4.5 중복 알림 방지

- 동일 방문자에 대해 **24시간 내 중복 알림은 발송되지 않습니다**.
- 방문자는 익명화된 핑거프린트(UA + IP 해시)로 추적되며, 개인정보는 저장되지 않습니다.

## 5. 콘텐츠 스튜디오 활용

> **경로**: `/broker/studio`

### 5.1 스튜디오 기능 일람 (8개 아코디언)

| # | 기능 | 설명 | 활용 빈도 |
|---|------|------|----------|
| ① | **뉴스레터 큐레이션** | 딜카드 + 뉴스 선택 → 매거진 에디터로 전송 | 주 1회 |
| ② | **AI 코멘트 비서** | 간단한 아이디어 입력 → 전문 브로커 코멘트 생성 | 수시 |
| ③ | **화이트라벨 매거진** | 커스텀 슬러그/제목/테마 색상 설정 및 공유 | 초기 1회 |
| ④ | **매각준비도 발송** | 건물주에게 매각준비도 점검 결과 카카오 공유 | 필요 시 |
| ⑤ | **공동중개 제안** | 동업 중개사에게 협업 제안서 발송 | 필요 시 |
| ⑥ | **매수자 관심 점수** | 리드 스코어 기반 상위 관심 고객 확인 | 수시 |
| ⑦ | **협력 벤더 리뷰** | 감정평가, 법무, 세무 협력사 관리 | 필요 시 |

### 5.2 AI 코멘트 비서 활용 예시

```
입력: "성수동 물류센터 천장고 높고 도크 많아서 좋음"

AI 변환 결과:
```

## 6. 행동 퍼널 분석

> **경로**: `/broker/funnel`

### 6.1 퍼널 6단계

```
딜카드 생성 → 딜카드 발송 → 고객 열람 → Gate 통과 → 임장 예약 → 미팅/계약
```

| 단계 | 데이터 소스 | 의미 |
|------|-----------|------|
| 생성 | building_ssot_lite 카운트 | 등록된 총 딜카드 수 |
| 발송 | activity_events (카카오 공유, 링크 복사) | 고객에게 공유된 수 |
| 열람 | activity_events (im_lite_view) | 실제 열람된 수 |
| Gate | gate_requests 카운트 | 상세 주소 열람 요청 수 |
| 예약 | bookings 카운트 | 임장 예약 수 |
| 계약 | pipeline (buyer_meeting~closed) | 미팅 이상 진행된 수 |

### 6.2 기간 필터

- 7일 / 30일 / 90일 / 전체 기간으로 분석 가능

### 6.3 AI 인사이트

퍼널 전환율에 따라 AI가 자동으로 개선 팁을 제공합니다:

- 열람율 낮음 → "딜카드 제목과 블라인드 네임을 더 매력적으로 수정해 보세요"
- Gate 전환율 낮음 → "투자 핵심 포인트를 딜카드 상단에 강조하세요"

## 9. 물류센터 전문 브로커를 위한 팁

### 9.1 물류센터 IM 특화 분석

매물의 자산유형이 `물류센터`, `창고`, `warehouse`, `logistics`로 설정되어 있으면,
AI가 자동으로 **17개 물류 특화 지표**를 분석합니다.

#### 섹션별 추가 분석 항목

| IM 섹션 | 물류센터 전용 분석 |
|---------|-------------------|
| **매물 개요** | 천장고, 도크 수/레벨러, 최대 차량 톤수, 바닥 하중, 기둥 간격, 냉동/냉장(유형+면적), 전력 용량 |
| **입지 분석** | 고속도로 IC 접근성, 차량 진입 유형(램프/도크/복합), 트레일러 회전반경 |
| **투자 포인트** | 권역 물류 시장 트렌드, e-커머스 풀필먼트/3PL 수요 분석, 냉동/냉장 프리미엄 |
| **리스크 체크** | 물류시설 등록/인허가, 소방 안전(스프링클러+소방 등급), 냉동 설비 노후도, 전기 용량 한계 |

### 9.2 물류센터 스펙 입력 가이드

IM 생성 시 보조 입력(`supplemental.logistics`)에 다음을 채워주세요:

| 필드 | 단위 | 예시 |
|------|------|------|
| ceiling_height_m | 미터 | 12.5 |
| dock_count | 개 | 8 |
| dock_leveler | boolean | true |
| floor_load_ton_m2 | 톤/㎡ | 5.0 |
| cold_storage_pyeong | 평 | 150 |
| frozen_storage_pyeong | 평 | 80 |
| nearest_ic | 문자열 | "서울외곽 군포IC 3.2km" |
| truck_access_ton | 톤 | 25 |
| fire_class | 문자열 | "1급" |
| ramp_structure | 문자열 | "양면 램프" |

> 💡 **전용 필드를 채울수록 AI의 분석 품질이 향상됩니다.**
> 특히 `ceiling_height_m`, `dock_count`, `nearest_ic`는 투자자 의사결정에 핵심적인 지표입니다.

## 10. 실전 시나리오별 활용 플레이북

### 시나리오 A: 신규 매물 영업 시작

```
1. 임장 → 음성 메모 녹음 (1분)
   └→ AI가 매물 정보 자동 추출 → 딜카드 초안 생성

2. 딜카드 검토·보완 → IM 생성 요청 (클릭 1회)
   └→ 7섹션 모바일 IM 자동 생성 (물류센터면 특화 분석 포함)

3. IM 승인 (검토 후 클릭 1회)
   └→ Vibe 명함에 자동 노출 + 매거진 소재 자동 적재

4. 카카오톡으로 IM 공유 + Vibe 명함 공유

5. 대기 → Hot Lead 알림 수신 시 즉시 연락
```

### 시나리오 B: 주간 매거진 발행

```
월요일 아침:
1. 모닝 인텔리전스 확인 (2분)
2. 매거진 에디터 접속
   - 필드노트 5필드 입력 (2분)
   - 커버 설정 (30초)
   - 뉴스 선택 (1분)
3. "발행" 버튼 클릭
   └→ 전체 구독자에게 카카오 알림톡 자동 배포
   └→ Vibe 명함에 최신 호 자동 반영

총 소요시간: 약 5~6분
```

### 시나리오 C: Hot Lead 대응

```
1. 카카오 알림톡 수신: "🔥 Hot Lead 감지! 스코어 95점"
2. 대시보드 → 퍼널 분석 확인
   - 어떤 매물을 조회했는지
   - 어떤 채널을 통해 접촉했는지
3. 해당 매물의 IM을 바로 카카오로 재공유
4. 전화 연결 → 맞춤 상담
```

### 시나리오 D: 매수 고객 신규 등록 → 자동 매칭

```
1. 매수 고객 미팅 후 의향서 등록
   - 희망 자산유형, 예산, 권역, 수익률 입력

2. AI 자동 매칭 실행
   └→ S/A/B/C 등급 매칭 결과 자동 생성

3. S급 매칭 확인 → 해당 매물의 IM 카카오 공유

4. 고객 열람 → 60초 이상 시 알림 수신
```

## 11. 알림 인프라 설정 (관리자)

카카오 알림톡 실발송을 위해서는 Solapi API 환경변수 설정이 필요합니다.

### 11.1 환경변수

| 변수 | 설명 |
|------|------|
| `SOLAPI_API_KEY` | Solapi API 키 |
| `SOLAPI_API_SECRET` | Solapi API 시크릿 |
| `SOLAPI_SENDER_PHONE` | 발신 전화번호 |
| `SOLAPI_PFID` | 카카오 채널 프로필 ID |

### 11.2 사전 등록 필요 템플릿

| 템플릿 ID | 용도 |
|----------|------|
| `TPL_IM_VIEW_ALERT` | IM 60초+ 열람 시 브로커 알림 |
| `TPL_HOT_LEAD` | Hot Lead 80점+ 감지 시 브로커 알림 |
| `TPL_MAGAZINE_NEW_ISSUE` | 매거진 신규 발행 시 구독자 배포 |

### 11.3 스텁 모드

환경변수가 설정되지 않은 경우, 시스템은 **실제 발송 없이 로그만 출력**합니다.
개발 및 테스트 환경에서는 콘솔 로그로 발송 내용을 확인할 수 있습니다.

## 12. 용어 사전

| 용어 | 설명 |
|------|------|
| **IM (Investment Memorandum)** | 투자설명서. AI가 7섹션으로 자동 생성하는 매물별 전문 분석 보고서 |
| **Vibe Card** | AI가 분석한 브로커 성격 유형과 포트폴리오를 담은 디지털 명함 |
| **VTI (Vibe Type Indicator)** | AI가 프로필 사진과 경력을 분석하여 산출한 브로커 성격 유형 |
| **딜카드** | 하나의 매물(빌딩)에 대한 핵심 정보를 정리한 카드형 레코드 |
| **Gate** | 블라인드 딜카드의 상세 주소를 열람하기 위한 요청/승인 절차 |
| **Hot Lead** | 교차 채널 리드 스코어 80점 이상의 고관심 잠재 고객 |
| **리드 스코어** | Vibe 명함, IM, 매거진 3채널의 방문자 행동을 가중 합산한 관심도 점수 |
| **필드노트** | 브로커가 현장 감각을 5개 구조화된 질문에 답하는 매거진 핵심 입력 |
| **파이프라인** | 매물의 거래 진행 단계를 추적하는 칸반 보드 (발굴→분석→제안→협상→계약→완료) |
| **WALE** | 가중평균잔여임대기간 (Weighted Average Lease Expiry). 임대차 안정성 지표 |
| **캡레이트(Cap Rate)** | 순영업소득(NOI) ÷ 매입가격 × 100. 수익형 부동산의 핵심 수익률 지표 |
| **NOI (Net Operating Income)** | 순영업소득. 총 임대수익에서 운영비용을 차감한 금액 |
| **SSoT (Single Source of Truth)** | 매물의 모든 데이터가 집중되는 단일 진실 원천 테이블 |
| **블라인드 딜카드** | 정확한 주소를 숨기고 권역/자산유형/수익률만 공개하는 매물 요약 카드 |
| **스튜디오** | 개별 매물에 대한 심층 관리 워크스페이스 (임대차, 증빙, 브리핑 등) |
| **모닝 인텔리전스** | AI가 매일 제공하는 시장 동향 브리핑 및 콜드콜 스크립트 |
| **Solapi** | 카카오 알림톡/SMS 발송에 사용되는 메시징 API 서비스 |
| **구독 카드** | Vibe Card, IM, 매거진 하단에 배치된 전화번호 입력형 구독 신청 UI |
