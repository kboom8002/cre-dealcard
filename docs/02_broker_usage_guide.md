## 3. 매물 등록 및 관리

> **경로**: `/broker/buildings`

### 3.1 딜카드 생성 (30초)

**최소 입력**: 주소만 입력하면 AI가 자동으로 딜카드를 생성합니다.

1. `/broker/deal-card/new` 접속
2. 주소 입력 → 좌표 자동 변환
3. 기본 정보 입력 (건물명, 자산유형, 면적, 호가)
4. AI가 블라인드 딜카드 + 시그널 카드 자동 생성

### 3.2 딜카드 상세 (`/broker/deal-card/[id]`)

딜카드 상세 페이지는 16개 서브 컴포넌트로 구성됩니다:

| 영역 | 컴포넌트 | 기능 |
|------|---------|------|
| 사진 | PhotoGallery | 수평 스크롤 갤러리 |
| 파이프라인 | DealCardPipelineContainer | 단계 상태 머신 |
| 시그널 | SignalSummary | 권역·자산·가격·공실 2×2 요약 |
| 편집기 | DealCardEditor | 제목, 요약, 투자 포인트, 주의 포인트 편집 |
| 매칭 매수자 | MatchedBuyersSection | 매칭된 매수 고객 목록 |
| 일정 | ScheduleSection | 임장 예약 현황 |
| 예측 | DealPredictionSection | AI 거래 성사 예측 |
| 이상적 매수자 | IdealBuyerPersonaSection | AI 생성 타겟 페르소나 |
| Gate 수신함 | GateRequestsInbox | Gate 요청 접수/심사 |

**하단 CTA 바 (4개 버튼)**:
1. 카카오 공유 → 딜카드 카카오톡 발송
2. 모바일 IM 생성 → 7섹션 IM 즉시 생성
3. 매수자 전체 보기 → 의향서 관리
4. 건물주 리포트 → 소유주용 보고서

### 3.3 매물 목록 탭

| 탭 | 기능 |
|----|------|
| 전체 매물 | 모든 딜카드 (promotion_score 내림차순) |
| 활성 딜 | 진행 중인 딜만 필터링 |
| IM 관리 | 생성된 모바일 IM 상태 관리 |
| 임대 | 임대 스페이스 관리 |

## 5. 모바일 투자설명서(IM)

> **생성**: `/broker/buildings/[id]/im-lite`
> **공개 URL**: `/im-lite/[buildingId]`

### 5.1 7개 섹션

| # | 섹션 | 아이콘 | 내용 |
|---|------|--------|------|
| ① | 물건 개요 | 🏢 | 건물 요약, 위치, 규모, 자산유형 |
| ② | 입지 접근성 | 📍 | 교통, 상권, 인구, 개발호재 |
| ③ | 임대 현황 | 👥 | Rent Roll, WALE, 공실률 |
| ④ | 수익 분석 | 💰 | NOI, 캡레이트, DCF 10년 NPV |
| ⑤ | 리스크 체크 | ⚠️ | 잠재 리스크 + 완화 방안 |
| ⑥ | 투자 포인트 | 🎯 | 핵심 투자 논거, 차별화 요인 |
| ⑦ | 다음 단계 | 🚀 | 거래 절차, 추천 일정 |

### 5.2 IM 생성 Ready 점수 (40점 이상 필요)

| 항목 | 점수 | 유형 |
|------|------|------|
| 주소 | 25 | 필수 |
| 월 임대료 | 20 | 필수 |
| 자산유형 | 10 | 기본 |
| 가격대 | 10 | 기본 |
| 권역 신호 | 10 | 기본 |
| 공실률 | 10 | 강화 |
| 사진 | 10 | 강화 |
| 하이라이트 | 5 | 강화 |

### 5.3 AI 생성 파이프라인

```
데이터 준비 → A/B 프롬프트 선택 → 골든 IM Few-shot → RAG 컨텍스트
→ 물류센터 오버레이(해당 시) → 7섹션 순차 생성
→ 용어 정규화 → 벤치마크 비교 → 품질 게이트 → IM Judge 스코어링
```

### 5.4 IM 상태 워크플로우

```
draft → generating → generated → pending_approval → published
                                                   → revision_needed → archived
```

### 5.5 공개 IM 기능

- **TTS 음성 브리핑**: `/api/public/im-lite/[id]/tts` — AI 음성으로 IM 낭독
- **PDF 내보내기**: `/api/public/im-lite/[id]/export`
- **영문 번역**: `/api/public/im-lite/[id]/translate`
- **체류시간 분석**: `beforeunload` 이벤트로 `sendBeacon` 자동 전송
- **60초+ 열람 시**: 브로커에게 카카오 알림톡 자동 발송

## 6. 딜 파이프라인

> **경로**: `/broker/pipeline`

### 6.1 8단계 칸반 보드

```
📝 메모 → 📋 딜카드 → 🔒 Gate → 📄 IM → 🤝 미팅 → ✍️ LOI → 📜 계약 → ✅ 완료
```

| 단계 | 체류 경고 | 전환 시 필수 필드 |
|------|----------|------------------|
| 메모 입력 | 1일 | building_ssot_lite_id |
| 딜카드 생성 | 7일 | gate_request_id |
| Gate 요청 | 7일 | im_project_id, readiness_score |
| IM 작성 | 14일 | buyer_intent_id, match_grade, meeting_schedule |
| 미팅 | 14일 | buyer_reaction, price_gap |
| LOI | 21일 | agreed_price, key_conditions |
| 계약 | 30일 | closing_date, fund_confirmed |
| 완료 | — | (최종) |

### 6.2 핵심 기능

- **드래그 앤 드롭**: 딜 카드를 단계 간 이동
- **체류 경고**: ⚠️ 뱃지로 정체된 딜 식별
- **전환 유효성 검증**: 필수 필드 미충족 시 전환 차단
- **전환 이력**: 모든 단계 변경이 `pipeline_stage_transitions`에 자동 기록

## 7. AI 매칭 엔진

> **경로**: `/broker/matching`

### 7.1 3단계 스코어링 파이프라인

| 단계 | 방법 | 기능 |
|------|------|------|
| Stage 1 | Boolean Hard Filter | 예산, 권역, 자산유형 룰 기반 필터링 |
| Stage 2 | Semantic Distance | OpenAI 임베딩 코사인 유사도 |
| Stage 3 | Ensemble | 목적별 가중 합산 → 등급 산출 |

### 7.2 등급 체계

| 등급 | 적합도 | 색상 | 조치 |
|------|--------|------|------|
| **S** | 90%+ | 🟡 Gold | 즉시 연결 추천 |
| **A** | 75~90% | 🟢 Green | 적극 제안 |
| **B** | 60~75% | 🔵 Blue | 조건 조율 시 가능 |
| **C** | 60% 미만 | ⚫ Slate | 참고용 |

### 7.3 뷰 모드

- **매물 중심(Deal-centric)**: 매물별로 매칭된 매수자 그룹핑
- **매수자 중심(Buyer-centric)**: 매수자별로 매칭된 매물 그룹핑

### 7.4 목적별 가중 프로필

| 목적 | 가중치 특징 |
|------|-----------|
| 사옥 | 입지·접근성 가중 |
| 투자 | 수익률·캡레이트 가중 |
| 증여 | 가격·지역 안정성 가중 |

```
2. 매거진 에디터 → 필드노트 5필드 (2분) + 커버 (30초)
3. "발행" 클릭
   └→ 전체 구독자 카카오 자동 배포
   └→ Vibe 명함에 최신 호 반영
```

### 시나리오 C: Hot Lead 대응

```
1. 🔥 카카오 알림 수신 "Hot Lead 스코어 95점"
2. 대시보드 → 퍼널 확인 (어떤 매물, 어떤 채널)
3. 해당 IM 카카오 재공유
4. 전화 연결 → 맞춤 상담
```

### 시나리오 D: 매수 고객 등록 → 자동 매칭

```
1. 매수 고객 미팅 후 의향서 등록
2. AI 3단계 매칭 자동 실행 → S/A/B/C 결과
3. S급 매칭 → IM 카카오 즉시 공유
4. 고객 IM 60초+ 열람 → 자동 알림 수신
```

## 22. 용어 사전

| 용어 | 설명 |
|------|------|
| **IM** | Investment Memorandum. AI 7섹션 투자설명서 |
| **Vibe Card** | AI 성격 분석 기반 디지털 명함 |
| **VTI** | Vibe Type Indicator. AI가 산출한 브로커 성격 유형 |
| **SSoT** | Single Source of Truth. 매물 데이터 단일 진실 원천 |
| **딜카드** | 매물 핵심 정보 카드 (블라인드 공유 가능) |
| **Gate** | 블라인드 딜카드의 상세 주소 열람 요청/승인 절차 |
| **Hot Lead** | 교차 채널 스코어 80점+ 고관심 잠재 고객 |
| **리드 스코어** | 3채널 방문자 행동 가중 합산 관심도 점수 |
| **필드노트** | 브로커 현장 감각 5필드 구조화 입력 |
| **파이프라인** | 8단계 딜 진행 추적 (메모→완료) |
| **WALE** | Weighted Average Lease Expiry. 가중평균잔여임대기간 |
| **캡레이트** | NOI ÷ 매입가 × 100. 수익형 부동산 수익률 |
| **NOI** | Net Operating Income. 순영업소득 |
| **Few-shot** | 골든 IM 예시 기반 AI 학습 방식 |
| **RAG** | Retrieval-Augmented Generation. 검색 보강 생성 |
| **RLS** | Row-Level Security. 행 수준 데이터 접근 제어 |
| **ISR** | Incremental Static Regeneration. 증분 정적 재생성 |
| **Solapi** | 카카오 알림톡/SMS 발송 API 서비스 |
| **콜드콜 스크립트** | 모닝 인텔리전스가 생성한 고객 접촉용 대화 스크립트 |
| **모닝 인텔리전스** | AI 매일 아침 시장 동향 브리핑 |
| **품질 게이트** | IM/매거진 생성 후 수치 정합성 교차 검증 |
| **A/B 테스트** | 프롬프트 레지스트리를 통한 AI 프롬프트 성과 비교 |
