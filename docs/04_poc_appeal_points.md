# PoC 소구 포인트 — 제이에스부동산중개(주) 20인 중개인

> **대상**: 서울 꼬마빌딩(10~100억) 매매 전문 법인 소속 중개인 20인  
> **핵심 모델**: 사내 중개인 간 매물↔매수자 크로스 매칭 + CRM/리드 확보 자동화

---

## 1. 중개인 현업 Pain Point → 시스템 해결 매핑

| # | Pain Point | 현재 방식 | DealCard Hub 해결 |
|---|-----------|----------|------------------|
| ① | **매물 정보 정리에 시간 낭비** | 카톡 메모 → 엑셀 수작업 (건당 30~45분) | 카톡 복붙 → 60초 AI 딜카드 (97% 단축) |
| ② | **같은 법인 내 매물·매수자 공유 비효율** | 단톡방에 텍스트 뿌리기 → 누가 관심인지 모름 | 블라인드 딜카드 자동 공유 → AI가 S/A/B/C 매칭 |
| ③ | **민감 정보 유출 리스크** | 주소·임대인명이 카톡으로 무분별 유통 | 7개 민감필드 자동 블라인드 + 3단계 게이트 |
| ④ | **매수자-매물 매칭이 기억에 의존** | "아, 김사장님이 이런 빌딩 찾고 있었는데..." | AI 3단계 매칭 엔진 → 등록 즉시 전사 매칭 |
| ⑤ | **고객(리드) 관리 부재** | 메모장/수첩, 카톡 대화방에 흩어진 고객 정보 | CRM 자동 등록 + 의향서 구조화 + 연락 이력 |

## 2. 개인 중개인에게 소구하는 6가지 가치

### 가치 ①: "카톡 한 번이면 딜카드 완성"

```
[Before] 매물 정보 받음 → 엑셀 정리 → 블라인드 문구 작성 → 단톡방 공유 (45분)
[After]  매물 정보 받음 → 카톡 복붙 → AI 딜카드 (60초) → 원클릭 공유
```

- **터치 포인트**: `/broker/deal-card/new`, `/broker/lease-card/new`
- **데모 시나리오**: 실제 카톡 메모 1건을 현장에서 즉시 변환 시연
- **KPI**: 건당 소요 시간 97% 단축

### 가치 ②: "내 매물에 맞는 매수자가 사내에 있다"

```
[Before] "혹시 성수동 30억대 빌딩 찾는 분 있으세요?" → 단톡방 질문 → 3명 응답 → 수동 비교
[After]  딜카드 등록 → AI가 사내 20인의 매수 의향서 전수 대조 → S등급 3건 즉시 알림
```

- **터치 포인트**: `/broker/matching`
- **핵심 소구**: "등록만 하면 20명 동료의 고객 DB와 자동 매칭됩니다"
- **KPI**: S등급 매칭 전환율, 매칭 → 미팅 전환율

### 가치 ③: "정보 유출 걱정 없이 매물 공유"

```
[Before] 정확한 주소가 카톡으로 돌면 → 타사 중개인에게 유출 → 매도인 신뢰 상실
[After]  블라인드 티저 (권역·면적·가격대만) → 관심 매수자만 게이트 통과 시 상세 공개
```

- **블라인드 자동 처리 항목**: 정확한 주소, 호실, 임대인명, 현 임차인명, 호별 임대료, 매도인 동기, 협상 메모
- **핵심 소구**: "오너가 가장 싫어하는 주소 유출, 시스템이 원천 차단합니다"

### 가치 ④: "매수 의향서도 카톡으로 끝"

```
[Before] "김사장님 GBD 50억 이하 오피스 찾으신대" → 메모장에 적어둠 → 잊어버림
[After]  카톡 내용 복붙 → AI가 구조화된 의향서 생성 → 기존 매물과 즉시 매칭
```

- **터치 포인트**: `/broker/buyer-intents/new`, `/broker/tenant-intents/new`
- **핵심 소구**: "고객이 원하는 조건을 시스템이 기억합니다. 매물이 나오면 자동 알림."

### 가치 ⑤: "고객이 알아서 CRM에 들어온다"

```
[Before] 전화 문의 → 수기 메모 → 나중에 정리해야지 → 까먹음
[After]  리싱 페이지 문의 → AI가 적합성 판정 → CRM 자동 등록 + 카톡 답변 초안 생성
```

- **터치 포인트**: `/leasing/[slug]` → 문의 → `/broker/clients`
- **핵심 소구**: "리드가 자동으로 쌓이고, AI가 답변 초안까지 만들어줍니다"

### 가치 ⑥: "내 딜이 어디까지 진행됐는지 한눈에"

```
[Before] "이 건 지금 미팅 했나? LOI 보냈나?" → 카톡 대화 뒤져서 확인
[After]  /broker/pipeline에서 전체 딜 파이프라인 시각화 → 체류일수 경고
```

- **터치 포인트**: `/broker/pipeline`
- **핵심 소구**: "14일 넘게 멈춰있는 딜을 자동으로 경고합니다"

## 3. 법인(조직)에게 소구하는 4가지 가치

### 조직 가치 ①: 사내 거래 활성화

| 지표 | 현재 (추정) | PoC 목표 |
|------|-----------|---------|
| 사내 매칭 건수/월 | 2~3건 (구두) | 10건+ (AI 매칭) |
| 매칭 → 미팅 전환율 | 불명 | 30%+ |
| 딜 체류일수 | 불명 | < 20일 |

> **핵심 메시지**: "20명이 각자 가진 매물·매수자를 AI가 자동 연결합니다. 사내 거래 건수가 3배 이상 늘어날 수 있습니다."

### 조직 가치 ②: 데이터 자산 축적

```
현재: 중개인 퇴사 시 고객·매물 정보 유실
미래: 법인 DB에 매물 SSoT + 고객 CRM + 매칭 이력 누적
```

### 조직 가치 ③: 팀장/대표 파이프라인 가시성

- `/broker/pipeline`: 전체 딜 진행 현황 실시간 확인
- `/admin/analytics`: 중개인별 딜카드 생성 수, 매칭 수, 전환율

### 조직 가치 ④: 시장 인텔리전스 공유

- `/pulse`: 권역별 주간 시장 온도 → 전사 공유
- `/market/[region]`: 데이터 기반 고객 상담 근거

## 4. 온보딩 3단계 로드맵

### Phase 1: 즉시 가치 (Week 1~2)

| 기능 | 행동 | 목표 |
|------|------|------|
| 딜카드 생성 | 각자 보유 매물 5건씩 등록 | 총 100건 딜카드 풀 |
| 매수 의향서 | 각자 보유 매수자 3건씩 등록 | 총 60건 의향서 풀 |
| **자동 매칭** | 100매물 × 60매수 = 6,000쌍 AI 매칭 | S등급 매칭 30건+ 도출 |

### Phase 2: 확장 가치 (Week 3~4)

| 기능 | 행동 | 목표 |
|------|------|------|
| CRM 활용 | 기존 고객 정보 이관 | 고객 200명+ 등록 |
| 임대 딜카드 | 임대 매물 등록 + 마켓플레이스 공개 | 임대 30건+ |
| 리싱 페이지 | AI 리싱 부스트 → 공개 URL | 리싱 10페이지+ |

### Phase 3: 습관 형성 (Week 5~8)

| 기능 | 행동 | 목표 |
|------|------|------|
| 파이프라인 | 모든 딜 파이프라인 추적 | 주간 리뷰 미팅에 활용 |
| Pulse 활용 | 시장 데이터 기반 고객 상담 | 주 1회 Pulse 리포트 공유 |
| 리드 자동화 | 리싱 페이지 → 문의 유입 | 월 20건+ 신규 리드 |

## 5. 예상 저항 포인트 & 대응

| 저항 | 대응 |
|------|------|
| "내 매물/고객 정보를 왜 공유해야 하나?" | 블라인드 시스템으로 민감 정보 보호. 공유한 만큼 매칭 받음. 공유하지 않으면 매칭 대상에서 제외. |
| "또 새로운 툴 배워야 하나?" | 카톡 복붙만 하면 됨. 별도 앱 설치 없이 모바일 웹. 60초 데모로 즉시 체감. |
| "기존 엑셀이 더 편한데?" | 엑셀에는 자동 매칭이 없음. 등록만 하면 20명 동료의 고객과 자동 대조. |
| "AI가 정보를 잘못 파싱하면?" | 가드레일 시스템 내장. 모든 AI 출력에 boundary_note(주의사항) 포함. 중개인이 최종 확인 후 공개. |
| "우리 정보가 외부로 유출되면?" | RLS(Row Level Security) 적용. 법인 단위 데이터 격리. 외부 접근 불가. |
