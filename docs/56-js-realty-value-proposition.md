# CRE Deal System: JS부동산중개 가치 제안서

> **핵심 한 문장:** "300인의 꼬마빌딩 중개 경험을 한 명이 퇴사해도 사라지지 않는 법인 자산(SSoT)으로 변환하고, AI가 매칭·IM·마케팅을 자동화하여 1인 중개인의 생산성을 3배로 끌어올립니다."

---

## 1. JS부동산중개의 5대 페인포인트 진단

| # | 페인포인트 | 현실 | 비용 |
|---|-----------|------|------|
| P1 | **이탈 리스크** | 에이스 중개인 퇴직 시 진성 매수자 리스트·딜 히스토리 소멸 | 리드 1명 재발굴 비용 50만~수백만원 |
| P2 | **정보 파편화** | 카카오톡·수첩·머릿속에 흩어진 매물 정보 | 중개인 1인당 추적 가능 딜 최대 20~30건 한계 |
| P3 | **매칭 비효율** | "이 매물 살 것 같은 매수자가 누구더라"를 수작업으로 기억에 의존 | 적기(타이밍) 매칭 실패 → 계약 손실 |
| P4 | **IM 제작 비용** | 50억+ 매물 IM 제작 외주 300~500만원·2주 소요 | 연간 법인 기준 수억 원 낭비 |
| P5 | **공실 영업 지연** | 공실 발생 후 텍스트 광고까지 최소 1~2주 | 공실 1개월 = 임대인 손실 월 임대료 100% |

## 3. 시스템별 상세 가치 제안

### 3-1. DealCard: "개인의 기억 → 법인의 자산"

**구현된 실제 기능:**
- `broker-deal-card.ts`: 중개인이 카카오 메모 수준으로 입력한 텍스트를 AI가 `building_ssot_lite`로 정형화 (위치·면적·가격·권리관계·강점·유의사항 자동 추출)
- `buyer-intent.ts`: 매수자 상담 메모를 AI가 `buyer_intent_lite`로 정형화 (예산범위·선호지역·매수목적·반드시 필요한 조건 자동 정리)
- `matching-engine.ts`: 3-Stage 매칭 (예산/지역 Hard Filter → Semantic 유사도 → 사옥/투자/증여 목적별 앙상블)
- `promotion-ranker.ts`: 매물별 노출 우선순위 자동 계산 (IoT 데이터·희소성·고객 관심도 가중치)

**JS부동산중개 적용 시나리오:**
```
중개인 박 팀장이 상담 후 카카오에 이런 메모를 남깁니다:
"성수동 꼬마빌딩 80억대, 1층 카페, 2~4층 사무실, 
 건물 상태 보통, 엘베 없음, 매도인 세금 문제로 급매 기미"

→ DealCard가 30초 안에:
  · 위치 시그널: 성수동 (교통/상권 자동 분석)
  · 가격 밴드: 75~85억
  · 자산 유형: 근생+오피스 복합
  · 리스크: 엘리베이터 없음, 세금 이슈 (공개 주의)
  · Blind Teaser 초안 자동 생성

→ 즉시 법인 SSoT DB에 저장 → 박 팀장이 퇴사해도 영구 보존
→ 이 조건에 맞는 매수자 자동 알림 발송
```

**P1 해결:** 중개인 이탈 시에도 딜 데이터 100% 보존
**P2 해결:** 1인 중개인이 관리 가능한 딜 수 3~5배 증가
**P3 해결:** 적기 매칭 자동화 → 계약 타이밍 손실 최소화

### 3-2. Full IM Studio: "2주 300만원 → 2시간 무료"

**구현된 실제 기능:**
- `readiness-service.ts`: 18섹션(등기·건축물대장·수익분석·현황사진·입지 등) 기반 Readiness Score 0~100점 자동 계산
- `gate-review-service.ts`: 8중 Gate (확정수익 표현 차단, 개인정보 마스킹, 전문가 범위 이탈 방지 등)
- `section-planner.ts`: 매물 유형별 최적 섹션 구성 자동 선택
- `expert-patch.ts`: 세무사·법무사·감정평가사 영역 자동 경고 및 전문가 검토 요청
- Mobile IM (50억 미만 매물용 7섹션 경량 버전)

**JS부동산중개 적용 시나리오:**
```
80억 성수 빌딩 → IM 제작이 필요한 상황

기존 방식:
  외주 디자인 업체 → 300~500만원 + 2주 소요
  결과물 나오면 이미 매물 식음

CRE Full IM Studio:
  Step 1. DealCard SSoT 기반 자동 섹션 생성 (입지/현황/수익분석/권리관계 등)
  Step 2. Readiness Score: 73점
         → "수익률 분석 섹션의 관리비 데이터가 누락되어 있습니다.
            입력하면 점수가 89점으로 상승합니다" 안내
  Step 3. 8중 Gate 자동 검토
         → "2층 임대수익률 X% 확정" → 차단 (확정 수익 표현 금지)
         → "건물 상태 최상급" → 차단 (증거 없는 최상급 표현)
  Step 4. 법인 승인 후 게이트 통과 → 전문 IM 완성

비용: 0원 | 시간: 2~3시간
```

**P4 해결:** IM 제작 비용 연간 수억 원 절감, 속도 10배 향상

### 3-3. Space AiPage: "공실 발생 즉시 → AI 마케팅 자동 가동"

**구현된 실제 기능:**
- `space-structuring-agent.ts`: 메모로 공간 SSoT 생성 (면적·용도·임대 조건·특장점 자동 구조화)
- `visual-classification-agent.ts` + `visual-album-agent.ts`: 현장 사진 업로드 → AI가 외관/내부/시설/주변으로 자동 분류 및 앨범 생성
- `tenant-fit-agent.ts`: 공간 특성 분석 → 적합 임차인 업종 자동 추천 (카페, 스튜디오, 클리닉 등)
- `vibe-fit-agent.ts`: 임차 희망자와 공간의 분위기 궁합 점수
- `leasing-page-writer-agent.ts`: 임대 랜딩 페이지 자동 생성 (URL 공유 가능)
- `campaign-copy-agent.ts`: SNS·카카오 홍보용 카피 자동 생성
- `owner-report-agent.ts`: 건물주 대상 공실 현황·임대 진행 리포트 자동 생성 (레트로핏 CTA 포함)
- `inquiry-qualifier-agent.ts`: 임차 문의 자동 등급 판정 (A급 리드 즉시 알림)

**JS부동산중개 적용 시나리오:**
```
성수 빌딩 1층 카페 공실 발생 (월세 350만원)

기존 방식:
  사진 촬영 → 네이버 매물 등록 텍스트 작성 → 1~2주 소요
  공실 기간 = 건물주 수익 손실

Space AiPage:
  Step 1. 현장 사진 10장 업로드
         → AI가 자동 분류: 외관 3장/내부 5장/주변 2장 + 비주얼 앨범 생성
  Step 2. 공간 SSoT 생성 (면적·채광·층고·전기용량 등)
         → 적합 임차인 업종: "카페, 플로리스트, 네일샵, 편집샵" 자동 추천
  Step 3. AI 임대 랜딩 페이지 자동 생성 (15분)
         → URL: star-aipage.com/spaces/seongsu-alpha-1f
         → 카카오 공유 링크 + SNS 카피 자동 생성
  Step 4. 임차 문의 접수 → A/B/C급 자동 분류
         → A급 문의(카페, 예산 충족) → 중개인에게 즉시 알림
  Step 5. 건물주에게 주간 리포트 자동 발송
         → "이번 주 랜딩 페이지 조회 47회, 문의 3건, A급 1건"

공실 마케팅 시작까지: 15분 (기존 1~2주 → 97% 단축)
```

**P5 해결:** 공실 마케팅 즉시 가동 → 공실 기간 단축 → 건물주 신뢰 확보

## 5. 단계별 도입 로드맵 (JS부동산중개 맞춤)

| 단계 | 기간 | 도입 기능 | 기대 성과 |
|------|------|---------|---------|
| **Phase 1: 파일럿** | 1개월 | DealCard (매물 SSoT) 선발 10명 테스트 | 매물 등록 속도 10배·이탈 리스크 검증 |
| **Phase 2: 확장** | 2개월 | 전체 300인 DealCard + 매수자 매칭 | 법인 SSoT DB 구축 완성 |
| **Phase 3: IM 전환** | 1개월 | Full IM Studio 도입 | IM 외주비 전액 절감 시작 |
| **Phase 4: 공실 자동화** | 1개월 | Space AiPage + 건물주 리포트 자동화 | 건물주 신뢰 기반 추가 매물 확보 |

## 6. 비즈니스 협력 모델 제안 (공동 사업 구조)

| 모델 | 구조 | JS부동산 역할 | 우리 역할 |
|------|------|-------------|---------|
| **A. SaaS 구독** | 월 구독료 (중개인 1인당 3~5만원) | 시스템 사용료 납부 | 운영·개발·유지보수 |
| **B. 성과 공유** | 계약 완료 건당 커미션 일부 공유 | 딜 클로징 | 매칭·IM·마케팅 지원 |
| **C. 공동 브랜드** | "JS-AI 부동산" 브랜드로 런칭 | 고객망·매물망 제공 | 시스템·AI 엔진 제공 |
| **D. 레트로핏 총판** | 건물주에게 IoT·설비 솔루션 판매 | 건물주 접점 제공 | 리드 생성·기술 제공 |

> **추천 협력 모델:** C + D 조합 — JS부동산의 300인 고객망과 우리 AI 시스템을 공동 브랜드로 묶고, 레트로핏 총판에서 하드웨어 수익을 캐시카우로 확보하는 구조.
