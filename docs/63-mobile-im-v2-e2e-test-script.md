## TC-01. 딜카드 → 바텀시트 → IM 생성 (핵심 플로우)

### 목적
딜카드에서 추가 정보 입력 후 모바일 IM이 정상 생성되는지 검증

### 절차

1. **딜카드 접속**
   - `/broker/deal-card/{buildingId}` 이동
   - "📝 프리미엄 투자설명서 만들기" 버튼 확인 및 클릭
   - ✅ 바텀시트가 열리는지 확인

2. **주소 검색 및 선택**
   - 주소 검색창에 "역삼동 823" 입력
   - ✅ 주소 후보 리스트가 드롭다운으로 표시되는지 확인
   - 도로명 주소 선택
   - ✅ PNU 자동 해석 및 표시 확인

3. **재무 데이터 입력**
   - 월 임대료: 1800 (만원)
   - 보증금: 30000 (만원)
   - 관리비: 250 (만원)
   - 대출 잔액: 2000000 (만원)
   - 매각 희망가: 5000000 (만원)
   - 공실률: 5%
   - ✅ 입력 시 Readiness Score 동적 증가 확인

4. **등급 승급 안내 확인**
   - 매각가 미입력 상태: "💡 매각 희망가를 입력하면 A등급(투자 검토 가능)으로 승급됩니다" 안내 표시
   - 매각가 입력 후 대출 미입력: "💡 대출 잔액을 입력하면 레버리지 분석이 추가됩니다" 안내 표시
   - ✅ 안내 메시지 정상 표시 확인

5. **사진 업로드**
   - 3장 이상 사진 업로드
   - ✅ 사진 프리뷰 썸네일 표시 확인
   - ✅ 캡션 입력 가능 확인

6. **IM 생성 실행**
   - "📄 프리미엄 투자설명서 만들기" 버튼 클릭
   - ✅ 프로그래스 표시 (섹션별 진행 상태)
   - ✅ 15~45초 내 완료
   - ✅ 승인 페이지로 자동 리다이렉트

### 예상 결과
- 승인 페이지(`/broker/im-approval/{docId}`)에서 7섹션 IM 표시
- 생성 시간 45초 이내

## TC-02. IM 승인 페이지 검증

### 목적
생성된 IM을 중개인이 검토/승인하는 플로우 검증

### 절차

1. **승인 페이지 접속**
   - TC-01에서 자동 이동된 `/broker/im-approval/{docId}` 확인

2. **사진 프리뷰 확인** *(신규)*
   - ✅ 상단에 "📷 사진 (N장)" 헤더 표시
   - ✅ 가로 스크롤 썸네일 갤러리 (96×64px)
   - ✅ 업로드한 사진 모두 표시

3. **7섹션 콘텐츠 검토**
   - ✅ 🏢 자산 개요 및 제원
   - ✅ 📍 입지 및 대중교통 분석
   - ✅ 📊 임대차 현황 및 공실 상태
   - ✅ 💸 수익률 및 공시지가 분석
   - ✅ ⚖️ 공법 규제 및 리스크 진단
   - ✅ 🎯 핵심 투자 메리트
   - ✅ 📅 향후 검토 및 진행 절차
   - ✅ 각 섹션 마크다운 정상 렌더링

4. **승인 실행**
   - "승인 및 공개" 버튼 클릭
   - ✅ "IM이 승인되어 공개되었습니다" 메시지 확인
   - ✅ 공개 뷰어 URL 확인 가능

### 예상 결과
- 승인 후 document_objects.status = `broker_reviewed`
- published_at 타임스탬프 설정됨

## TC-03. 공개 뷰어 — 핵심 UI 컴포넌트 검증

### 목적
승인된 IM의 모바일 뷰어에서 모든 신규 기능이 정상 렌더링되는지 검증

### 절차

1. **공개 뷰어 접속**
   - `/im-lite/{buildingId}?doc={docId}` 이동 (모바일 기기 권장)

2. **[G5] 품질 등급 배지 확인** *(개선)*
   - ✅ "🟢 A등급 — 투자 검토 가능 (XX점)" 배지 표시
   - ✅ 매각가 + 임대료 + 주소 + 공공데이터 입력 시 A등급
   - SSoT 완성도 프로그래스 바 (N/7 섹션 공개)

3. **[G2] 승인일자 표시** *(신규)*
   - ✅ approvedAt 날짜 표시 (2026. 7. 12.)
   - 기존: 항상 undefined → 수정 후: 실제 승인 시각

4. **Hero Card 확인** *(기존 정상)*
   - ✅ 매각 희망가 표시 (50억)
   - ✅ Cap Rate 표시 (%)
   - ✅ NOI 표시 (억)
   - ✅ 핵심 투자 포인트 1줄
   - ✅ 핵심 리스크 1줄
   - ✅ DCF 10년 NPV 표시

5. **사진 갤러리 확인** *(기존 정상)*
   - ✅ 업로드한 사진 모두 표시 (캡션 포함)
   - ✅ 라이트박스 스와이프 작동
   - ✅ 지도 표시 (주소 기반 좌표)

6. **[G3] 출처 배지 확인** *(신규)*
   - ✅ 각 섹션 헤더 하단에 출처 배지 표시:
     - `✓ 공부 확인` (녹색) — 공공데이터 기반
     - `👤 중개인 입력` (황색) — 중개인이 입력한 데이터
     - `⚙ AI 추정` (보라색) — AI가 계산/추론한 데이터
   - ✅ 섹션별 출처 조합이 다르게 표시

7. **[G1] DCF 히트맵 확인** *(신규 — 핵심!)*
   - ✅ 💸 수익률 섹션 하단에 DCF 3×3 매트릭스 표시
   - ✅ 할인율(Cap Rate) × 매각 배수 조합별 NPV 값
   - ✅ WACC 기반 색상 히트맵 (녹색-노랑-빨강)

8. **[G1] 레버리지 차트 확인** *(신규 — 핵심!)*
   - ✅ DCF 히트맵 하단에 자금 구조 차트 표시
   - ✅ 자기자본 / 보증금 / 대출 비율 시각화
   - ✅ 레버리지 수익률 표시

9. **Double-Track CTA 확인** *(기존 정상)*
   - ✅ "📄 프라이빗 투자설명서(IM) 신청" 버튼
   - ✅ "📞 {중개인명} 즉시 상담" 전화 버튼

10. **Voice Briefing 확인** *(기존 정상)*
    - ✅ 음성 브리핑 재생 버튼 표시
    - ✅ 재생/일시정지/프로그래스 바 작동

### 예상 결과
- 모든 컴포넌트가 모바일 화면에서 레이아웃 깨짐 없이 정상 표시
- DCF 히트맵과 레버리지 차트가 **처음으로** 실제 데이터로 렌더링

## TC-04. 카카오톡 / OG 공유 테스트

### 목적
카카오톡 공유 시 전문적인 카피와 OG 이미지가 정상 표시되는지 검증

### 절차

1. **카카오 공유 버튼 클릭**
   - 공개 뷰어 하단의 카카오 공유 버튼 클릭

2. **[G4] 카카오 카드 카피 확인** *(개선)*
   - ✅ **title**: `[강남 GBD] *** 오피스빌딩` (areaSignal + blindName)
   - ✅ **description**: `매각 희망가 50억대 · Cap Rate 4.2% · 크리딜 프리미엄 투자설명서`
   - ✅ **button**: `투자설명서 즉시 열람`
   - ❌ 이전: "투자 매물" / "AI 자동 생성 모바일 투자설명서" / "투자설명서 보기"

3. **[G6] OG 이미지 확인** *(개선)*
   - `/api/og/deal/{buildingId}` 접속
   - ✅ 1200×630 이미지 생성
   - ✅ 권역 배지, 가격대, Cap Rate, NOI 포함
   - ✅ 기술 용어("IM Lite", "모바일") 노출 없음

4. **링크 복사 공유 테스트**
   - 카카오 미지원 환경에서 공유 버튼 클릭
   - ✅ Web Share API 또는 클립보드 복사 정상 작동

### 예상 결과
- 카카오톡에서 전문적인 부동산 투자 카드 표시
- 수신자가 "투자설명서 즉시 열람" 클릭 시 뷰어 정상 접속

   - `/magazine/{brokerId}/{date}` 이동

2. **[G8] 카카오 공유 확인** *(신규)*
   - 하단바 공유 버튼 클릭

## TC-06. 골든셋/퓨샷 시스템 검증

### 목적
승인된 IM이 골든셋에 자동 등록되고, 다음 IM 생성에 퓨샷으로 활용되는지 검증

### 절차

1. **골든셋 자동 등록 확인**
   - TC-02에서 승인 후 `/admin/golden-admin` 접속
   - ✅ 방금 승인한 IM의 섹션들이 `im_golden_sets` 테이블에 등록
   - ✅ 각 섹션의 `judge_score`가 3.5 이상인 것만 등록

2. **[G10] 자동 승격 확인** *(신규)*
   - AI Judge 점수 4.5 이상 섹션이 있었다면:
   - ✅ `im_golden_sets`에 해당 섹션이 `promoteToGoldenCandidate`로 자동 등록

3. **퓨샷 활용 확인**
   - 동일 자산 유형+가격대의 새 딜카드에서 IM 생성
   - ✅ 서버 로그에서 `[approve] Golden Set: N sections registered` 확인
   - ✅ 생성된 IM 섹션 스타일이 골든셋과 유사

### 예상 결과
- 승인 → 골든셋 등록 → 퓨샷 활용의 선순환 루프 정상 작동

## TC-07. 외부 API 폴백 처리 검증

### 목적
공공데이터 API 실패 시 하드코딩된 허구 데이터 대신 정상 처리되는지 검증

### 절차

1. **[G7] API 실패 시나리오 테스트**
   - 존재하지 않는 주소(예: "제주시 한경면 xxx")로 IM 생성 시도
   - ✅ 건축물대장 API 실패 → null 반환 (이전: "강남 센트럴타워" 허구 데이터)
   - ✅ 공시지가 API 실패 → null 반환 (이전: 2500만원/㎡ 하드코딩)
   - ✅ 용도지역 API 실패 → null 반환 (이전: "일반상업지역" 하드코딩)

2. **IM 콘텐츠 확인**
   - ✅ API 실패 시 해당 데이터 미포함 (또는 "확인 필요" 표기)
   - ✅ 허구 건물명이 IM에 표시되지 않음
   - ✅ 나머지 섹션은 정상 생성

### 예상 결과
- API 실패 시 에러 없이 정상 생성되되, 허구 데이터 미포함

## 알려진 제한 사항

1. **골든셋 초기 데이터**: 첫 번째 IM 생성 시 골든셋이 비어 있으면 하드코딩된 예시가 사용됨 (정상 동작)
2. **Kakao SDK 도메인 제한**: `credeal.net` 외 도메인(vercel preview 등)에서는 카카오 공유가 실패할 수 있음 → 프로덕션 도메인으로 강제 변환 로직 구현됨
3. **Voice Briefing**: TTS API 할당량 초과 시 음성 브리핑이 생성되지 않을 수 있음
4. **외부 API 지연**: 공공데이터 API 응답이 느릴 경우 IM 생성 시간이 45초를 초과할 수 있음
