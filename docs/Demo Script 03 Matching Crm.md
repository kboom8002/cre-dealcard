## ACT 1. 매수자 의향서 AI 등록 — 조건 메모 한 줄 (3분)

### 📍 화면: `https://www.credeal.net/broker/buyer-intents/new`

> [!IMPORTANT]
> 이 화면에서 입력한 매수 조건이 이후 **AI 매칭 엔진의 핵심 연료**가 됩니다.

#### Step 1: 매수자 조건 메모 입력

아래 예시를 `메모 입력란`에 붙여넣으세요:

```
강남역 도보 10분 이내 오피스빌딩. 연면적 2,000평 이상.
예산은 200억~300억. 자기자본 40% 이상 가능.
IT 기업 사옥으로 쓸 예정이라 주차 넉넉하고 층고 높은 게 필수.
리모델링도 OK인데 입주까지 18개월 이내.
```

#### Step 2: AI 구조화 결과 확인

**"AI 분석"** 버튼을 누르면 AI가 자동으로:

| 분석 항목 | AI 추출 결과 |
|-----------|-------------|
| 권역 | 강남 GBD |
| 가격대 | 200~300억 |
| 면적 | 연면적 2,000평+ |
| 자산유형 | 오피스빌딩 |
| 용도 | IT 기업 사옥 |
| 필수조건 | 주차 충분, 층고 높음 |
| 리모델링 | 허용 (18개월 내) |
| 자기자본 비율 | 40%+ |

#### 🎤 멘트 예시
> "고객이 카톡으로 보내온 메모를 **그대로 붙여넣기만** 하면, AI가 매수 조건을 자동 분석합니다. 한 글자도 직접 정리할 필요가 없습니다. 이 데이터가 쌓이면 쌓일수록 매칭 정밀도가 올라갑니다."

#### 📊 기대 가치
| 기존 방식 | DealCard 방식 |
|-----------|---------------|
| 엑셀에 수기 정리 → 30분/건 | AI 자동 파싱 → **30초/건** |
| 조건 누락·오기 빈번 | 구조화된 8개 축으로 완전 정리 |
| 과거 고객 조건 망각 | DB 영구 보관 → **AI 매칭 자동 연결** |

#### ➡️ 다음 액션

## ACT 3. ⭐ AI 매칭 센터 — 듀얼뷰 + 3단계 정밀분석 (4분)

### 📍 화면: `https://www.credeal.net/broker/matching`

> [!CAUTION]
> **이 화면이 데모의 핵심입니다.** 매물과 매수자가 자동으로 연결되는 순간을 보여주세요.

#### 🆕 Step 1: 듀얼뷰 토글 시연

화면 상단에 **Pill 스타일 탭 바**가 있습니다:

| 뷰 모드 | 설명 | 사용 시나리오 |
|---------|------|--------------|
| **📦 내 매물 기준** | 내가 올린 딜카드별로 매칭된 매수자 그룹핑 | "이 매물에 관심 가질 매수자가 누구지?" |
| **👤 내 고객 기준** | 내 매수자별로 매칭된 매물 그룹핑 | "이 고객에게 보여줄 매물이 뭐지?" |

> [!TIP]
> **시연 순서**: 먼저 "내 매물 기준"으로 S등급 매칭을 보여주고, 탭을 전환하여 "내 고객 기준"으로 같은 매칭이 고객 관점에서 어떻게 보이는지 보여주세요.

#### Step 2: 매칭 등급 시스템 확인

**매칭 카드**마다 색상 구분된 등급 배지가 표시됩니다:

| 등급 | 색상 | 적합도 | 권장 액션 |
|------|------|--------|----------|
| **S등급** | 🟢 에메랄드 | 95%+ | 🔥 즉시 연락 — "매수자에게 이 딜카드 전송하기" |
| **A등급** | 🔵 블루 | 80%+ | 적극 검토 권유 |
| **B등급** | 🟡 앰버 | 60%+ | 참고용 |
| **C등급** | 🔴 로즈 | 60% 미만 | 조건 변경 시 재매칭 |

#### 🆕 Step 3: AI 3단계 정밀분석 열기

매칭 카드의 **"AI 3단계 정밀 분석 열기"** 패널을 펼치면:

```
┌─────────────────────────────────────────────┐
│  🔍 Stage 1: Hard Filter (통과/탈락)         │
│  ✅ 권역 일치    ✅ 예산 범위    ✅ 자산유형  │
│                                             │
│  📊 Stage 2: Semantic Similarity            │
│  ████████████████████░░░░  82%              │
│  "AI가 매물 설명과 매수 조건의 문맥 유사도를   │
│   분석했습니다"                               │
│                                             │
│  🎯 Stage 3: Ensemble Weight Profile        │
│  최종 종합 점수: A등급 (84.2%)               │
└─────────────────────────────────────────────┘
```

#### 🆕 Step 4: 공동중개 업무확약서 (S/A등급 전용)

S등급 또는 A등급 매칭에서는 **"공동중개 업무확약서"** 섹션이 자동 표시됩니다:
- AI가 매물 정보와 매수자 조건을 기반으로 **확약서 초안을 자동 생성**
- **"📋 복사하기"** 버튼으로 클립보드에 즉시 복사
- 매수측 중개인과의 공동중개 합의에 바로 활용 가능

#### 🆕 Step 5: 피드백 루프

각 매칭 카드 하단에 **👍 적합 / 👎 부적합** 피드백 버튼:
- 피드백 데이터가 쌓이면 매칭 알고리즘이 **개인화 학습**
- "이 중개인은 리모델링 가능 물건을 선호한다" 등 패턴 반영

#### 🎤 멘트 예시
> "매칭 센터가 크게 업그레이드됐습니다. **'내 매물 기준'과 '내 고객 기준' 두 가지 관점**으로 동시에 매칭을 확인할 수 있습니다. 여기서 S등급이 떴네요 — **AI 3단계 분석 패널**을 열어보면, 단순히 '점수가 높다'가 아니라 Hard Filter를 통과했는지, 의미론적으로 얼마나 유사한지까지 투명하게 보여줍니다. 그리고 S등급에는 **공동중개 확약서까지 자동 생성**됩니다!"

#### 📊 기대 가치
| 지표 | 기존 수동 방식 | AI 매칭 활용 후 |
|------|---------------|-----------------|
| 매칭 소요 시간 | 건당 30분~1시간 | **건당 3초** |
| 매칭 누락률 | 30~50% (기억 의존) | **0%** (전수 검사) |
| 분석 투명성 | "감이 좋아서" | **3단계 근거 공개** |
| 공동중개 합의 | 수기 작성 30분 | **AI 자동 초안 5초** |
| 월간 성사 딜 | 1~2건 | **3~5건** (추정) |

#### 🔗 연계 활용
- **내 매물 기준 뷰** → S등급 → "매수자에게 이 딜카드 전송하기" 클릭
- **내 고객 기준 뷰** → "이 매수자를 위한 맞춤 보고서 작성" 클릭
- 매칭 결과 → 파이프라인에 자동 등록 → 진행 단계 추적

## ACT 4. 딜 파이프라인 — 모달 기반 단계 전환 + Gate 응대 수락 (3분)

### 📍 화면: `https://www.credeal.net/broker/pipeline`

#### 보여줄 것

1. **8단계 파이프라인** — 딜 진행 단계별 카드:
   ```
   📝 메모입력 → 📋 딜카드 → 🔒 Gate → 📄 IM작성 → 🤝 미팅 → ✍️ LOI → 📜 계약 → ✅ 완료
   ```

2. **각 카드에 표시되는 정보**:
   - 매물 블라인드명
   - 현재 단계 + **체류일수**
   - 담당 고객명
   - 예상 수수료

3. **⚠️ 체류 경고 시스템** — 단계별 기준일수 초과 시 경고 배지:
   | 단계 | 기준일수 | 초과 시 |
   |------|----------|--------|
   | 메모입력 | 1일 | ⚠️ "딜카드로 전환하세요" |
   | 딜카드 | 7일 | ⚠️ "IM 생성을 고려하세요" |
   | Gate | 7일 | ⚠️ "고객 팔로업이 필요합니다" |
   | IM작성 | 14일 | ⚠️ "미팅을 제안하세요" |
   | LOI | 21일 | ⚠️ "딜이 정체되고 있습니다" |
   | 계약 | 30일 | ⚠️ "클로징 조치가 필요합니다" |

4. **전체 파이프라인 금액** — 상단에 총 예상 수수료 합계

#### 🆕 Step 1: 단계 전환 모달 (StageTransitionModal) 시연

딜 카드의 **"다음 단계로"** 버튼을 클릭하면 **모달 폼**이 열립니다:

| 전환 구간 | 모달에서 필수 입력 항목 |
|----------|----------------------|
| 메모 → 딜카드 | 연결할 SSOT 건물 ID |
| 딜카드 → Gate | Gate 요청 ID |
| Gate → IM 작성 | IM 프로젝트 ID, 준비도 점수 |
| IM → 미팅 | 매수자 의향 ID, 매칭 등급 |
| 미팅 → LOI | 매수자 반응, 가격 괴리 |
| **LOI → 계약** | **합의 가격, 핵심 조건** (수동 입력) |
| **계약 → 완료** | **클로징 날짜, 자금 확인 여부** (수동 입력) |

> [!IMPORTANT]
> LOI와 계약 단계는 시스템이 자동으로 감지할 수 없는 이벤트이므로, **중개인이 모달을 통해 직접 로깅**합니다. 필수 항목(합의 가격, 클로징 날짜 등)을 입력해야만 다음 단계로 전환되어 **데이터 정합성**이 보장됩니다.

#### 🆕 Step 2: Gate 응대 수락 리뷰 (GateRequestReviewModal) 시연

**Gate 단계**의 딜 카드에서 **🛡️ "응대 수락 (리뷰)"** 버튼을 클릭하면 전용 리뷰 모달이 열립니다:

```
┌─────────────────────────────────────────────┐
│  🛡️ Gate Request 리뷰                       │
│                                             │
│  매수자 유형: 법인 (자산운용사)               │
│  예산 범위:   200~300억                      │
│  매입 목적:   사옥 이전                       │
│  NDA 서명:   ✅ 완료됨                       │
│                                             │
│  ┌───────────────┐  ┌───────────────┐       │
│  │ ✅ 수락 및     │  │ ❌ 거절       │       │
│  │    IM 공유     │  │              │       │
│  └───────────────┘  └───────────────┘       │
│                                             │
│        [ 나중에 결정하기 ]                    │
└─────────────────────────────────────────────┘
```

- **수락** → 자동으로 `im_created` 단계로 전환 + IM 공유 프로세스 시작
- **거절** → `failed` 상태로 전환 + 사유 기록
- NDA 미완료 시 경고 배지 표시 → 수락 전 NDA 확인 유도

#### 🎤 멘트 예시
> "파이프라인이 크게 업그레이드됐습니다. 각 단계를 넘길 때마다 **모달 폼에서 핵심 정보를 입력**하게 됩니다. LOI 단계에서는 합의 가격과 핵심 조건을, 계약 단계에서는 클로징 날짜를 기록합니다. Gate 단계에서는 **매수자 자격 리뷰 전용 모달**이 뜹니다 — NDA 서명 여부, 예산, 매입 목적을 확인한 뒤 '수락'하면 IM이 자동으로 공유됩니다. **무마찰이면서도 데이터 정합성이 완벽합니다.**"

#### 💡 프로 팁
- 파이프라인에 딜이 쌓일수록, 대시보드의 **ROI 분석**과 **주간 리포트**가 정확해집니다.
- "메모입력" 단계의 딜은 **유니버설 메모 FAB**에서 자동 추가됩니다.
- 매칭 센터의 S등급 딜은 자동으로 "딜카드" 단계로 파이프라인에 등록됩니다.

## ACT 5. 고객 CRM — 내 고객을 놓치지 않는 시스템 (2분)

### 📍 화면: `https://www.credeal.net/broker/clients`

#### 보여줄 것

1. **고객 리스트** — 이름, 유형(매수/매도/임차), 최근 접촉일
2. **고객 상세 페이지** (`/broker/clients/{id}`):
   - 기본 정보 + 연락처
   - 거래 이력 타임라인
   - 연관 딜카드 / 매수 의향서
   - **AI 큐레이션** 버튼 → 이 고객에게 맞는 매물 자동 추천
3. **새 고객 등록** (`/broker/clients/new`) — 간편 등록 폼

#### 🎤 멘트 예시
> "고객이 6개월 전에 '여의도 쪽 건물 알려줘'라고 했던 거 기억나세요? DealCard는 잊지 않습니다. **AI가 조건에 맞는 새 매물이 들어오면 자동으로 알려줍니다.** 고객과의 관계를 기록으로 관리하세요."

#### 🔗 연계 활용
```
고객 등록 → 매수/임차 의향서 연결 → AI 매칭 자동 실행
                                        ↓
                              S등급 매칭 발생 시
                                        ↓
                              고객에게 IM 링크 자동 공유
```
