## 1. 고객 CRM 관리

### 1.1 개요

**고객 CRM**은 매도자와 매수자를 **'사람' 단위**로 통합 관리하는 기능입니다. 기존에는 매물과 매수자 의향서만 개별적으로 관리했지만, CRM을 통해 **고객 중심**으로 모든 정보를 연결합니다.

```
고객 (사람)
├── 매도자 → 보유 매물 (건물) 연결
├── 매수자 → 매수 의향서 연결
└── 연락 이력 → 전화/카톡/미팅 기록
```

### 1.2 고객 목록 페이지

#### 접근 방법

- 하단 네비게이션 → **👥 고객** 탭
- URL: `/broker/clients`

#### 화면 구성

**상단**: 고객 수 표시 + **`+ 등록`** 버튼

**검색**: 이름, 회사, 연락처로 실시간 검색

**필터 (2단)**:

| 1단 필터 | 2단 필터 |
|---------|---------|
| 전체 / 매도자 / 매수자 / 매도·매수 | 전체 등급 / VIP / 일반 / 잠재 / 휴면 |

**고객 카드**: 각 고객이 카드로 표시되며, 다음 정보를 한눈에 확인할 수 있습니다.

| 표시 정보 | 설명 |
|----------|------|
| 아이콘 | 🏠 매도자 / 🎯 매수자 / 🔄 매도·매수 |
| 이름 | 고객 이름 |
| 등급 배지 | ⭐ VIP(금색) / 일반(파랑) / 잠재(노랑) / 휴면(회색) |
| 유형 + 회사 | "매수자 · (주)ABC" |
| 연락처 | 📞 010-0000-0000 |
| 보유 매물 수 | 🏢 3건 (매도자인 경우) |
| 의향서 수 | 🎯 2건 (매수자인 경우) |
| 등록일 | 고객 등록 날짜 |

### 1.3 고객 등록

#### 접근 방법

- 고객 목록 → **`+ 등록`** 버튼
- 코크핏 대시보드 → **👥 고객 등록** 빠른 작업
- URL: `/broker/clients/new`

#### 입력 항목

| 항목 | 필수 여부 | 설명 |
|------|----------|------|
| **고객 유형** | 필수 | 🏠 매도자 / 🎯 매수자 / 🔄 매도·매수 |
| **이름** | 필수 | 개인명 또는 법인명 |
| **회사/법인명** | 선택 | 소속 회사 또는 법인 |
| **연락처** | 선택 | 전화번호 |
| **이메일** | 선택 | 이메일 주소 |
| **고객 등급** | 선택 | ⭐ VIP / 일반(기본) / 잠재 |
| **메모** | 선택 | 특이사항, 선호 조건, 상담 내용 |

#### 등록 절차

1. 각 항목을 입력합니다.
2. **`고객 등록하기`** 버튼을 클릭합니다.
3. 등록 완료 후 자동으로 **고객 상세 페이지**로 이동합니다.

### 1.4 고객 상세 페이지

#### 접근 방법

- 고객 목록 → 고객 카드 클릭
- URL: `/broker/clients/[id]`

#### 화면 구성

##### ① 고객 프로필 헤더

이름, 등급 배지, 고객 유형, 소속이 표시됩니다.

##### ② 연락처 정보

전화번호(클릭 시 바로 통화)와 이메일이 표시됩니다. 메모가 있으면 하단에 회색 박스로 표시됩니다.

##### ③ 관련 매물 (매도자인 경우)

이 고객에게 연결된 매물 목록이 표시됩니다. 각 매물을 클릭하면 딜카드 상세로 이동합니다.

##### ④ 매수 조건 (매수자인 경우)

이 고객에게 연결된 매수자 의향서 목록이 표시됩니다.

##### ⑤ 연락 이력 (타임라인)

고객과의 모든 연락 이력이 시간순으로 타임라인 형태로 표시됩니다.

### 1.5 연락 기록 추가

#### 방법

1. 고객 상세 페이지 → 연락 이력 섹션 → **`+ 기록 추가`** 클릭
2. 연락 유형 선택:

| 유형 | 아이콘 | 설명 |
|------|--------|------|
| 전화 | 📞 | 전화 통화 기록 |
| 카카오톡 | 💬 | 카카오톡 메시지 기록 |
| 문자 | 📱 | SMS 기록 |
| 이메일 | 📧 | 이메일 기록 |
| 미팅 | 🤝 | 대면 미팅 기록 |
| 현장 방문 | 🏢 | 매물 현장 방문 기록 |
| 메모 | 📝 | 기타 내부 메모 |

3. 내용을 간단히 입력합니다.
4. **`기록 저장`** 버튼을 클릭합니다.

> **💡 Tip**: 전화 후 바로 기록하는 습관을 들이면, 나중에 고객의 요구사항 변화를 추적할 수 있습니다.

## 2. 매칭 센터

### 2.1 개요

**매칭 센터**는 시스템이 자동으로 수행한 **매물↔매수자 매칭 결과**를 한눈에 확인하는 대시보드입니다.

### 2.2 접근 방법

- 하단 네비게이션 → **🎯 매칭** 탭
- 코크핏 대시보드 → KPI 카드 중 **S/A 매칭** 클릭
- URL: `/broker/matching`

### 2.3 매칭 등급 이해

시스템은 3단계 AI 매칭 엔진을 통해 S/A/B/C 등급을 부여합니다.

| 등급 | 색상 | 의미 | 권장 액션 |
|------|------|------|----------|
| **S** | 🟡 금색 | 최우선 — 모든 조건이 높은 수준으로 부합 | 즉시 연락하여 딜 진행 |
| **A** | 🟢 녹색 | 높은 적합도 — 대부분의 조건이 부합 | 우선 순위로 접근 |
| **B** | 🔵 파랑 | 참고 가능 — 일부 조건만 부합 | 조건 조정 시 가능성 있음 |
| **C** | ⚪ 회색 | 매칭 미흡 — 적합하지 않음 | 참고용으로만 보관 |

### 2.4 3단계 매칭 엔진 작동 원리

```
[1단계] Hard Filter (경성 필터)
  └─ 예산 범위, 지역, 자산 유형 일치 여부
      → 불일치 시 즉시 탈락

[2단계] Semantic Matching (의미론적 매칭)
  └─ 매수 목적, 선호 조건, 건물 특성 간 AI 유사도 분석
      → 0~100점 점수 산출

[3단계] Ensemble Scoring (종합 평가)
  └─ Hard Filter 결과 + Semantic Score + 가중치
      → S/A/B/C 등급 부여
```

### 2.5 매칭 보드 사용법

#### 등급 요약 카드

화면 상단에 4개 등급별 매칭 건수가 표시됩니다. 각 카드를 클릭하면 해당 등급만 필터링됩니다. 다시 클릭하면 전체 보기로 돌아갑니다.

```
┌────────┬────────┬────────┬────────┐
│  S  5  │  A  8  │  B 12  │  C  3  │
└────────┴────────┴────────┴────────┘
```

#### 매칭 결과 카드

각 매칭 결과는 다음 정보를 카드로 표시합니다:

| 영역 | 표시 내용 |
|------|----------|
| 등급 배지 | S/A/B/C 등급 + 색상 |
| 매물 정보 | 권역 + 자산유형 + 가격대 |
| 매칭 점수 | 0~100점 |
| 매수자 정보 | 🎯 유형 + 예산 + 매수목적 |
| 매칭 사유 | AI가 생성한 매칭 이유 설명 |
| 날짜 | 매칭 수행일 |
| 바로가기 | "딜카드 보기" / "매수자 보기" 링크 |

> **💡 Tip**: S급 매칭이 발생하면 즉시 매수자에게 블라인드 딜카드를 전달하세요. 최적의 타이밍을 놓치지 마세요!

## 3. 딜 파이프라인 칸반 보드

### 3.1 개요

**딜 파이프라인**은 매물의 거래 라이프사이클을 8단계로 관리하는 **칸반 보드**입니다. 각 딜이 현재 어느 단계에 있는지 한눈에 파악할 수 있습니다.

### 3.2 접근 방법

- 하단 네비게이션 → **📊 파이프** 탭
- 코크핏 대시보드 → 파이프라인 미니 현황 클릭
- URL: `/broker/pipeline`

### 3.3 8단계 파이프라인

| 단계 | 이모지 | 설명 | 필요 조건 |
|------|--------|------|----------|
| **메모 입력** | 📝 | 카톡 메모가 시스템에 입력된 상태 | 메모 텍스트 |
| **딜카드 생성** | 📋 | AI가 블라인드 딜카드를 생성한 상태 | 건물 SSoT 생성 |
| **Gate 요청** | 🔒 | 매수자가 상세 정보 열람을 요청한 상태 | Gate 요청서 |
| **IM 작성** | 📄 | 투자 설명서(IM)를 작성 중인 상태 | IM 문서 |
| **매수자 미팅** | 🤝 | 매수자와 미팅/현장 방문 단계 | 미팅 일정 |
| **LOI** | ✍️ | 의향서(Letter of Intent) 제출 단계 | LOI 문서 |
| **계약** | 📜 | 매매계약서 체결 단계 | 계약 조건 합의 |
| **완료** | ✅ | 거래 완료 (잔금 처리 등) | 잔금 완료 |

### 3.4 칸반 보드 사용법

#### 스테이지별 그룹

각 단계별로 해당 딜 목록이 세로로 그룹핑됩니다. 각 스테이지 헤더에는 이모지와 딜 수가 표시됩니다.

#### 딜 카드 정보

각 딜 카드에는 다음 정보가 표시됩니다:

| 정보 | 설명 |
|------|------|
| 매물 요약 | 권역 + 자산유형 |
| 가격대 | 매물 가격 범위 |
| 매칭 수 | 🎯 매칭된 매수자 수 |
| 체류 일수 | 현 단계에 머문 일수 |
| ⚠️ 경고 | 체류 기간 초과 시 빨간 경고 배지 |

#### 체류 경고 기준

| 단계 | 경고 기준 | 의미 |
|------|----------|------|
| 메모 입력 | 1일 | 메모만 남기고 딜카드를 안 만든 경우 |
| 딜카드 생성 | 7일 | 딜카드 생성 후 Gate 요청이 없는 경우 |
| Gate 요청 | 7일 | Gate 요청 후 진전이 없는 경우 |
| IM 작성 | 14일 | IM 작성 후 미팅으로 이어지지 않는 경우 |
| 매수자 미팅 | 14일 | 미팅 후 LOI로 이어지지 않는 경우 |
| LOI | 21일 | LOI 제출 후 계약으로 이어지지 않는 경우 |
| 계약 | 30일 | 계약 체결 후 잔금 처리가 지연되는 경우 |

> **⚠️ 주의**: 빨간 경고 배지(⚠️)가 표시된 딜은 즉시 현황을 점검하고, 필요 시 매도자/매수자에게 연락하세요.

#### 딜 상세 이동

각 딜 카드를 클릭하면 해당 딜카드 상세 페이지로 이동합니다.

## 4. Gate 시스템 (정보 단계별 공개)

### 4.1 개요

**Gate 시스템**은 매물의 민감 정보를 **3단계로 나누어 순차적으로 공개**하는 보안 메커니즘입니다.

### 4.2 3단계 Gate

| Gate | 공개 범위 | 요구 사항 |
|------|----------|----------|
| **G1** | 기본 정보 (권역, 가격대, 자산유형) | 매수자 기본 인증 |
| **G2** | 상세 정보 (정확한 주소, 임대 현황) | NDA 서명 |
| **G3** | 전체 정보 (등기, 임대차 원문, 매도자 정보) | 실사 진행 합의 |

### 4.3 Gate 요청 처리

1. 딜카드 상세 페이지 하단의 **Gate 요청 폼**에서 요청이 들어옵니다.
2. 중개인은 요청을 확인하고 **승인** 또는 **반려**할 수 있습니다.
3. 승인 시 해당 단계의 정보가 매수자에게 공개됩니다.

## 5. 모바일 IM & Full IM

### 5.1 모바일 IM

딜카드 상세 페이지 → **📱 모바일 IM 만들기** 버튼

모바일에 최적화된 투자 설명서를 자동 생성합니다. 매수자에게 카카오톡이나 문자로 공유할 수 있는 링크가 제공됩니다.

특징:
- 모바일 화면에 최적화된 레이아웃
- 빠른 로딩 속도
- 핵심 정보만 압축 표시
- 터치 인터랙션 지원

### 5.2 Full IM (전체 투자 설명서)

딜카드 상세 페이지 → **📄 Full IM 만들기** 버튼

전문적인 투자 설명서를 생성합니다. 기관 투자자나 법인 매수자에게 공식 문서로 전달하기에 적합합니다.

포함 내용:
- 건물 개요 및 위치 분석
- 임대 현황 요약
- 투자 수익률 분석
- 리스크 요인 정리
- 건물주 리포트

### 5.3 Space AI 분석

딜카드 상세 페이지 → **🔍 Space AI 분석** 버튼

건물의 공간 활용 최적화 AI 분석을 제공합니다.

## 6. 전문가 코멘트 시스템

### 6.1 접근 방법

- 딜카드 상세 페이지 → **✨ 전문가 코멘트 받기** 버튼
- URL: `/expert-note/request?building=[id]`

### 6.2 기능

AI가 해당 매물에 대해 전문가 관점의 **3줄 요약 의견**을 생성합니다:

1. **투자 관점**: 이 매물의 투자 매력은?
2. **리스크 관점**: 주의해야 할 점은?
3. **추천 관점**: 어떤 매수자에게 적합한가?

## 7. 건물주 준비도 체크

### 7.1 접근 방법

- URL: `/owner-readiness`

### 7.2 기능

건물주(매도자)의 매도 준비 상태를 체크리스트로 평가합니다:

- 매도 의사 확정 여부
- 희망 매도가 설정 여부
- 필요 서류 준비 상태
- 임차인 동의 여부 (해당 시)
- 법적 이슈 확인 여부

이 정보는 딜 파이프라인 진행 시 참고자료로 활용됩니다.

## 8. 업무 시나리오별 활용 가이드

```

5. 📞 연락 기록 → 매수자 반응 기록
6. 📊 파이프라인 → 거래 진행 단계 추적
```

## 9. 자주 묻는 질문 (FAQ)

### Q1. 딜카드를 수정할 수 있나요?

A: 딜카드는 AI가 자동 생성한 초안이며, 현재 직접 수정 기능은 없습니다. 메모를 다시 입력하면 새로운 딜카드를 생성할 수 있습니다.

### Q2. 매칭은 언제 수행되나요?

A: 매칭은 다음 시점에 자동으로 수행됩니다:
- 새 매물(딜카드)이 생성될 때
- 새 매수자 의향서가 등록될 때

기존 데이터와의 매칭도 자동으로 실행됩니다.

### Q3. 고객 정보는 다른 중개인에게 공유되나요?

A: **아닙니다.** 모든 데이터에 RLS(Row Level Security) 정책이 적용되어 있어, 본인이 등록한 고객 정보만 조회할 수 있습니다.

### Q4. 카톡 문구를 카카오톡으로 어떻게 보내나요?

A: **`카톡 문구 복사`** 버튼을 클릭하면 클립보드에 자동으로 복사됩니다. 카카오톡 대화창에서 붙여넣기(Ctrl+V 또는 길게 눌러 붙여넣기)하면 됩니다.

### Q5. 한 고객이 매도자이면서 매수자일 수 있나요?

A: **네.** 고객 등록 시 유형을 **🔄 매도/매수**로 선택하면 됩니다. 이 경우 보유 매물과 매수 의향서 모두 연결할 수 있습니다.

### Q6. 숨긴 정보를 나중에 공개할 수 있나요?

A: **네.** Gate 시스템을 통해 단계적으로 공개할 수 있습니다. G1 → G2 → G3 순서로 정보 공개 범위가 넓어집니다.

### Q7. 매칭 등급 기준은 무엇인가요?

A: 3단계 AI 매칭 엔진이 예산/지역/자산유형(Hard Filter) → 매수목적/선호조건(Semantic) → 종합점수(Ensemble)를 산출하여 S(80점 이상) / A(60~79점) / B(40~59점) / C(40점 미만)로 분류합니다.

### Q8. 모바일에서도 사용할 수 있나요?

A: **네.** 모든 페이지가 모바일에 최적화되어 있습니다. 하단 네비게이션 바를 통해 한 손으로 편리하게 탐색할 수 있습니다.

### Q9. 데이터 백업은 되나요?

A: 모든 데이터는 Supabase 클라우드 데이터베이스에 저장되므로 자동 백업됩니다.

### Q10. 파이프라인 단계를 직접 변경할 수 있나요?

A: 파이프라인 단계는 시스템이 자동으로 관리합니다. 딜카드 생성, Gate 요청, IM 작성 등의 작업을 수행하면 자동으로 다음 단계로 전환됩니다.

## 10. 용어 사전

| 용어 | 영문 | 설명 |
|------|------|------|
| **딜카드** | Deal Card | 매물의 핵심 정보를 요약한 블라인드 카드. 민감정보가 자동으로 차단됨 |
| **블라인드 티저** | Blind Teaser | 정확한 주소 없이 매물의 매력을 전달하는 요약 문서 |
| **SSoT** | Single Source of Truth | 건물 정보의 단일 진실 원천. 모든 문서가 이 데이터를 기반으로 생성됨 |
| **IM** | Investment Memorandum | 투자 설명서. 매수자에게 전달하는 상세 분석 문서 |
| **Gate** | Gate | 정보 공개 단계. G1/G2/G3로 순차적으로 민감정보를 공개하는 메커니즘 |
| **매칭 등급** | Matching Grade | S/A/B/C 4등급으로 매물-매수자 적합도를 나타냄 |
| **파이프라인** | Pipeline | 거래가 진행되는 8단계 라이프사이클 (메모→딜카드→Gate→IM→미팅→LOI→계약→완료) |
| **CRM** | Customer Relationship Management | 고객 관계 관리. 매도자/매수자를 '사람' 단위로 관리하는 시스템 |
| **RLS** | Row Level Security | 행 수준 보안. 각 중개인이 자기 데이터만 볼 수 있도록 하는 데이터베이스 보안 정책 |
| **LOI** | Letter of Intent | 매수 의향서. 매수자가 매수 의사를 공식적으로 표명하는 문서 |
| **NDA** | Non-Disclosure Agreement | 비밀유지계약서. Gate 시스템에서 상세 정보 열람 전 서명 |
| **Hard Filter** | Hard Filter | 경성 필터. 예산, 지역, 자산유형 등 절대 조건에 의한 1차 필터링 |
| **Semantic Matching** | Semantic Matching | 의미론적 매칭. AI가 텍스트 의미를 분석하여 적합도를 평가하는 2차 필터링 |
| **건물주 준비도** | Owner Readiness | 매도자의 매도 준비 상태를 체크하는 평가 시스템 |
| **코크핏** | Cockpit | 중개인 전용 통합 대시보드. 모든 핵심 정보를 한 화면에서 관리 |
