## TC-1.2 IM Lite (투자설명서) 생성

### TC-1.2.1 IM 데이터 바텀시트 — 보강 데이터 입력

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/broker/deal-card/{id}` → "📱 모바일 투자설명서 만들기" 버튼 클릭 | IM 데이터 바텀시트 (`ImDataBottomSheet`) 오픈 |
| 2 | Readiness Score 게이지 확인 | 현재 점수 / 55점 기준 표시 (area +10, price +10, asset +10, address +25, rent +20, vacancy +10, comment +5) |
| 3 | 🏠 건물 주소 검색 → 키워드 입력 | `GET /api/public/address?keyword=...` → 실시간 검색 결과 표시 |
| 4 | 검색 결과 주소 선택 | PNU 자동 결정, Readiness +25점 반영 |
| 5 | 💰 월 임대료 총액 입력 (만원) | 숫자 입력 → Readiness +20점 반영 |
| 6 | 📊 공실률 퀵셀렉트 (만실 / ~10% / ~20% / ~30%) | 버튼 선택 → Readiness +10점 반영 |
| 6a | 📋 **렌트롤 CSV/XLSX 업로드** (선택) | `RentRollImporter` → 파일 선택 → 층별 임대차 데이터 자동 파싱 (`xlsx` 라이브러리), `floor_leases[]` 배열로 주입 |
| 6b | 업로드 결과 미리보기 확인 | 층별 임차인/면적/보증금/월세/관리비/계약기간 테이블 표시 |
| 6c | **WALE 지표 자동 계산** 확인 | 렌트롤 기반 임대료 가중평균 잔여기간(WALE) 및 12개월 내 만기 임대료 비중(%) 표시 |
| 7 | 📝 브로커 한줄 코멘트 입력 | 텍스트 입력 → Readiness +5점 반영 |
| 8 | Readiness ≥ 55 시 "AI 투자설명서 생성" 버튼 활성화 | 버튼 활성화 확인 |
| 9 | 생성 버튼 클릭 | `createMobileIMAction()` → 프로그레스 5단계: ① 데이터 검증 → ② 공공데이터 수집(건축물대장/공시지가/토지이용/실거래/SEMAS 상권) → ③ RAG 유사사례 검색 → ④ Few-shot 예시 선별 → ⑤ AI 7섹션 생성 |
| 9a | **SEMAS 상권 분석 데이터** 수집 확인 | PNU → 법정동코드 변환 → 소상공인진흥공단 5개 엔드포인트 병렬 호출 (점포현황/업종밀도/유동인구/매출지수/폐업률) |
| 9b | **비교사례 벤치마킹** 자동 산출 확인 | 주변 시세 대비 대상 물건의 경쟁력 판정 (Highly Competitive / Market Rate / Overpriced) |
| 10 | — | 성공 시 `✅ {N}섹션 생성 완료!` → 1.5초 후 `/broker/im-approval/{docId}` 리다이렉트 |

### TC-1.2.2 IM 승인 리뷰 — 인라인 편집 & 검증

| 단계 | 액션 | 기대 결과 |

| 9 | "승인 및 공개" 클릭 | `POST /api/broker/im-lite/{docId}/approve` → status: `published` |

## TC-1.9 유니버설 메모 — 딜 생성 연계

### TC-1.9.1 음성 메모 → 딜카드 라우팅

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 화면 우측 하단 메모 FAB 아이콘 클릭 | 메모 모드 선택 시트 오픈 |
| 2 | "🎤 음성 메모" 선택 | VoiceRecorder UI 활성화, 마이크 권한 요청 |
| 3 | 음성 녹음 → 중지 | STT 변환 (`/api/broker/memo/voice`) → 텍스트 확인 |
| 4 | 변환된 텍스트 확인 후 "저장" | `/api/broker/memo` 호출 → AI 라우팅 (new_deal / buyer_condition / update_building / **schedule_event** / general) |
| 4a | 라우팅 결과가 `schedule_event`인 경우 | "일정 관리로 이동" CTA 표시 → 클릭 시 `/broker/schedule`로 이동 |
| 5 | 라우팅 결과가 `new_deal`인 경우 | "딜카드 등록으로 이동" CTA 표시 → 클릭 시 `/broker/deal-card/new`로 이동 |

### TC-1.9.2 텍스트 메모 → 저장

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 메모 FAB → "📝 텍스트 메모" 선택 | 텍스트 입력 영역 활성화 |
| 2 | 메모 입력 후 "저장" | AI 라우팅 결과 + 메모함에 저장 (`/api/broker/memo/save`) |

---

> [!IMPORTANT]
> **각 TC의 성공 판정 기준**: 해당 단계에서의 HTTP 응답 코드가 2xx이고, UI에 에러 토스트가 발생하지 않으며, 기대 결과에 기술된 데이터가 화면에 정상 렌더링되어야 합니다.
