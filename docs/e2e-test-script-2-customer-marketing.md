## TC-2.1 매수자 의향서 (Buyer Intents)

### TC-2.1.1 매수 의향서 신규 등록

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 코크핏 → "🎯 매수자 등록" 퀵 액션 클릭 | `/broker/buyer-intents/new` 이동 |
| 2 | 필수 정보 입력 | 아래 입력 항목 참조 |
| 3 | "등록" 버튼 클릭 | 의향서 생성 → 상세 페이지 리다이렉트 |

**입력 항목:**

| 필드 | 샘플 값 | 유효성 |
|------|---------|--------|
| 매수자 유형 (buyer_type) | 법인 | 필수 |
| 예산 범위 (budget_min ~ max) | 50억 ~ 100억 | 필수 |
| 선호 권역 (preferred_regions) | 성수동, 강남, GBD | 최소 1개 |
| 자산 유형 (asset_types) | 근린생활시설, 오피스 | 최소 1개 |
| 투자 목적 (purchase_purpose) | 임대수익 💰 | 필수 |
| 위험 성향 (risk_tolerance) | 보수적 | 선택 |

### TC-2.1.2 매수 의향서 — 메모 기반 자동 생성

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 유니버설 메모 → "매수자가 50~100억 범위에서 성수동 근생빌딩 찾고 있음. 수익률 4% 이상 선호" 입력 | AI 라우팅 결과: `buyer_condition` |
| 2 | "매수자 등록으로 이동" CTA 클릭 | `/broker/buyer-intents/new`로 이동, 메모 내용이 사전 입력됨 |

### TC-2.1.3 매수 의향서 상세 페이지

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/broker/buyer-intents/{id}` 접속 | 의향서 상세 정보 렌더링 (viewer 모드) |
| 2 | "편집" 버튼 클릭 | buyer-intent-edit-form 활성화 (17KB 폼) |
| 3 | 예산 범위 변경 → "저장" | PUT API 호출 → 성공 토스트 |
| 4 | "📊 매칭 히스토리" 탭 확인 | 이 매수자와 매칭된 매물 이력 표시 (buyer-match-history) |
| 5 | "📝 매수자 메모" 섹션 확인 | 메모 추가/수정/삭제 가능 (buyer-memo-section, 30KB) |
| 6 | 새 메모 작성 → "저장" | 메모 저장 성공 |

### TC-2.1.4 매수 의향서 목록

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/broker/buyer-intents` 접속 | 등록된 매수 의향서 목록 렌더링 |
| 2 | 의향서 카드 내 투자 목적 이모지 확인 | 💰 임대수익, 🏛️ 사옥, 🎁 증여, 🏗️ 개발, 📈 갭투자 |
| 3 | 카드 클릭 | 상세 페이지 이동 |

## TC-2.3 고객 CRM

### TC-2.3.1 고객 신규 등록

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 더보기(☰) → "고객 CRM" 클릭 | `/broker/clients` 이동 |
| 2 | "➕ 신규 고객 등록" 클릭 | `/broker/clients/new` 이동 |
| 3 | 고객 정보 입력 | 아래 항목 참조 |
| 4 | "등록" 클릭 | `POST /api/broker/clients` → 성공 → 상세 페이지 이동 |

**입력 항목:**

| 필드 | 샘플 값 | 유효성 |
|------|---------|--------|
| 이름 (display_name) | 홍길동 | 필수 |
| 유형 (client_type) | buyer | 필수 (seller/buyer/both) |
| 회사 (company) | JS부동산 | 선택 |
| 연락처 (phone) | 010-1234-5678 | 선택 |
| 등급 (tier) | VIP | 기본: normal |
| 태그 (tags) | ["강남", "사옥"] | 선택 |

### TC-2.3.2 고객 목록 필터링

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/broker/clients` 에서 유형 필터 "매도자" 선택 | seller 타입만 필터링 |
| 2 | 유형 필터 "매수자" 선택 | buyer 타입만 필터링 |
| 3 | 등급 필터 "VIP" 선택 | VIP 등급만 필터링 |
| 4 | 검색 키워드 입력 | 이름/회사 기반 검색 결과 |

### TC-2.3.3 고객 상세 — 연락 이력 관리

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/broker/clients/{id}` 접속 | 고객 상세 정보 + 연결된 매물/의향서 + **예약 이력** 표시 |
| 1a | "📅 임장/미팅 예약" 섹션 확인 | 연결된 buyer_intent의 bookings 목록 표시 (상태 배지: 확정🟢/대기🟡/완료🔘) |
| 2 | "➕ 연락 이력 추가" 클릭 | 연락 유형(전화/문자/미팅/이메일), 메모 입력 폼 |
| 3 | 이력 저장 | `POST /api/broker/clients/{id}/contacts` → 이력 리스트에 추가 |
| 4 | 기존 연락 이력 "편집" 클릭 | 인라인 편집 → 수정 저장 |
| 5 | 연락 이력 "삭제" 클릭 | `DELETE /api/broker/clients/{id}/contacts/{contactId}` → 삭제 확인 |

### TC-2.3.4 고객 AI 큐레이션

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 고객 상세 → "AI 큐레이션" 클릭 | `POST /api/broker/clients/{id}/curation` 호출 |
| 2 | — | AI가 이 고객의 프로필/이력에 기반한 매물 추천 결과 표시 |

### TC-2.3.5 고객 정보 수정 및 삭제

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | 고객 상세 → 프로필 필드 수정 → 저장 | `PUT /api/broker/clients/{id}` → 성공 |
| 2 | 등급(Tier) VIP → 휴면 변경 | 즉시 반영, 배지 색상 변경 |

## TC-2.7 Vibe AI 명함 — 생성·공유·분석 전체 플로우

|------|------|-----------|
