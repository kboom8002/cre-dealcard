## 1. 모바일 IM vs 기존 방식 비교 분석

### 1-1. 3가지 IM 작성 방식 종합 비교

| 항목 | 🅰️ Full IM (수동) | 🅱️ AI 개인 작성 | 🅲️ 크리딜 모바일 IM |
|------|------------------|----------------|------------------|
| **작성 시간** | 3~7일 | 2~4시간 | **5분** |
| **작성 비용** | 50~200만원 | 무료~5만원 | **무료** (베타) |
| **공공데이터 연동** | ❌ 수동 | ❌ 수동 | ✅ **자동 7종** |
| **재무 분석** | ⚠️ 엑셀 직접 | ⚠️ AI (검증 어려움) | ✅ **자동 + 교차검증** |
| **DCF 10년** | ⚠️ 전문가만 | ❌ 환각 위험 | ✅ **자동 산출** |
| **정보 블라인드** | ❌ 수동 편집 | ❌ 수동 | ✅ **Disclosure Guard** |
| **공유 방식** | 📧 이메일 | 📧 이메일/파일 | ✅ **카카오톡 1탭** |
| **모바일 최적화** | ❌ PDF | ❌ 문서 | ✅ **네이티브 모바일** |
| **환각 방지** | N/A | ❌ 없음 | ✅ **Hallucination Guard** |
| **섹션 편집** | 전체 재작성 | 전체 재작성 | ✅ **모바일 탭 에디터** |
| **섹션 숨김** | ❌ | ❌ | ✅ **선택적 숨김/공개** |
| **사진 캡션** | 수동 편집 | 수동 | ✅ **인라인 캡션 편집** |
| **IM 관리** | 파일 관리 | 파일 관리 | ✅ **보관함 수정/삭제** |
| **지도 정확도** | 별도 첨부 | ❌ | ✅ **카카오 로컬 API** |

### 1-2. 비용 효율성

- Full IM 대비 **288배** 효율 개선
- AI 개인 대비 **36배** 효율 개선

## 2. 크리딜 모바일 IM 파이프라인 감사 결과

### 2-1. 외부 데이터 소스 연동 현황

| API | 데이터 | 활용 | 상태 |
|-----|--------|------|------|
| 국토부 건축물대장 | 면적, 구조, 용도, 층수 | 1섹션 | ✅ 연동 |
| 토지이음 | 용도지역, 건폐율/용적률 | 2섹션 | ✅ 연동 |
| 공시지가 API | ㎡당 공시지가 | 2섹션 | ✅ 연동 |
| 카카오 POI | 지하철역, 편의시설 | 2섹션 | ✅ 연동 |
| **카카오 로컬 API** | **주소→좌표 지오코딩** | **지도** | ✅ **v3.0 신규** |
| 등기정보광장 | 등기 요약 | 5섹션 | ⚠️ 부분 |

### 2-2. 재무 엔진 상세

| 계산 항목 | 공식 |
|----------|------|
| **NOI** | (월세×12) + (관리비×12) - 공실손실 |
| **Cap Rate** | NOI ÷ 매각가 × 100 |
| **자기자본** | 매각가 - 대출 - 보증금 |
| **레버리지 수익률** | NOI ÷ 자기자본 × 100 |
| **10년 DCF NPV** | Σ(연도별 NCF ÷ (1+WACC)^n) |
| **WALE** | Σ(면적 × 잔여기간) ÷ Σ(면적) |

## 3. 특장점 심층 분석

### 3-1. 정보 보호 자동화 (Disclosure Guard)

| 블라인드 대상 | 처리 방식 |
|-------------|---------|
| 정확한 주소 | 자동 제거 → "강남 핵심 상권" |
| 건물명 | 자동 제거 |
| 소유자명 | 완전 차단 |
| 임차인 실명 | 자동 제거 → "프랜차이즈 카페" |

### 3-2. 모바일 네이티브 경험 (v3.0 고도화)

| 기능 | 설명 | v3.0 변경 |
|------|------|----------|
| 1탭 공유 | 카카오톡/LINE 즉시 공유 | — |
| 지도 통합 | 카카오 로컬 API + 카카오맵 | ✅ Kakao 지오코딩 |
| 사진 슬라이더 | 갤러리 + 라이트박스 + 캡션 | ✅ 캡션 표시 |
| 섹션 편집 | 마크다운 인라인 편집 | ✅ 모바일 탭 전환 |
| 섹션 숨김 | 선택적 섹션 비공개 | ✅ NEW |
| 등급 모니터링 | 편집자 전용 품질 패널 | ✅ NEW |
| IM 보관함 | 수정/삭제 관리 | ✅ NEW |

### 3-3. 지도 정확도 개선 (v3.0)

기존 Nominatim(OpenStreetMap) → **카카오 로컬 API**로 교체:
- 한국 주소 정확도 대폭 향상
- 주소 검색 → 키워드 검색 이중 매칭
- Nominatim은 fallback으로 유지

## 5. 개선 의견 수집 프레임워크

### 5-1. 의견 분류 체계

| 카테고리 | 코드 | 설명 |
|---------|------|------|
| **UI/UX** | UX | 화면 구성, 사용성 |
| **데이터 정확성** | DA | 수치, 계산 오류 |
| **AI 품질** | AQ | AI 생성 내용 문제 |
| **기능 추가** | FR | 없는 기능 요청 |
| **성능** | PF | 속도, 안정성 |
| **비즈니스** | BZ | 업무 흐름, 가치 |

### 5-2. v3.0 고도화 기능 관련 의견 수집 포인트 (NEW)

#### 📌 모바일 편집기 관련
- [ ] 편집/미리보기 탭 전환이 직관적인가?
- [ ] 모바일에서 마크다운 편집 시 가독성이 충분한가?
- [ ] PC Split-pane과 모바일 탭 전환의 일관성이 있는가?

#### 📌 섹션 숨김 관련
- [ ] 숨김 토글 버튼의 위치가 직관적인가?
- [ ] 반투명 표시가 "숨김 상태"를 명확히 전달하는가?
- [ ] 어떤 상황에서 섹션 숨김을 활용하고 싶은가?

#### 📌 사진 캡션 관련
- [ ] 캡션 입력란의 크기가 적절한가?
- [ ] 캡션이 공개 IM 갤러리에서 가독성이 좋은가?
- [ ] 사진 순서 변경 기능이 필요한가?

#### 📌 데이터 품질 모니터링 관련
- [ ] Readiness Score와 Data Quality 등급의 의미가 이해되는가?
- [ ] 등급 기준이 현실적인가?

#### 📌 IM 보관함 관련
- [ ] 수정/삭제 버튼의 위치가 적절한가?
- [ ] 삭제 확인 모달이 실수 방지에 충분한가?
- [ ] IM 목록의 정렬/검색 기능이 필요한가?

#### 📌 지도 관련
- [ ] 카카오 지도의 위치 정확도가 만족스러운가?
- [ ] 주소 표시 방식이 적절한가? (블라인드 유지 확인)

## 부록: 용어 정리

| 용어 | 의미 |
|------|------|
| **IM** | Investment Memorandum, 투자설명서 |
| **SSoT** | Single Source of Truth, 건물 정보 단일 진실 원본 |
| **NOI** | Net Operating Income, 순영업이익 |
| **Cap Rate** | Capitalization Rate, 자본환원율 |
| **DCF** | Discounted Cash Flow, 할인현금흐름 |
| **WALE** | Weighted Average Lease Expiry, 가중평균 잔여 임대기간 |
| **WACC** | Weighted Average Cost of Capital, 가중평균자본비용 |
| **NPV** | Net Present Value, 순현재가치 |
| **Disclosure Guard** | 민감정보 자동 블라인드 처리 시스템 |
| **Hallucination Guard** | AI 환각 탐지 시스템 |
| **Provenance** | 데이터 출처 추적 시스템 |
| **Vibe 명함** | 중개인 디지털 프로필 카드 |
| **Data Quality** | A~D 등급 데이터 품질 지표 (v3.0 편집자 전용) |
| **Readiness Score** | IM 생성 준비도 점수 |
