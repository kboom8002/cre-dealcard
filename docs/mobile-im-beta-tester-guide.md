## 1. 기획 의도: 왜 모바일 IM인가?

### 1-1. 해결하려는 문제

상업용 부동산 중개 현장에는 세 가지 고질적 문제가 있습니다:

| # | 문제 | 현재 상황 |
|---|------|---------|
| ① | **투자설명서(IM) 작성이 너무 오래 걸린다** | 30~50페이지, 3~7일, 50~200만원 |
| ② | **모바일 시대에 PDF는 부적합하다** | 투자자는 카카오톡으로 매물을 받고 싶어한다 |
| ③ | **AI로 직접 쓰면 환각·정보유출 위험이 있다** | ChatGPT에 매물 정보를 넣으면 정확성 검증 불가 |

### 1-2. 크리딜의 해법

```
브로커 메모 5분 입력
     ↓
AI + 공공데이터 7종 자동 연동 + 3중 가드레일
     ↓
7섹션 모바일 IM 자동 생성
     ↓
카카오톡 1탭 공유 → 투자자 즉시 열람
```

**한 줄 요약**: 중개인이 5분만 투자하면, AI가 공공데이터를 교차검증하여 전문가 수준의 투자설명서를 만들고, 카카오톡으로 바로 공유할 수 있습니다.

### 1-3. 왜 "모바일 퍼스트"인가?

대한민국 상업용 부동산 시장의 현실:

- 📱 투자자의 **85% 이상**이 첫 매물 정보를 **카카오톡**으로 수신
- 📧 이메일 IM은 열람률이 낮고 피드백 루프가 느림
- 📄 PDF IM은 모바일에서 확대/축소가 불편하고 데이터 분석 불가
- ⚡ 매물 발굴 → 투자자 공유까지 **속도**가 곧 경쟁력

## 2. 모바일 IM vs 기존 방식 비교 분석

### 2-1. 3가지 IM 작성 방식 종합 비교

| 항목 | 🅰️ Full IM (수동) | 🅱️ AI 개인 작성 | 🅲️ 크리딜 모바일 IM |
|------|------------------|----------------|------------------|
| **작성 시간** | 3~7일 | 2~4시간 | **5분** |
| **작성 비용** | 50~200만원 | 무료~5만원 | **무료** (베타) |
| **공공데이터 연동** | ❌ 수동 | ❌ 수동 | ✅ **자동 7종** |
| **재무 분석** | ⚠️ 엑셀 직접 | ⚠️ AI (검증 어려움) | ✅ **자동 + 교차검증** |
| **DCF 10년** | ⚠️ 전문가만 | ❌ 환각 위험 | ✅ **자동 산출** |
| **정보 블라인드** | ❌ 수동 편집 | ❌ 수동 | ✅ **Disclosure Guard** |
| **공유 방식** | 📧 이메일 | 📧 이메일/파일 | ✅ **카카오톡 1탭** |
| **모바일 최적화** | ❌ PDF | ❌ 문서 | ✅ **네이티브 모바일** |
| **OG 이미지 공유** | ❌ | ❌ | ✅ **1200×630 자동 생성** (v4.0) |
| **제목/서브제목 편집** | 전체 재작성 | 전체 재작성 | ✅ **투자 요약 에디터** (v4.0) |
| **환각 방지** | N/A | ❌ 없음 | ✅ **Hallucination Guard** |
| **섹션 편집** | 전체 재작성 | 전체 재작성 | ✅ **모바일 탭 에디터** |
| **중개인 명함** | 수동 첨부 | 없음 | ✅ **Vibe 명함 자동 통합** (v4.0) |
| **IM 관리** | 파일 관리 | 파일 관리 | ✅ **보관함 수정/삭제** |

### 2-2. 비용 효율성

- Full IM 대비 **288배** 효율 개선
- AI 개인 대비 **36배** 효율 개선
- OG 공유까지 포함하면 **추가 마케팅 비용 0원**

## 3. 핵심 기능 3종 실습 가이드

### 🎯 실습 1: Vibe 명함 만들기 (5분)

```
1. 로그인 → 브로커 홈
2. 좌측 메뉴 "🃏 나의 명함" 클릭
3. 프로필 사진 업로드
4. 이름, 회사명, 전문 분야 입력
5. FAQ 1~3개 입력 (예: "전문 취급 물건은?", "수수료 기준은?")
6. 디자인 템플릿 선택
7. "명함 저장" → 완료!
```

**확인**: `/vibe-card/{slug}` URL로 접속하여 명함이 정상 표시되는지 확인

---

### 🎯 실습 2: 모바일 IM 생성 + 투자 요약 편집 (15분)

```
1. 메모함 → 딜카드 선택 → "🚀 모바일 투자설명서 만들기"
2. 바텀시트에서 매각가, 대출, 공실, 사진, 코멘트 입력
3. "IM 생성" → 30~60초 대기
4. ↓ 승인 페이지 자동 이동 ↓

5. [📋 투자 요약 편집]
   ├─ IM 제목 수정 (골든셋 참고)
   │   예: "역삼역 초역세권 만실 꼬마빌딩 매각"
   ├─ 서브제목 작성
   │   예: "Cap Rate 5.2% · 전층 만실 · 장기 임차 기반 안정 수익형"
   ├─ 핵심 투자 포인트 확인/수정
   └─ 💾 투자 요약 저장

6. [🖼️ OG 메타 편집]
   ├─ OG 타이틀 입력
   ├─ OG 설명 입력
   ├─ OG 이미지 미리보기 확인 (1200×630)
   └─ 💾 OG 정보 저장

7. 사진 캡션 → 7섹션 검토 → 승인 및 공개
```

---

### 🎯 실습 3: 카카오톡 공유 테스트 (3분)

```
1. 공개 IM 뷰어 접속 (/im-lite/{buildingId})
2. 하단 🟡 카카오톡 버튼 탭
3. 대화 상대 선택 → 공유
4. ✅ 확인 포인트:
   ├─ OG 이미지가 잘리지 않고 표시되는가?
   ├─ 제목이 편집한 OG 타이틀과 일치하는가?
   ├─ 설명이 편집한 OG 설명과 일치하는가?
   └─ "투자설명서 즉시 열람" 버튼이 동작하는가?
```

## 4. 특장점 심층 분석

### 4-1. 정보 보호 자동화 (Disclosure Guard)

| 블라인드 대상 | 처리 방식 |
|-------------|---------|
| 정확한 주소 | 자동 제거 → "강남 핵심 상권" |
| 건물명 | 자동 제거 |
| 소유자명 | 완전 차단 |
| 임차인 실명 | 자동 제거 → "프랜차이즈 카페" |

### 4-2. 외부 데이터 소스 연동 현황

| API | 데이터 | 활용 | 상태 |
|-----|--------|------|------|
| 국토부 건축물대장 | 면적, 구조, 용도, 층수 | 1섹션 | ✅ 연동 |
| 토지이음 | 용도지역, 건폐율/용적률 | 2섹션 | ✅ 연동 |
| 공시지가 API | ㎡당 공시지가 | 2섹션 | ✅ 연동 |
| 카카오 POI | 지하철역, 편의시설 | 2섹션 | ✅ 연동 |
| **카카오 로컬 API** | **주소→좌표 지오코딩** | **지도** | ✅ 연동 |
| 등기정보광장 | 등기 요약 | 5섹션 | ⚠️ 부분 |

### 4-3. 재무 엔진 상세

| 계산 항목 | 공식 |
|----------|------|
| **NOI** | (월세×12) + (관리비×12) - 공실손실 |
| **Cap Rate** | NOI ÷ 매각가 × 100 |
| **자기자본** | 매각가 - 대출 - 보증금 |
| **레버리지 수익률** | NOI ÷ 자기자본 × 100 |
| **10년 DCF NPV** | Σ(연도별 NCF ÷ (1+WACC)^n) |
| **WALE** | Σ(면적 × 잔여기간) ÷ Σ(면적) |

### 4-4. 가드레일 3중 방어

| 가드레일 | 역할 | 예시 |
|---------|------|------|
| **Risk Boundary** | 확정적 투자 수익 약속 방지 | "반드시 수익" → "추정 수익률" |
| **Disclosure Guard** | 민감 정보 자동 블라인드 | 정확한 주소, 소유자명 자동 제거 |
| **Hallucination Guard** | 비정상 수치 탐지 | Cap Rate 50% 같은 이상값 경고 |

## 5. 신규·고도화 기능 안내


### ⭐ 5-1. 📋 투자 요약 편집 (IM 승인 화면)

**무엇인가?**: IM 승인 화면에서 **제목, 서브제목, 핵심 투자 포인트**를 직접 편집할 수 있는 기능입니다.

| 필드 | 역할 | 공개 IM 표시 위치 |
|------|------|----------------|
| **IM 제목** | 매물의 공식 명칭 | 뷰어 `<h1>` 최상단 + OG 제목 |
| **서브제목** | 핵심 투자 하이라이트 한 줄 헤드카피 | 가격 아래 에메랄드 색상 텍스트 |
| **핵심 투자 포인트** | 왜 이 매물에 투자해야 하는지 1~2문장 | 히어로 카드 내 그래디언트 박스 |

**왜 중요한가?**: AI가 자동 생성한 내용을 중개인이 **현장 감각**으로 보완할 수 있습니다. 특히 서브제목은 투자자가 IM을 열었을 때 **3초 내에 투자 매력을 파악**할 수 있게 해줍니다.

### ⭐ 5-2. 🖼️ OG 공유 메타 + 이미지 수정

**무엇인가?**: 카카오톡/SNS 공유 시 표시되는 제목, 설명, 미리보기 이미지를 관리하는 기능입니다.

| 변경 사항 | v3.0 (이전) | v4.0 (현재) |
|----------|------------|------------|
| 딜카드 OG 이미지 | 건물 사진 → 잘림 | **1200×630 정규 OG** → 잘리지 않음 |
| IM 카카오 공유 제목 | 하드코딩 기본값 | **편집한 OG 타이틀** 우선 |
| IM 카카오 공유 설명 | 하드코딩 기본값 | **편집한 OG 설명** 우선 |

### ⭐ 5-3. IM 제목 문체 자동 개선

**무엇인가?**: AI가 IM 제목 생성 시 `"~계열로 추정"`, `"또는 ~"` 등 어색한 분석적 어투를 자동 제거합니다.

- 변경 전: `"중구권역 근린생활시설 또는 다가구·상가주택 계열로 추정 투자설명서"`
- 변경 후: `"중구권역 근린생활시설 매각"`


### ⭐ 5-4. 🃏 Vibe 명함 (프로필 카드)

**무엇인가?**: 중개인의 디지털 프로필 카드입니다. IM 하단에 자동 표시됩니다.

- 프로필 사진, 이름, 회사, 전문 분야
- 이메일, 전화번호 (1탭 연결)
- 디자인 템플릿 선택 가능 (미드나잇, 글라스모피즘, 네온 등)
- FAQ 1~7개 직접 입력 (아코디언 표시)
- `/broker/my-card` 에서 편집

### ⭐ 5-5. CRE IM 제목 골든셋 (20개)

자산 유형별로 IM 업계 표준 문체의 **20개 참조 예시**가 시스템에 내장되었습니다.
AI 제목 생성 시 참조되며, 중개인이 제목을 직접 편집할 때도 참고할 수 있습니다.

## 8. 🏆 골든셋·퓨샷·용어사전 수집 참여 안내

> 🎯 **이 섹션이 가장 중요합니다.** 여러분의 현장 경험과 전문 용어가 AI의 품질을 결정합니다.

### 8-1. "골든셋"이란?

**골든셋(Golden Set)**은 AI가 참조하는 **모범 답안 모음**입니다.

크리딜은 현재 20개의 IM 제목/서브제목 골든셋을 내장하고 있습니다. 하지만 실제 현장 중개인이 작성한 **실전 예시**가 추가될수록 AI의 품질이 비약적으로 향상됩니다.

### 8-2. 골든셋 수집 요청 항목

아래 3가지 영역에서 **실제 현장에서 사용하는 표현**을 수집합니다.

#### 📝 A. IM 제목 골든셋

**질문**: "이 매물을 투자자에게 한 줄로 소개한다면?"

```markdown
## IM 제목 골든셋 제출

**자산유형**: [꼬마빌딩 / 오피스 / 상가 / 물류 / 호텔 / 토지 / 기타]
**권역**: [예: 강남, 여의도, 성수]

### 제목 (한 줄)
예: 역삼역 초역세권 프라임 꼬마빌딩 매각

### 서브제목 (투자 하이라이트 한 줄)
예: 연 수익률 5.2% · 공실 Zero · 장기 임차 기반 안정 수익형

### 핵심 투자 포인트 (1~2문장)
예: 역삼역 도보 3분 초역세권 대로변 코너 입지로, 전층 만실 운영 중이며...

### 💡 이 문체를 쓴 이유
예: 투자자가 Cap Rate와 공실률을 가장 먼저 확인하기 때문에 서브제목에 수치를 넣었습니다.
```

> 💡 **목표**: 자산 유형별 최소 3~5개씩 수집 → 현장감 있는 골든셋 50~100개 구축

---

#### 📝 B. 퓨샷(Few-shot) 예시 수집

**질문**: "이 상황에서 AI가 어떻게 쓰면 좋겠는가?"

"퓨샷"은 AI에게 "이렇게 써라"라고 보여주는 **예시 쌍(입력→출력)**입니다.

```markdown
## 퓨샷 예시 제출

**상황**: [어떤 매물에 대해 어떤 텍스트가 필요한지]

### 입력 (브로커가 제공하는 정보)
예: "역삼동 꼬마빌딩, 5층, 만실, 월세 2800만, 매각가 85억"

### 기대 출력 (AI가 이렇게 써줬으면 하는 결과)
예: "역삼역 도보 5분 초역세권에 위치한 5층 규모의 만실 꼬마빌딩입니다. 
연간 NOI 약 3.36억 원, Cap Rate 약 3.9%로 강남 핵심 상권 내 
안정 수익형 자산으로 분류됩니다."

### ❌ 이렇게 쓰면 안 되는 출력 (있다면)
예: "이 건물은 약 85억 원 수준으로 추정되며, 근린생활시설 또는 
다가구·상가주택 계열로 추정되는 자산입니다." (추정 남발, 어색)
```

아래 항목별로 수집합니다:

| # | 수집 항목 | 설명 |
|---|---------|------|
| B-1 | **투자 포인트 작성** | "이 매물의 핵심 매력을 1~2문장으로" |
| B-2 | **리스크 요약 작성** | "이 매물의 리스크를 투자자에게 어떻게 전달하는가?" |
| B-3 | **매물 한줄 소개** | "카카오톡으로 투자자에게 보내는 한 줄 소개" |
| B-4 | **매물 비교 문구** | "A물건 vs B물건 비교 시 어떤 표현을 쓰는가?" |
| B-5 | **가격 표현** | "55억 협의가능" vs "마케팅 가격 60억, 제안 가격 55억" 등 |

```markdown
## 용어사전 항목 제출

**용어**: 
**뜻 (설명)**: 
**사용 예시**: 
**유의어/관련어**: (있다면)

**용어**: 깡통빌딩

```

| 분야 | 수집 대상 |

- ✅ 📋 투자 요약 편집 (제목/서브제목/핵심 투자 포인트)
- ✅ 🖼️ OG 이미지 정규화 (1200×630) + 카카오 공유 수정
- ✅ 🃏 Vibe 명함 (프로필 카드) + FAQ
- ✅ 📝 CRE IM 제목 골든셋 20개 내장
- ✅ 🔧 IM 제목 문체 자동 정리
- ✅ 🔇 음성 브리핑 임시 제거 (TTS 안정화 대기)

### Phase 4 (계획) — 플랫폼화 + 베타 피드백 반영
- 📋 Full IM (18섹션) 자동 업그레이드
- 📋 투자자 매칭 시스템
- 📋 실사 체크리스트 자동 생성
- 📋 거래 파이프라인 관리
- 📋 **베타 테스터 골든셋/용어사전 반영** (50~100개 목표)
- 📋 **자산 유형별 맞춤 IM 템플릿**
- 📋 음성 브리핑 재도입 (안정화 후)

## 10. 부록: 용어 정리

| 용어 | 의미 |
|------|------|
| **IM** | Investment Memorandum, 투자설명서 |
| **SSoT** | Single Source of Truth, 건물 정보 단일 진실 원본 |
| **NOI** | Net Operating Income, 순영업이익 |
| **Cap Rate** | Capitalization Rate, 자본환원율 |
| **DCF** | Discounted Cash Flow, 할인현금흐름 |
| **WALE** | Weighted Average Lease Expiry, 가중평균 잔여 임대기간 |
| **WACC** | Weighted Average Cost of Capital, 가중평균자본비용 |
| **NPV** | Net Present Value, 순현재가치 |
| **OG 이미지** | Open Graph Image, SNS 공유 시 표시되는 미리보기 이미지 (1200×630) |
| **골든셋** | AI가 참조하는 모범 답안 모음 (Few-shot examples) |
| **퓨샷** | Few-shot, AI에게 보여주는 입력→출력 예시 쌍 |
| **서브제목** | 핵심 투자 하이라이트를 한 줄로 요약한 헤드카피 (v4.0) |
| **히어로 카드** | IM 최상단 핵심 투자 지표 요약 카드 |
| **Disclosure Guard** | 민감정보 자동 블라인드 처리 시스템 |
| **Hallucination Guard** | AI 환각 탐지 시스템 |
| **Provenance** | 데이터 출처 추적 시스템 |
| **Vibe 명함** | 중개인 디지털 프로필 카드 (디자인 템플릿 + FAQ) |
| **Data Quality** | A~D 등급 데이터 품질 지표 (편집자 전용) |
| **Readiness Score** | IM 생성 준비도 점수 |
| **GBD** | Gangnam Business District, 강남 업무지구 |
| **CBD** | Central Business District, 도심 업무지구 (광화문·을지로) |
| **YBD** | Yeouido Business District, 여의도 업무지구 |
| **S&LB** | Sale & Leaseback, 매각 후 재임차 |
| **Triple Net** | NNN 임대, 세금·보험·관리비 모두 임차인 부담 |
