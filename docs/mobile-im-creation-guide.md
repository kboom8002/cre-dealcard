## 1. 모바일 IM이란?

**모바일 IM(Investment Memorandum)**은 기존 30~50페이지의 투자설명서를 **7섹션 모바일 최적화 포맷**으로 자동 생성하는 크리딜의 핵심 기능입니다.

### 핵심 가치
- ⏱️ **작성 시간**: 기존 수일 → **5분 이내** 자동 생성
- 📱 **모바일 퍼스트**: 카카오톡/LINE으로 즉시 공유 가능
- 🔒 **정보 보호**: 민감 정보 자동 블라인드 처리
- 📊 **데이터 기반**: 건축물대장, 토지대장, 공시지가 등 공공데이터 자동 연동
- 🤖 **AI 분석**: GPT 기반 투자 분석 + 리스크 검토 + DCF 수익 분석

### 어떤 물건에 적합한가?
| 유형 | 적합도 | 설명 |
|------|--------|------|
| 꼬마빌딩 (30~200억) | ⭐⭐⭐⭐⭐ | 가장 최적화된 대상 |
| 상가 (10~50억) | ⭐⭐⭐⭐⭐ | 임대차 분석 강점 |
| 물류센터 | ⭐⭐⭐⭐ | 전용 물류 필드 지원 |
| 오피스빌딩 | ⭐⭐⭐⭐ | NOI/Cap Rate 분석 강점 |
| 토지/개발부지 | ⭐⭐⭐ | 기본 분석 가능 |

## 2. 작성 전 준비사항

### 필수 준비물
| 항목 | 설명 | 데이터 충실도 점수 영향 |
|------|------|----------------------|
| ✅ 매각 희망가 | 만원 단위 입력 | +15점 |
| ✅ 공실 현황 | 만실/~10%/~20% 등 선택 | +10점 |
| ✅ 건물 대표 사진 1장 이상 | 외관, 내부, 입지 등 | +10점 |
| ✅ 중개인 한줄 코멘트 | 물건의 핵심 매력 포인트 | +5점 |

### 선택 준비물 (높은 충실도 달성 시)
| 항목 | 설명 | 점수 영향 |
|------|------|---------|
| 선순위 대출 잔액 | 만원 단위 | +10점 |
| 층별 임대차 현황 | 층/업종/보증금/월세/계약기간 | +15점 |
| 보증금 합계 | 만원 단위 | +5점 |
| 관리비 합계 | 만원 단위 | +5점 |

> **💡 참고**: 수익률(Cap Rate)은 **월 임대료 × 12 ÷ 매각가**로 자동 계산됩니다. 별도 입력이 필요 없습니다. 매각가 입력란 옆의 "수익률로 역산" 버튼을 사용하면, 원하는 수익률을 입력하여 매각가를 역산할 수도 있습니다.

> **💡 팁**: 데이터 충실도 60점 이상이면 IM 자동 생성 가능. 80점 이상이면 고품질 IM 생성!

## 3. 모바일 IM 생성 Step-by-Step

### Step 1: 딜카드 상세 페이지 접근
```
메모함 → 딜카드 선택 → 상세 페이지
```

### Step 2: "🚀 프리미엄 투자설명서 만들기" 버튼 클릭
- 하단의 초록색 버튼을 탭합니다
- "투자설명서 데이터 보강" 바텀시트가 열립니다

### Step 3: 핵심 데이터 입력

#### 3-1. 매각 희망가 & 대출 잔액
- **매각 희망가**: 만원 단위로 입력 (예: 550000 = 55억)
- **선순위 대출 잔액**: 만원 단위 (예: 350000 = 35억)

#### 3-2. 공실 현황
4가지 중 선택:
- **만실**: 공실 없음
- **~10%**: 소규모 공실
- **~20%**: 중규모 공실
- **직접 입력**: 정확한 공실률 %

#### 3-3. 건물 대표 사진
- 최대 **12장** 업로드 가능
- 각 사진에 설명(캡션) 추가 가능
- 권장: 외관 정면, 1층 입구, 주요 임차인, 주차장, 옥상

#### 3-4. 중개인 한줄 코멘트
- 물건의 핵심 매력을 한 문장으로 작성
- 예: "상업지역, 리모델링 여지가 큽니다"

### Step 4: 데이터 충실도 확인
- 하단에 **데이터 충실도 점수** (0~100점)가 표시됩니다
- ✅ 표시: "투자설명서 직성 가능" (60점 이상)
- ⚠️ 표시: "데이터 보강 필요" (60점 미만)

### Step 5: IM 생성 시작
- "🚀 프리미엄 투자설명서 만들기" 버튼 클릭
- AI 분석이 시작됩니다 (~30~60초 소요)
- 진행 상황이 실시간으로 표시됩니다

### Step 6: IM 승인 및 발행
- 생성 완료 후 **승인 페이지**로 자동 이동
- 7섹션 내용을 검토합니다
- "승인 및 발행" 또는 "수정 요청" 선택

## 4. 7섹션 구성과 각 섹션 설명

크리딜 모바일 IM은 **매수자가 실제로 묻는 7가지 핵심 질문**에 답하는 구조입니다:

### 섹션 1: 🏢 이 건물, 어떤 자산인가?
**대응 질문**: "이 건물이 뭔가요?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| 자산 유형, 권역, 규모 | 브로커 메모 + AI 분석 |
| 건축물대장 요약 | 국토부 API 자동 연동 |
| 대지면적, 연면적, 용적률, 건폐율 | 건축물대장 API |
| 준공연도, 구조, 주용도 | 건축물대장 API |
| 승강기, 주차대수 | 건축물대장 총괄표제부 |

### 섹션 2: 📍 입지와 접근성은?
**대응 질문**: "위치가 어딘가요? 교통은?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| 권역 분석 | AI 분석 |
| 최근접 지하철역/거리 | 카카오 POI API |
| 주변 편의시설 (카페, 음식점 등) | 카카오 POI API |
| 용도지역/용도지구 | 토지이음 API |
| 공시지가 | 공시지가 API |
| 위치 지도 | OpenStreetMap |

### 섹션 3: 💰 현재 임대차 현황은?
**대응 질문**: "임대 수입이 얼마나 되나요?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| 층별 임대차 현황 (Rent Roll) | 브로커 직접 입력 |
| WALE (가중평균 잔여 임대기간) | 자동 계산 |
| 공실률 분석 | 브로커 입력 + AI |
| 권역 벤치마크 비교 | 벤치마크 엔진 |

### 섹션 4: 📊 수익 구조 분석
**대응 질문**: "얼마를 벌 수 있나요?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| NOI (순영업이익) | 자동 계산 |
| Cap Rate (기본/스트레스) | 자동 계산 |
| 레버리지 수익률 | 자동 계산 |
| 10년 DCF 분석 | DCF 엔진 |
| WACC (가중평균자본비용) | 자동 계산 |
| 자기자본 소요액 | 자동 계산 |

### 섹션 5: ⚠️ 숨은 리스크는 없는가?
**대응 질문**: "뭘 조심해야 하나요?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| 리스크 요인 분석 | AI 분석 |
| 실사 체크리스트 | AI + 템플릿 |
| 등기/건축물대장 교차검증 | 자동 교차검증 |

### 섹션 6: 🎯 왜 지금 이 매물을 사야 하는가?
**대응 질문**: "투자 가치가 있나요?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| 투자 가설 (Investment Thesis) | AI 분석 |
| 매수자 적합성 판단 | AI 분석 |
| Value-Add 시나리오 | Value-Add 엔진 |

### 섹션 7: 🔧 검토 후 다음 단계는?
**대응 질문**: "관심 있으면 어떻게 하나요?"
| 포함 정보 | 데이터 출처 |
|-----------|-----------|
| 다음 절차 안내 | 템플릿 |
| 중개인 연락처 | 프로필 연동 |
| 프라이빗 IM 신청 CTA | 신청 시스템 |

## 5. 히어로 카드 이해하기

IM 최상단에 표시되는 **히어로 카드**는 투자 핵심 지표를 한눈에 보여줍니다:

```
┌─────────────────────────────────────┐
│  📊 자산유형        📍 권역 신호     │
│  💰 매각가          📈 Cap Rate     │
│  🔑 핵심 투자 포인트                 │
│  ⚠️ 핵심 리스크                     │
│  ████████████░░░ 투자 검토 가능     │
│        데이터 충실도 85점            │
└─────────────────────────────────────┘
```

| 지표 | 의미 | 산출 근거 |
|------|------|---------|
| Cap Rate | 기본 수익률 | NOI ÷ 매각가 |
| NOI | 순영업이익 | (월세×12) - 관리비 |
| 자기자본 소요 | 실제 투자금 | 매각가 - 대출 - 보증금 |
| 레버리지 수익률 | 실질 수익률 | NOI ÷ 자기자본 |
| DCF NPV | 10년 순현재가치 | 할인현금흐름 분석 |
| 데이터 충실도 | IM 완성도 | 입력 데이터 기반 |

## 6. 보강 데이터 입력 가이드

### 층별 임대차 데이터 (고품질 IM의 핵심)

층별 데이터를 입력하면 **자동으로 계산**되는 항목:
- ✅ Rent Roll 테이블 자동 생성
- ✅ WALE (가중평균 잔여 임대기간) 자동 계산
- ✅ 공실률 정밀 계산
- ✅ 층별 단가 비교
- ✅ 임대차 만기 분산도 분석

입력 필드:
| 필드 | 필수 | 예시 |
|------|------|------|
| 층 | ✅ | "1F", "2F", "B1" |
| 업종/업체명 | - | "약국", "이비인후과" |
| 전용면적 (평) | - | 30 |
| 보증금 (만원) | - | 5000 |
| 월 임대료 (만원) | - | 300 |
| 관리비 (만원) | - | 50 |
| 계약 시작일 | - | 2024-01-01 |
| 계약 종료일 | - | 2026-12-31 |
| 공실 여부 | - | 체크 |

### 물류센터 전용 필드
물류 물건의 경우 추가 입력 가능:
- 천장고 (m), 도크 수, 도크 레벨러
- 바닥 하중, 냉동/냉장 면적
- 스프링클러, 내화등급
- 최근접 IC 거리

## 7. IM 뷰어 기능 총정리

### 모바일 뷰어 UI
- **아코디언 네비게이션**: 7섹션을 펼치기/접기로 탐색
- **상단 진행바**: 섹션별 슬라이딩 도트 인디케이터
- **맵 통합**: 위치 지도 + 카카오맵 연동 버튼
- **사진 갤러리**: 대표 사진 슬라이더 + 라이트박스
- **한국어/영문 전환**: 영문 1-Pager 자동 번역

### 공유 버튼 (하단 고정)
| 버튼 | 기능 |
|------|------|
| 🟡 카카오톡 | 카카오톡으로 IM 링크 공유 |
| 🟢 LINE | LINE으로 공유 |
| 📋 링크복사 | IM URL 클립보드 복사 |
| 📄 PDF 저장 | PDF 변환 후 다운로드 |

### 담당 중개인 카드 (Vibe 명함)
- IM 하단에 중개인의 **Vibe 명함**이 자동 표시
- 프로필 사진, 전문 분야, 최근 딜카드 수
- 매거진 연동 (주간 시장 리포트 링크)

## 8. 공유 및 배포

### 공유 방법
1. **카카오톡 공유**: 가장 많이 사용. 썸네일 + 요약 + 링크 자동 생성
2. **LINE 공유**: 일본/해외 투자자 대상
3. **링크 복사**: 이메일, SMS, 기타 메신저
4. **PDF 저장**: 오프라인 제출용

### 공유 URL 구조
```
https://credeal.net/im-lite/{buildingId}
```
- 블라인드 처리된 공개 URL (정확한 주소 미노출)
- 비로그인 사용자도 열람 가능
- 모바일 최적화

## 9. 프라이빗 IM 신청 기능

IM 하단의 "📄 프라이빗 투자설명서(IM) 신청" 버튼을 통해:

1. 투자자가 **이름, 연락처, 이메일, 메시지**를 입력
2. 접수 즉시 담당 중개인에게 알림 전송
3. 중개인이 프라이빗(상세) IM을 별도 제공

> 이 기능을 통해 **리드 수집 + 관심 투자자 파악**이 가능합니다.

## 10. IM 승인 및 발행 프로세스

IM 생성 후 자동으로 **승인 페이지**(`/broker/im-approval/[id]`)로 이동합니다:

### 승인 페이지 기능
| 기능 | 설명 |
|------|------|
| **섹션별 검토** | 각 섹션의 신뢰도 배지 확인 |
| **인라인 편집** | 마크다운 에디터 + 실시간 미리보기 |
| **섹션별 확인** | "미확인" ↔ "✓ 확인완료" 토글 |
| **제목 편집** | 상단 제목 클릭하여 수정 |
| **사진 미리보기** | 업로드된 사진 가로 스크롤 |
| **자동 저장** | 수정 시 자동 저장 |

### 신뢰도 배지
| 배지 | 의미 |
|------|------|
| 🟢 `공공데이터 확인됨` | 건축물대장 등 검증된 데이터 기반 |
| 🟡 `AI 추론 — 검토 권장` | AI가 추론한 내용, 전문가 검토 필요 |
| 🔴 `보완 필요` | 추가 데이터 입력 권장 |

### 발행 조건
- **모든 섹션이 "✓ 확인완료"** 상태여야 "🚀 승인 및 공개" 가능
- 발행 후 공개 URL이 생성되어 즉시 공유 가능

## 11. 고급 뷰어 기능

### 🎧 음성 브리핑 (Voice Briefing)
- IM 상단의 재생 버튼으로 **TTS 음성 브리핑** 청취 가능
- AI가 핵심 내용을 요약하여 음성으로 전달
- 이동 중이거나 화면을 볼 수 없을 때 유용

### 📊 DCF 히트맵 (민감도 분석)
| | Exit Cap 3.5% | Exit Cap 4.0% | Exit Cap 4.5% |
|---|---|---|---|
| **WACC 6%** | NPV / IRR | NPV / IRR | NPV / IRR |
| **WACC 7%** | NPV / IRR | NPV / IRR | NPV / IRR |
| **WACC 8%** | NPV / IRR | NPV / IRR | NPV / IRR |

- 3×3 격자에 WACC × Exit Cap Rate 조합별 NPV/IRR 표시
- 색상 코드로 투자 매력도 직관적 파악

### 📈 레버리지 차트
- SVG 도넛 차트로 **자기자본 / 보증금 / 대출** 비중 시각화
- 투자자가 필요한 실제 투입 금액을 한눈에 파악

### 📊 투자자 열람 분석
- 어떤 투자자가 IM을 열었는지 추적
- 어떤 섹션을 얼마나 오래 봤는지 분석
- **60초 이상 체류 시** 브로커에게 알림 전송
- Hot Lead 자동 판별

## 12. AI 엔진 작동 원리

### AI 파이프라인 흐름
```
브로커 메모
    ↓
[1] MemoParser (AI) — 자유 텍스트 → 구조화 데이터
    ↓
[2] BuildingMiniTruth (AI) — SSoT Lite 생성
    ↓
[3] 외부 데이터 연동 — 건축물대장/토지/공시지가/POI
    ↓
[4] 데이터 충실도 계산 — 60점 이상이면 IM 생성 가능
    ↓
[5] 7섹션 AI 내러티브 생성 — 각 섹션별 맥락형 분석
    ↓
[6] 가드레일 적용 — Risk Boundary + Disclosure Guard + Hallucination Guard
    ↓
[7] 교차 검증 — 금액/면적 수치 정합성 자동 검증
    ↓
[8] 품질 게이트 — CRE 전문 기준 품질 점검
    ↓
[9] RAG 인덱싱 — 벡터 DB에 IM 임베딩 저장 (향후 검색용)
    ↓
[10] 발행
```

### 가드레일 3중 방어
| 가드레일 | 역할 |
|---------|------|
| **Risk Boundary** | 확정적 투자 수익 약속 방지 |
| **Disclosure Guard** | 민감 정보 (정확한 주소, 소유자, 임차인명) 자동 블라인드 |
| **Hallucination Guard** | 금액/면적 수치 비정상 탐지 (20배 이상 편차 차단) |

### 데이터 출처 추적 (Provenance)
모든 데이터 포인트에 출처 배지가 표시됩니다:
- 🟢 **공공데이터**: 건축물대장, 토지대장 등 검증된 출처
- 🔵 **브로커 입력**: 중개인이 직접 입력한 정보
- 🟡 **AI 추론**: AI가 분석/추론한 정보
- ⚪ **미검증**: 확인이 필요한 정보

### AI 모델 구성
| 단계 | 모델 | 역할 |
|------|------|------|
| MemoParser | GPT-5.4 | 자유 텍스트 → 구조화 |
| BuildingMiniTruth | GPT-5.4 | SSoT 초안 생성 |
| 내러티브 작성 | GPT-5.4 | 7섹션 분석문 생성 |
| 번역 | GPT-5.4 | 한→영/중/일 번역 |
| 음성 브리핑 | TTS API | 텍스트→음성 변환 |

## 14. Rent Roll 대량 입력 (Rent Roll Importer)

### 엑셀/텍스트에서 임대차 데이터 한번에 입력
1. "보강 데이터" 바텀시트에서 **Rent Roll Importer** 버튼 클릭
2. 엑셀에서 복사한 데이터를 붙여넣기 (탭/쉼표 구분)
3. 자동으로 파싱되어 다음 필드 자동 채움:
   - 월세 합계
   - 보증금 합계
   - 관리비 합계
   - 공실률
   - 층별 임대차 데이터

> **💡 팁**: 5층 이상 건물의 경우 Rent Roll Importer를 사용하면 입력 시간을 **80% 이상** 절약할 수 있습니다.

## 15. 자주 묻는 질문

### Q: IM 생성에 얼마나 걸리나요?
A: 데이터 충실도와 서버 상태에 따라 **30초~60초**가 소요됩니다.

### Q: 생성된 IM을 수정할 수 있나요?
A: **승인 페이지에서 섹션별 인라인 편집**이 가능합니다. 마크다운 에디터로 직접 수정하고 실시간 미리보기를 확인하세요.

### Q: 정확한 주소가 노출되나요?
A: 아닙니다. **Disclosure Guard**가 자동으로 정확한 주소, 건물명, 소유자명, 임차인 실명을 블라인드 처리합니다.

### Q: 여러 물건의 IM을 동시에 만들 수 있나요?
A: 딜카드별로 1개씩 생성합니다. IM 보관함에서 모든 생성된 IM을 관리할 수 있습니다.

### Q: 영문 IM도 지원하나요?
A: 네. IM 뷰어에서 "English" 탭을 누르면 **AI 자동 번역**된 영문 버전을 볼 수 있습니다. 중국어(zh), 일본어(ja)도 지원합니다.

### Q: 투자자가 프라이빗 IM을 신청하면?
A: 신청자의 이름, 연락처, 메시지가 담당 중개인에게 전달됩니다. 추후 카카오 알림톡 연동 예정입니다.

### Q: 투자자가 얼마나 관심을 보이는지 알 수 있나요?
A: 네. **열람 분석** 기능을 통해 어떤 투자자가 얼마나 오래 IM을 봤는지, 어떤 섹션에 관심을 보이는지 추적할 수 있습니다. 60초 이상 체류 시 알림이 전송됩니다.

### Q: PDF로 출력할 수 있나요?
A: 네. IM 뷰어 하단의 PDF 버튼을 눌러 A4 포맷의 인쇄용 PDF를 생성할 수 있습니다.
