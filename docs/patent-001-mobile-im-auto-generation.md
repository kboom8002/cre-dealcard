# 특허명세서 제1호

## 발명의 명칭

**비정형 중개 메시지로부터 모바일 네이티브 투자설명서를 자동 생성하고 메신저 기반으로 유통하는 시스템 및 방법**

System and Method for Automatically Generating Mobile-Native Investment Memoranda from Unstructured Brokerage Messages and Distributing via Messenger Platforms

---

## 기술분야

본 발명은 상업용 부동산(Commercial Real Estate, CRE) 거래 정보처리 시스템에 관한 것으로, 보다 상세하게는 중개인이 메신저 앱(카카오톡, WhatsApp 등)에서 자유 형식으로 작성한 비정형 매물 메모를 자연어처리(NLP) 및 대규모 언어모델(LLM)을 통해 자동으로 구조화하고, 이를 모바일 단말에 최적화된 스와이프 카드 형식의 투자설명서(Information Memorandum, IM)로 변환하여, 메신저 플랫폼을 통해 바이럴 유통할 수 있도록 하는 시스템 및 방법에 관한 것이다.

---

## 배경기술

### 종래 기술의 문제점

상업용 부동산 거래에서 투자설명서(IM 또는 Offering Memorandum, OM)는 매수 의사결정의 핵심 문서이다. 종래의 IM 작성 및 유통 과정은 다음과 같은 한계를 갖는다.

**① 수동 작성의 비효율성**: 기존 IM은 30~80페이지 분량의 PDF 문서로, 전문 인력이 2~4주에 걸쳐 수동으로 작성한다. 이 과정에서 건축물대장, 등기부등본, 임대차계약서, 시장 데이터 등을 개별적으로 수집·정리해야 하며, 딜 초기 단계에서 신속한 정보 공유가 불가능하다.

**② 데스크탑 PDF 중심의 열람 환경**: 종래의 IM은 PDF 파일로 이메일에 첨부되어 유통된다. 이는 모바일 환경에서의 열람이 불편하고, 투자자가 이동 중에 딜 정보를 검토하기 어렵다. 특히 아시아 시장에서는 메신저 앱(카카오톡, 위챗 등)이 비즈니스 커뮤니케이션의 주요 채널임에도 불구하고, 기존 IM 포맷은 이러한 채널에 적합하지 않다.

**③ 비정형 입력 대응 불가**: 중개인은 현장에서 카카오톡 등을 통해 자유 형식의 메모("성수 꼬마빌딩 80억, 1층 카페 월 400만원, 2~4층 공실, 엘베 없음, 급매 가능")를 주고받는다. 종래의 IM 자동화 시스템(Dealpath AI Extract, Crexi Create 등)은 **이미 작성된 PDF OM에서 데이터를 추출(Extraction)**하는 기능만 제공하며, 비정형 메모에서 **IM을 처음부터 생성(Generation)**하는 기능은 존재하지 않는다.

**④ 민감정보 보호 부재**: 비정형 메모에는 정확한 주소, 임차인 실명, 개별 임대료, 매도자 사정 등 비공개 정보가 혼재되어 있으나, 이를 자동으로 탐지·분류·차폐(redaction)하는 시스템이 없다.

따라서, 중개인의 비정형 메신저 메시지로부터 민감정보를 자동 탐지·보호하면서 모바일에 최적화된 투자설명서를 실시간으로 생성하고, 메신저 플랫폼을 통해 원클릭으로 유통할 수 있는 새로운 시스템이 요구된다.

---

## 발명의 목적

본 발명의 목적은 다음과 같다.

1. 중개인이 메신저 앱에서 자유 형식으로 작성한 비정형 매물 메모를 입력받아, 구조화된 부동산 데이터 객체(Single Source of Truth, SSoT)로 자동 변환하는 기술을 제공한다.
2. 상기 구조화된 데이터 객체로부터 모바일 단말에 최적화된 스와이프 카드 형식의 복수 섹션 투자설명서를 자동 생성하는 기술을 제공한다.
3. 비정형 입력에 포함된 민감 정보(정확한 주소, 임차인 실명, 개별 임대료, 매도자 사정 등)를 자동 탐지하고, 출력 가시성 수준에 따라 단계적으로 차폐(Progressive Disclosure)하는 기술을 제공한다.
4. 생성된 모바일 투자설명서를 메신저 플랫폼(카카오톡, SMS 등)을 통해 리치 프리뷰 카드 형태로 원클릭 공유할 수 있는 유통 기술을 제공한다.
5. 열람자의 행동 데이터(섹션별 체류 시간, 관심 표명, 공유 이력)를 수집·분석하여 중개인에게 딜 인사이트를 제공하는 기술을 제공한다.

### 1. 입력 수신 모듈 (110)

입력 수신 모듈(110)은 중개인의 비정형 텍스트 입력을 수신한다. 상기 입력은 다음 중 하나 이상의 형태를 포함한다:
- (a) 메신저 앱(카카오톡, WhatsApp 등)에서 작성된 자유 형식의 매물 메모
- (b) 음성 녹음을 텍스트로 변환한 전사(transcription) 텍스트
- (c) 수동 입력 폼을 통한 구조화된 입력
- (d) 주소(도로명 또는 지번)만의 최소 입력

상기 입력 수신 모듈(110)은 입력 유형(input_type)을 자동 분류하여 후속 처리 파이프라인을 결정한다.

### 2. 메모 파서 모듈 (120)

메모 파서 모듈(120)은 대규모 언어모델(LLM)을 이용하여 비정형 텍스트를 구조화된 데이터 객체로 변환한다. 상기 파서는 다음 필드를 추출한다:

- **추출 사실(extractedFacts)**: 권역(region), 주소 후보(exactAddressCandidate), 자산 유형(assetType), 가격 텍스트(priceText), 규모 텍스트(sizeText), 현재 용도(currentUse), 임대 신호(leaseSignal), 공실 신호(vacancySignal), 임차인 실명(tenantNames), 개별 임대료 텍스트(unitRentTexts), 매도자 사정(sellerMotivationText), 중개사 메모(brokerNotes)
- **탐지된 민감 필드(detectedSensitiveFields)**: exact_address, tenant_name, unit_rent, seller_motivation, negotiation_memo, owner_identity, buyer_identity 중 해당 항목
- **모호한 필드(ambiguousFields)**: 추정 또는 검증 불가능한 정보
- **경고 사항(warnings)**: 법적·윤리적 주의사항

상기 메모 파서 모듈(120)은 **검증되지 않은 메모를 확정된 사실로 변환하지 않는 안전 규칙**을 내장하며, 이는 프롬프트 레벨의 제어로 구현된다.

### 3. SSoT 생성기 모듈 (130)

SSoT(Single Source of Truth) 생성기 모듈(130)은 파싱된 데이터를 표준화된 건물 데이터 레코드(building_ssot_lite)로 변환한다. 상기 레코드는 다음을 포함한다:

- 권역 신호(area_signal): 정확한 주소 대신 권역 수준의 위치 정보
- 자산 유형(asset_type): 표준화된 분류 체계에 따른 자산 유형
- 가격대(price_band): 범위 형태의 가격 정보
- 규모 신호(size_signal): 대략적 규모 표현
- 적합성 요약(fit_summary): AI가 생성한 매수자 관점 검토 가설
- 주의 사항 요약(caution_summary): 확인 필요 사항
- 숨김 필드(hidden_fields): 공개 불가 민감 정보 목록
- 신뢰도 맵(confidence): 각 필드의 출처 및 신뢰 수준

**여기서 핵심적인 기술적 특징은**, 각 데이터 포인트에 신뢰 수준(confidence level)을 부여하는 것이다. 상기 신뢰 수준은 다음 중 하나이다:
- `confirmed`: 공공 데이터 또는 증빙으로 확인됨
- `user_provided`: 사용자(중개인)가 직접 제공
- `public_data_inferred`: 공공 데이터에서 추론됨
- `ai_hypothesis`: AI가 생성한 가설
- `needs_verification`: 검증 필요
- `unknown`: 알 수 없음

### 4. 민감정보 탐지기 모듈 (140)

민감정보 탐지기 모듈(140)은 **이중 탐지 아키텍처**를 채택한다:

**(a) 1차 탐지 — LLM 기반 의미 분석**: 메모 파서 모듈(120)에서 LLM이 의미적으로 민감 정보를 분류한다.

**(b) 2차 탐지 — 규칙 기반 패턴 매칭**: 생성된 출력 텍스트에 대해 정규표현식 기반의 패턴 매칭을 수행한다. 상기 패턴은 다음을 포함한다:
- 정확한 주소 패턴: `[가-힣]+구\s+[가-힣]+동\s*\d+[-\d]*`
- 임차인 실명 패턴: 브랜드명 데이터베이스 매칭
- 개별 임대료 패턴: `월세\s*\d+만\s*원`, `보증금\s*\d+억`
- 매도자 사정 패턴: `상속\s*문제`, `급매`, `이혼\s*매물`
- 협상 내용 패턴: `\d+억까지\s*가능`, `네고\s*가능`

민감 정보가 탐지되면, **출력 유형(output_type)과 게이트 수준(gate_level)에 따라 차폐 수준이 결정된다**:
- G0~G3 (공개~제한 공개): 민감 정보를 일반화된 표현으로 대체 (예: "성수동2가 123-4" → "[성수권역]")
- G4~G5 (인가된 열람): 원본 정보 노출 허용

### 5. 공개정보 보강기 모듈 (145)

공개정보 보강기 모듈(145)은 입력된 주소 정보를 기반으로 공공 데이터 API를 자동 조회하여 SSoT 레코드를 보강한다. 보강 대상은 다음을 포함한다:
- 건축물대장 데이터(연면적, 대지면적, 건축연도, 구조, 층수)
- 개별공시지가(원/㎡, 기준년도)
- 토지이용계획(용도지역, 건폐율, 용적률)
- 주변 실거래 데이터(최근 거래가, 면적, 거래일)
- 근접 교통 인프라(최근접 지하철역, 거리)

보강된 데이터 포인트에는 `confidence: "public_data_confirmed"` 태그가 부여되어, AI 추론과 확인된 사실을 명확히 구분한다.

### 6. 모바일 IM 생성기 모듈 (150)

모바일 IM 생성기 모듈(150)은 SSoT 레코드와 보강 데이터를 기반으로 **N개의 섹션(바람직하게는 7개 섹션)**으로 구성된 모바일 최적화 투자설명서를 생성한다. 상기 7개 섹션은 다음과 같다:

| 섹션 순서 | 섹션 유형 | 제목 | 내용 구조 |
|---------|----------|------|----------|
| 1 | property_overview | 물건 개요 | 자산유형/위치/규모/가격대/현재상태 |
| 2 | location_access | 입지·상권 | 역세권 분석/도보 접근성/주변 인프라 |
| 3 | lease_status | 임대 현황 | 임대율/월 수입/주요 임차 업종/공실 |
| 4 | income_analysis | 수익 분석 | 추정 수익률/가정 사항/주의점 |
| 5 | risk_check | 확인 필요 사항 | 주요 리스크/DD 체크 항목 |
| 6 | investment_thesis | 투자 포인트 | 딜 포인트/매수자 적합성/가치상승 가설 |
| 7 | next_steps | 다음 단계 | 자료 보강 사항/전문가 추천/Full IM 안내 |

각 섹션에는 다음이 포함된다:
- 마크다운 형식의 본문(markdown)
- 신뢰도 표시(confidence)
- 면책 문구(boundary_note)
- 대응하는 Full IM 18섹션 참조(full_im_source)

**모바일 표현 형식의 특징**:
- 화면을 좌우 스와이프하여 섹션을 탐색하는 **카드 네비게이션 인터페이스**
- 진행률 표시 바(progress bar) 및 도트 인디케이터
- 헤더 영역에 핵심 지표(KPI) 3개를 글라스모피즘 카드로 표시
- 섹션별 색상 그래디언트 및 아이콘 코드 자동 매핑
- **Full IM 업그레이드 CTA**: 준비도 점수(readiness_score), 부족 데이터 목록, 업그레이드 버튼을 표시하여 18섹션 상세 IM으로의 전환을 유도

### 7. 메신저 유통 엔진 모듈 (160)

메신저 유통 엔진 모듈(160)은 생성된 모바일 IM을 메신저 플랫폼을 통해 유통할 수 있도록 다음 기능을 제공한다:

**(a) 공유 URL 생성**: 고유 슬러그(slug) 기반의 공개 접근 가능 URL을 생성한다 (예: `https://js.im/m/{slug}`)

**(b) 카카오톡 공유용 텍스트 자동 생성**: 핵심 지표와 URL을 포함한 메신저 친화적 텍스트를 자동 구성한다:
```
📋 성수권역 꼬마빌딩 70~80억 | 투자 메모
✅ 수익률(추정): ~4.5% | 임대율: 85%
⚠️ 확인 필요: 임대차 만기, 주차 조건
👉 상세 보기: https://js.im/m/seongsu-001
※ 본 자료는 예비 검토용이며 투자 권유가 아닙니다.
```

**(c) OG 메타 리치 프리뷰**: 메신저에서 URL을 공유할 때 건물 사진, 권역명, 핵심 지표가 포함된 시각적 프리뷰 카드가 자동 표시되도록 Open Graph 메타데이터를 동적으로 생성한다.

**(d) 원클릭 클립보드 복사**: 사용자가 "카톡 공유" 버튼을 누르면 (b)의 텍스트가 즉시 클립보드에 복사된다.

### 8. 열람 분석 엔진 모듈 (170)

열람 분석 엔진 모듈(170)은 모바일 IM 열람자의 행동 데이터를 수집·분석하여 중개인에게 딜 인사이트를 제공한다:

- 총 열람 수 및 고유 열람자 수
- 섹션별 체류 시간(dwell_time_seconds)
- 카카오톡 공유 횟수
- Full IM 요청 CTA 클릭 여부
- 관심 표명 여부 및 시점
- 열람 단말 유형(모바일/태블릿/데스크탑)

### 9. 준비도 점수 및 풀IM 전환 모듈 (180)

준비도 점수 모듈(180)은 모바일 IM(7섹션)에서 풀 IM(18섹션)으로의 업그레이드 가능성을 정량 평가한다. 상기 평가는 다음 데이터 포인트의 가중 합산으로 계산된다:

| 데이터 포인트 | 가중치 | 판정 기준 |
|-------------|-------|----------|
| 권역 정보 | 15점 | asset_identity.area_signal 존재 |
| 가격대 | 15점 | asset_identity.price_band 존재 |
| 자산 유형 | 15점 | asset_identity.asset_type 존재 |
| 월세 총액 | 20점 | 보충 입력으로 제공 |
| 공실 현황 | 10점 | 보충 입력 또는 SSoT에서 추출 |
| 건물 사진 | 15점 | 사진 URL 1개 이상 |
| 상세 위치 | 10점 | 위치 분석 데이터 존재 |

**임계값**: 40점 이상 시 모바일 IM 생성 가능, 71점 이상 시 풀 IM 초안 생성 가능.

부족 데이터 목록은 모바일 IM 하단의 "Full IM 준비도" 패널에 표시되어, 중개인이 어떤 자료를 추가하면 풀 IM으로 업그레이드할 수 있는지 안내한다.

## 발명의 효과

1. **딜 정보 유통 속도 향상**: 종래 2~4주 소요되던 IM 작성 과정을 **수 초~수 분**으로 단축하여, 딜 초기 단계에서 즉시 매수자에게 정보를 제공할 수 있다.

2. **모바일 환경 최적화**: 투자자가 이동 중에도 스마트폰에서 최적화된 인터페이스로 딜 정보를 검토할 수 있어, 의사결정 속도가 향상된다.

3. **민감정보 자동 보호**: 이중 탐지 아키텍처(LLM + 규칙 기반)를 통해 정확한 주소, 임차인 실명 등의 민감 정보가 공개 출력에 포함되는 것을 방지하여, 정보 보안 사고를 예방한다.

4. **메신저 바이럴 유통**: 카카오톡 등 메신저를 통한 원클릭 공유로 딜 정보의 도달 범위가 확대되며, 리치 프리뷰 카드를 통해 클릭률이 향상된다.

5. **데이터 기반 딜 인사이트**: 열람 행동 분석을 통해 중개인이 매수자의 관심도를 정량적으로 파악하여, 후속 영업 활동의 효율성이 향상된다.

6. **풀 IM 전환 퍼널 구축**: 모바일 IM을 풀 IM(18섹션)의 진입점으로 활용하여, 단계적으로 자료를 보강하면서 거래를 진행할 수 있는 워크플로를 제공한다.

## 도면의 간단한 설명

- **도 1**: 시스템 전체 구성도 (입력 수신 → 메모 파싱 → SSoT 생성 → 민감정보 탐지 → 공개정보 보강 → 모바일 IM 생성 → 메신저 유통 → 열람 분석)
- **도 2**: 비정형 메모 파싱 프로세스 흐름도 (LLM 기반 구조화 추출 상세)
- **도 3**: 민감정보 이중 탐지 아키텍처 (1차 의미 분석 + 2차 패턴 매칭)
- **도 4**: 모바일 IM 스와이프 카드 UI 구성도 (헤더/KPI 카드/섹션 네비게이터/콘텐츠 카드/면책/Full IM CTA)
- **도 5**: 준비도 점수 산출 및 Full IM 전환 퍼널 플로우
- **도 6**: 메신저 공유 시 OG 리치 프리뷰 카드 생성 프로세스
- **도 7**: 열람자 행동 분석 데이터 흐름도

## 발명의 실시예

### 실시예 1: 브로커 카카오톡 메모 → 모바일 IM 생성

1. 중개인 A가 카카오톡에서 "성수 꼬마빌딩 80억, 1층 카페 월 400, 2~4층 공실, 엘베 없음, 2018년 리모델링, 급매 가능 협의"라고 메모를 입력한다.
2. 메모 파서(120)가 LLM을 통해 `{ region: "성수", assetType: "꼬마빌딩", priceText: "80억", tenantNames: ["카페"], unitRentTexts: ["월 400"], sellerMotivationText: "급매 가능" }`을 추출한다.
3. 민감정보 탐지기(140)가 `detectedSensitiveFields: ["tenant_name", "unit_rent", "seller_motivation"]`을 분류한다.
4. SSoT 생성기(130)가 `area_signal: "성수권역", price_band: "70~80억", hidden_fields: ["tenant_name", "unit_rent", "seller_motivation"]`을 포함하는 building_ssot_lite 레코드를 생성한다.
5. 공개정보 보강기(145)가 건축물대장 API를 조회하여 `연면적: 926.4㎡, 건축연도: 2018, 구조: RC, 4층`을 보강한다.
6. 모바일 IM 생성기(150)가 7개 섹션의 스와이프 카드 투자설명서를 생성하되, "카페"는 "1층 F&B 앵커 테넌트"로, "월 400"은 "[임대수익 존재, 상세 내용 비공개]"로, "급매 가능"은 "[매도자 사정 비공개]"로 차폐한다.
7. 메신저 유통 엔진(160)이 `https://js.im/m/seongsu-001` URL과 카카오톡 공유 텍스트를 생성한다.
8. 열람 분석 엔진(170)이 투자자 B의 섹션별 체류 시간을 기록하고, "수익 분석 섹션 체류 45초, 입지·상권 섹션 체류 38초"의 인사이트를 중개인 A에게 제공한다.
