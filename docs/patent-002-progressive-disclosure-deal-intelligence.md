# 특허명세서 제2호

## 발명의 명칭

**부동산 투자문서의 단계적 정보공개 게이트 시스템 및 열람자 행동 기반 거래 인텔리전스 제공 방법**

Gate System for Progressive Information Disclosure in Real Estate Investment Documents and Method for Providing Transaction Intelligence Based on Viewer Behavior Analysis

---

## 기술분야

본 발명은 상업용 부동산(Commercial Real Estate, CRE) 거래에서 생성되는 투자문서의 정보 공개 수준을 거래 단계에 따라 자동 제어하고, 문서 열람자의 행동 패턴을 분석하여 거래 성사를 가속하는 인텔리전스를 제공하는 시스템 및 방법에 관한 것이다.

보다 상세하게는, 인공지능(AI) 기반 위험 표현 경계 검증(Risk Boundary Check), 민감 정보 공개 가드(Disclosure Guard), 다단계 게이트 리뷰(Multi-level Gate Review)를 결합한 삼중 안전 아키텍처를 통해 문서 출력의 정보 공개 수준을 자동 제어하고, 이렇게 제어된 문서의 열람자 행동 데이터를 수집·분석하여 매도 측 중개인에게 매수자 관심도 인텔리전스를 제공하며, 누적된 행동 데이터를 시장 수준의 딜 수요 인텔리전스로 집계하는 시스템 및 방법에 관한 것이다.

---

## 배경기술

### 종래 기술의 문제점

상업용 부동산 거래에서 정보 비대칭은 핵심 문제이다. 매도 측은 딜 정보를 널리 유통하여 매수자를 모집하고 싶지만, 민감한 정보(정확한 주소, 임차인 실명, 개별 임대료, 매도자 사정, 매도 가격 협상 여지)가 무분별하게 노출되면 거래에 부정적 영향이 발생한다.

**① 정보 공개 수준 제어의 부재**: 기존 투자문서(OM/IM)는 PDF 형태로 유통되며, 한 번 공유되면 수신자가 해당 문서의 모든 내용에 접근할 수 있다. 거래 단계(초기 관심 → NDA 서명 → 상세 실사 → 최종 매수)에 따라 정보 공개 범위를 조절하는 기술적 수단이 존재하지 않는다.

**② AI 생성 문서의 위험 표현 관리 한계**: AI를 이용하여 투자문서를 자동 생성할 때, "수익률 보장", "안전한 투자처", "대출 가능합니다" 등의 위험한 표현이 생성될 수 있으나, 이를 자동으로 탐지하고 안전한 표현으로 대체하는 체계적 시스템이 없다. 이러한 표현은 자본시장법 위반 또는 투자자 오인의 법적 리스크를 야기한다.

**③ 열람자 행동 정보의 미활용**: 종래의 PDF 기반 OM 유통에서는 매수자가 어느 부분에 관심을 가지는지, 얼마나 오래 검토하는지에 대한 행동 데이터를 수집할 수 없다. 이는 중개인이 경험과 직관에만 의존하여 후속 영업을 진행해야 함을 의미한다.

**④ 시장 수준의 수요 인사이트 부재**: 개별 딜 수준의 관심도를 넘어, "현재 시장에서 어느 권역/자산유형/수익률 구간이 가장 수요가 높은가?"에 대한 실시간 인사이트를 제공하는 시스템이 존재하지 않는다.

따라서, AI 생성 부동산 투자문서의 정보 공개를 거래 단계에 따라 자동 제어하고, 위험 표현을 체계적으로 관리하며, 열람 행동 데이터에서 거래 인텔리전스를 추출하는 통합 시스템이 요구된다.

---

## 발명의 목적

본 발명의 목적은 다음과 같다.

1. AI가 생성한 부동산 투자문서에 포함된 위험 표현(투자 추천, 수익률 보장, 대출 확약 등)을 자동 탐지하고, 위험 수준에 따라 차단(blocked) 또는 안전 표현 대체(revise)하는 기술을 제공한다.
2. 문서에 포함된 민감 정보(정확한 주소, 임차인 실명, 개별 임대료, 매도자 사정 등)를 자동 탐지하고, 거래 단계(Gate Level)에 따라 단계적으로 정보를 공개하는 기술을 제공한다.
3. 상기 위험 표현 검증과 민감 정보 공개 제어를 결합한 다단계 게이트 시스템을 통해, 문서가 각 거래 단계에 적합한 수준으로만 공개되도록 보장하는 기술을 제공한다.
4. 정보 공개가 제어된 문서의 열람자 행동 데이터(섹션별 체류 시간, 스와이프 패턴, 공유 이력, 관심 표명)를 수집·분석하여 매도 측 중개인에게 매수자별 관심도 인텔리전스를 제공하는 기술을 제공한다.
5. 다수의 딜에서 수집된 열람 행동 데이터를 집계하여, 시장 수준의 수요 트렌드 인텔리전스를 제공하는 기술을 제공한다.

### 1. 위험 경계 검증 모듈 (RiskBoundary) (210)

위험 경계 검증 모듈(210)은 AI가 생성한 투자문서의 텍스트를 검사하여, 법적·윤리적으로 위험한 표현을 자동 탐지하고 안전한 표현으로 대체한다.

상기 모듈은 **금지 패턴 테이블(Forbidden Pattern Table)**을 포함하며, 각 패턴은 다음 속성을 갖는다:
- 패턴(pattern): 정규표현식으로 정의된 위험 표현
- 이슈 유형(issue_type): 위험 카테고리
- 심각도(severity): P0(최고위험), high, medium, low
- 메시지(message): 위험 설명
- 권장 대체 텍스트(recommended_text): 안전한 표현

#### 금지 패턴 테이블 (핵심 구성)

| 심각도 | 이슈 유형 | 금지 패턴 (예시) | 안전 대체 표현 |
|-------|----------|----------------|-------------|
| **P0** | investment_recommendation | 매수 추천, 투자 가치가 높, 안전한 투자처, 강력히 추천 | "투자 적합 여부는 별도 실사와 전문가 검토를 통해 판단해야 합니다." |
| **P0** | financial_certainty | 수익률 보장, NOI 확정, 현금흐름 보장, Cap Rate 안정 | "수익률은 가정과 실사 결과에 따라 달라질 수 있습니다." |
| **P0** | loan_certainty | 대출 가능합니다, LTV N% 가능, 금리 확정, DSCR 충분, 대출 승인 | "대출 가능성과 조건은 금융기관 심사, 담보평가, 차주 신용도, 시장금리에 따라 달라질 수 있습니다." |
| **P0** | legal_certainty | 법적 문제 없음, 위반건축물 문제 없 | "법적 사항은 별도 법률 전문가 검토가 필요합니다." |
| **High** | valuation_certainty | 적정 가격, 저평가, 시장가보다 저렴, 가격 상승 확실 | "주변 사례와 보정 기준을 함께 검토해야 합니다." |
| **High** | value_add_certainty | 리모델링하면 임대료 상승, 공실 쉽게 해소 | "가치상승 가능성은 공사비, 공실기간, 주변 임대사례 확인이 필요합니다." |
| **High** | tax_certainty | 세금상 유리, 절세 가능, 세금 없 | "세무 사항은 전문 세무사 검토가 필요합니다." |
| **High** | permit_certainty | 용도변경 가능, 증축 가능 | "용도변경 및 증축 가능 여부는 관할 관청 확인이 필요합니다." |

#### 검증 결과 판정 로직

```
IF P0 심각도 패턴 탐지 → status = "blocked" (출력 차단)
ELSE IF High 심각도 패턴 탐지 → status = "revise" (자동 대체 후 출력)
ELSE → status = "pass" (원본 출력 허용)
```

**핵심 기술적 특징**: 위험 경계 검증은 **섹션 유형(section_type)을 고려한 컨텍스트 인식(context-aware) 판정**을 수행한다. 예를 들어, "가치평가(valuation)" 섹션에서의 금융 표현은 "위치(location)" 섹션에서의 동일 표현보다 높은 심각도로 판정된다.

### 2. 공개 가드 모듈 (DisclosureGuard) (220)

공개 가드 모듈(220)은 문서에 포함된 **5종의 보호 필드**에 대해 자동 탐지 및 차폐를 수행한다.

#### 보호 필드 탐지기 테이블

| 보호 필드 | 탐지 패턴 | 공개 출력 차폐 | 대체 텍스트 |
|----------|----------|-------------|-----------|
| 정확한 주소 (exact_address) | `[가-힣]+구\s+[가-힣]+동\s*\d+` | 항상 차폐 | "[지역 신호로 대체됨]" |
| 임차인 실명 (tenant_name) | 브랜드명 DB + `임차인/입점/세입자` 패턴 | 항상 차폐 | "[임차인 업종 정보로 대체됨]" |
| 개별 임대료 (unit_rent) | `월세\s*\d+만\s*원`, `보증금\s*\d+억` | 항상 차폐 | "[임대수익 존재, 상세 내용 비공개]" |
| 매도자 사정 (seller_motivation) | `상속\s*문제로`, `급매`, `이혼\s*매물` | 항상 차폐 | "[매도자 사정 비공개]" |
| 협상 내용 (negotiation_memo) | `\d+억까지\s*가능`, `네고\s*가능` | 항상 차폐 | "[내부 협상 메모 비공개]" |

#### 게이트 수준별 공개 정책

공개 가드 모듈(220)은 **출력 유형(output_type)과 게이트 수준(gate_level)의 조합**에 따라 차폐 여부를 결정한다:

| 게이트 수준 | 의미 | 공개 가능 정보 | 차폐 대상 |
|-----------|------|-------------|----------|
| **G0** | 공개 블라인드 | 권역, 자산유형, 가격대 | 5종 전체 차폐 |
| **G1** | 비밀유지확약(NDA) 서명 후 | G0 + 대략적 위치 | 정확한 주소, 매도자 사정, 협상 내용 차폐 |
| **G2** | 관심 표명 + 브로커 승인 | G1 + 임차인 업종, 총 임대료 | 임차인 실명, 매도자 사정 차폐 |
| **G3** | 상세 실사 진입 | G2 + 임차인 실명, 개별 임대료 | 매도자 사정, 협상 내용 차폐 |
| **G4** | 매수 의향서(LOI) 제출 후 | G3 + 매도자 사정 | 협상 내용만 차폐 |
| **G5** | 최종 협상 단계 | 전체 공개 | 차폐 없음 |

**핵심 기술적 특징**: 게이트 수준은 **거래 파이프라인 상태와 자동 연동**된다. 예를 들어, 열람자가 관심을 표명하고 브로커가 승인하면 자동으로 G2로 승급되어, 해당 열람자에게는 더 많은 정보가 공개되는 문서가 렌더링된다.

### 3. 게이트 리뷰 모듈 (230)

게이트 리뷰 모듈(230)은 위험 경계 검증(210)과 공개 가드(220)의 결과를 종합하여 **문서의 최종 출력 가능 여부**를 판정한다.

```
최종 판정 로직:
  IF RiskBoundary.status == "blocked" → 출력 차단 (P0 위험 표현 존재)
  IF DisclosureGuard.status == "blocked" → 출력 차단 (G4 미만에서 민감정보 미차폐)
  IF RiskBoundary.status == "revise" → 자동 대체 후 출력 허용
  IF DisclosureGuard.status == "redacted" → 차폐 적용 후 출력 허용
  ELSE → 원본 출력 허용
```

게이트 리뷰 결과는 다음 데이터를 포함한다:
- `disclosure_status`: "pass" | "redacted" | "blocked"
- `risk_status`: "pass" | "revise" | "blocked"
- `redacted_fields`: 차폐된 필드 목록
- `risk_issues`: 탐지된 위험 이슈 목록 (심각도, 메시지)
- `boundary_note`: 면책 문구

### 4. 출력 분기기 모듈 (235)

출력 분기기 모듈(235)은 동일한 원본 데이터로부터 **열람자의 게이트 수준에 따라 서로 다른 문서**를 동적으로 렌더링한다.

```
동일 물건에 대해:
  투자자 A (G0, 공개 블라인드) → "[성수권역] 꼬마빌딩 70~80억, [임대수익 존재]"
  투자자 B (G2, NDA 서명 완료) → "성수동2가 꼬마빌딩 76억, 1층 F&B 월 400만원"
  투자자 C (G4, LOI 제출 후)  → "성수동2가 123-4, 1층 스타벅스 월 400만원, 매도자 상속 사유"
```

이는 하나의 데이터 소스에서 복수의 가시성 수준(visibility level)을 갖는 문서를 동적으로 생성하는 것으로, **열람자 인증 정보 + 게이트 수준 + 출력 유형**의 삼원 결합(three-way join)으로 렌더링 범위가 결정된다.

### 5. 열람 행동 분석 엔진 (250)

열람 행동 분석 엔진(250)은 정보 공개가 제어된 문서의 열람자 행동 데이터를 수집·분석한다.

#### 수집 데이터

| 데이터 항목 | 설명 | 수집 방법 |
|-----------|------|----------|
| 섹션별 체류 시간 | 각 섹션 카드를 열람한 초 단위 시간 | Intersection Observer API |
| 스와이프 패턴 | 섹션 탐색 순서 및 되돌아간 섹션 | 터치/마우스 이벤트 |
| 공유 이벤트 | 카카오톡/SMS/이메일 공유 클릭 | 버튼 이벤트 |
| 관심 표명 | "관심 있음" 버튼 클릭 | 버튼 이벤트 |
| Full IM 요청 | "상세 IM 요청" CTA 클릭 | 버튼 이벤트 |
| 재방문 패턴 | 동일 열람자의 재방문 횟수 및 간격 | 쿠키/세션 |
| 디바이스 유형 | 모바일/태블릿/데스크탑 | User-Agent |

#### 분석 산출물 — 개별 딜 인텔리전스

**(a) 열람자 관심도 점수 (Viewer Interest Score)**

각 열람자에 대해, 행동 데이터를 가중 합산하여 0~100의 관심도 점수를 산출한다:

| 행동 | 가중치 | 산출 로직 |
|------|-------|----------|
| 총 체류 시간 | 30% | min(total_dwell / 120, 1.0) × 30 |
| 수익 분석 체류 비중 | 20% | (income_dwell / total_dwell) × 20 |
| 재방문 횟수 | 15% | min(revisit_count / 3, 1.0) × 15 |
| 관심 표명 | 20% | interest_expressed ? 20 : 0 |
| Full IM 요청 | 15% | full_im_requested ? 15 : 0 |

**(b) 섹션 관심 히트맵 (Section Interest Heatmap)**

각 섹션의 평균 체류 시간을 시각적 히트맵으로 제공하여, 투자자가 어느 부분에 가장 관심이 높은지를 중개인이 직관적으로 파악할 수 있도록 한다.

**(c) 후속 행동 권장 (Next Best Action)**

관심도 점수와 행동 패턴을 기반으로, 중개인에게 다음 최적 행동을 권장한다:

| 관심도 범위 | 행동 패턴 | 권장 행동 |
|-----------|----------|----------|
| 80~100 | 관심 표명 + 수익 분석 집중 | "즉시 연락하여 현장 방문 일정을 잡으세요" |
| 60~79 | 재방문 있으나 관심 미표명 | "추가 자료(비교사례, 시장 리포트)를 제공하세요" |
| 40~59 | 1회 열람, 입지 섹션만 관심 | "유사 권역의 다른 물건도 함께 제안하세요" |
| 0~39 | 빠른 이탈 | "이 투자자에게는 다른 물건을 우선 제안하세요" |

### 6. 시장 인텔리전스 집계 엔진 (260)

시장 인텔리전스 집계 엔진(260)은 다수의 딜에서 수집된 열람 행동 데이터를 **익명화(anonymization)**하여 집계하고, 시장 수준의 수요 트렌드를 도출한다.

#### 집계 지표

| 지표 | 산출 방법 | 활용 |
|------|----------|------|
| 권역별 관심도 지수 | 권역별 모바일 IM 열람수 / 발행수 | 투자 수요 핫스팟 식별 |
| 자산유형별 수요 비중 | 자산유형별 관심 표명 비율 | 시장 선호 자산유형 트렌드 |
| 수익률 구간별 수요 분포 | 수익률 구간별 체류 시간 분포 | 시장 기대 수익률 파악 |
| 가격대별 활성도 | 가격대별 열람~관심 전환율 | 가격 적정성 시그널 |
| 섹션별 글로벌 관심도 | 전체 딜 평균 섹션별 체류 시간 | 콘텐츠 최적화 지침 |

**핵심 기술적 특징**: 개별 열람자의 행동 데이터는 **익명화** 및 **차분 프라이버시(differential privacy)** 처리를 거쳐 집계되며, 개인 식별이 불가능한 형태로만 시장 인텔리전스가 산출된다. 또한, 특정 딜의 관심도가 시장 평균 대비 유의미하게 높거나 낮은 경우, 해당 중개인에게 **벤치마크 알림**이 제공된다.

## 발명의 효과

1. **법적 리스크 자동 방지**: AI가 생성한 투자문서에서 자본시장법 위반 가능성이 있는 위험 표현을 자동 탐지·차단하여, 중개인 및 투자자의 법적 리스크를 사전에 방지한다.

2. **거래 단계별 최적 정보 공개**: 동일한 물건에 대해 열람자의 거래 단계에 따라 정보 공개 범위를 자동으로 조절하여, 딜 초기에는 관심을 유발하면서도 민감 정보는 보호하고, 거래가 진전됨에 따라 점진적으로 상세 정보를 공개하는 최적의 정보 흐름을 구현한다.

3. **매수자 관심도 정량화**: 열람 행동 데이터에 기반한 관심도 점수를 통해, 중개인이 "어느 매수자에게 얼마나 집중해야 하는가"를 데이터로 판단할 수 있다.

4. **후속 행동 자동 권장**: 관심도 점수와 행동 패턴에 기반하여 중개인에게 최적의 후속 행동을 자동으로 권장하여, 거래 성사율을 향상시킨다.

5. **시장 수요 인텔리전스**: 누적된 열람 행동 데이터의 익명 집계를 통해, 시장 참여자들에게 권역별·자산유형별·수익률 구간별 실시간 수요 트렌드를 제공하여, 데이터 기반 의사결정을 지원한다.

6. **데이터 네트워크 효과**: 시스템 사용자가 증가할수록 행동 데이터의 집계 품질이 향상되어, 시장 인텔리전스의 정확도와 가치가 비선형적으로 증가하는 네트워크 효과가 발생한다.

## 도면의 간단한 설명

- **도 1**: 시스템 전체 구성도 (삼중 안전 아키텍처 + 열람 분석 + 시장 인텔리전스)
- **도 2**: 위험 경계 검증 모듈(210)의 금지 패턴 매칭 및 안전 대체 프로세스
- **도 3**: 공개 가드 모듈(220)의 보호 필드 탐지 및 게이트 수준별 차폐 로직
- **도 4**: 게이트 수준(G0~G5) 상태 전이도 및 거래 파이프라인 연동
- **도 5**: 출력 분기기(235)의 동적 렌더링 — 동일 데이터에서 게이트 수준별 서로 다른 문서 생성
- **도 6**: 열람자 관심도 점수 산출 로직 흐름도
- **도 7**: 후속 행동 권장(Next Best Action) 의사결정 트리
- **도 8**: 시장 인텔리전스 집계 및 벤치마크 알림 프로세스

## 발명의 실시예

### 실시예 1: 삼중 안전 아키텍처의 동작

1. AI가 성수동 꼬마빌딩에 대한 수익 분석 섹션을 생성하면서 "이 물건은 수익률이 보장됩니다"라는 표현을 포함하였다.
2. 위험 경계 검증 모듈(210)이 "수익률 보장" 패턴을 탐지하고, P0 심각도의 `financial_certainty` 이슈로 판정한다.
3. 해당 섹션의 상태가 `blocked`로 설정되어, 원본 출력이 차단된다.
4. 시스템이 자동으로 "수익률은 가정과 실사 결과에 따라 달라질 수 있습니다."로 대체한 안전 텍스트(safe_text)를 생성한다.
5. 공개 가드 모듈(220)이 "성수동2가 123-4" 주소 패턴과 "스타벅스" 임차인 실명을 탐지하고, G0 열람자에 대해 각각 "[성수권역]"과 "[1층 F&B 앵커 테넌트]"로 차폐한다.
6. 게이트 리뷰(230)가 최종 판정: `risk_status: "revise"`, `disclosure_status: "redacted"` → 대체/차폐 적용 후 출력 허용.

### 실시예 2: 열람 행동 기반 거래 인텔리전스

1. 투자자 B가 성수 꼬마빌딩 모바일 IM을 열람한다.
2. 열람 분석 엔진(250)이 다음 행동 데이터를 수집한다:
   - property_overview: 12초, location_access: 38초, lease_status: 22초, income_analysis: 45초, risk_check: 5초, investment_thesis: 35초, next_steps: 8초
   - 총 체류: 165초, 재방문: 2회, 관심 표명: ✅, Full IM 요청: ❌
3. 관심도 점수 산출: (165/120 × 30=30) + (45/165 × 20=5.5) + (2/3 × 15=10) + (20) + (0) = **65.5점**
4. 후속 행동 권장: "추가 자료(비교사례, 시장 리포트)를 제공하세요. 수익 분석에 가장 관심이 높으므로, 상세 NOI 분석 자료를 준비하여 보내면 효과적입니다."
5. 중개인 A의 대시보드에 인사이트가 표시된다.

### 실시예 3: 시장 인텔리전스 산출

1. 100개의 모바일 IM 딜에서 30일간 수집된 열람 행동 데이터를 익명 집계한다.
2. 시장 인텔리전스 산출:
   - 성수/성동 권역: 관심도 지수 78 (전월 대비 +35%)
   - 꼬마빌딩(50~100억): 수요 비중 42% (최고)
   - 수익률 4~5% 구간: 수요 집중도 58%
3. 성수동 꼬마빌딩을 보유한 중개인 C에게 벤치마크 알림: "귀하의 물건은 열람~관심 전환율 6.4%로, 시장 평균 3.2% 대비 2배 높습니다."

## 요약서

본 발명은 AI가 생성한 부동산 투자문서에 대해, 위험 표현 자동 탐지·차단(RiskBoundary), 민감 정보 단계적 차폐(DisclosureGuard), 다단계 게이트 리뷰(Gate Review)로 구성된 삼중 안전 아키텍처를 적용하여 정보 공개 수준을 거래 단계에 따라 자동 제어하고, 제어된 문서의 열람자 행동 데이터(섹션별 체류 시간, 재방문, 관심 표명 등)를 수집·분석하여 열람자별 관심도 점수를 산출하고 매도 측 중개인에게 후속 행동 권장사항을 포함하는 거래 인텔리전스를 제공하며, 복수 딜의 익명 행동 데이터를 집계하여 시장 수준의 수요 트렌드 인텔리전스를 산출하는 시스템 및 방법에 관한 것이다.
