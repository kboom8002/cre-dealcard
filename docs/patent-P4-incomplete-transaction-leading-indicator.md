# 특허명세서 — P4

## 발명의 명칭

**미완료 거래 파이프라인 데이터를 활용한 시장 선행지표 예측 방법 및 시스템**

Method and System for Predicting Market Leading Indicators Using Incomplete Transaction Pipeline Data

---

## 기술분야

본 발명은 고관여 거래(high-involvement transaction) 중개 플랫폼에서 생성되는 거래 파이프라인 데이터의 분석에 관한 것으로, 보다 상세하게는 거래가 성사되지 않고 종료되거나 중단된 **미완료 거래(incomplete transaction)** 데이터를 체계적으로 수집하고, 해당 데이터에서 다차원 피처 벡터를 자동 추출하며, 적응형 예측 모델을 통해 개별 거래의 전환 확률 및 시장 수준의 선행 지표(수요 강도, 적정 가격 밴드, 흡수율 등)를 산출하는 방법 및 시스템에 관한 것이다.

---

## 배경기술

### 종래 기술의 문제점

고관여 거래 시장(상업용 부동산, M&A, 기업 대출, 프랜차이즈 양수도 등)에서 시장 분석은 거래 의사결정의 핵심 기반이다. 종래의 시장 분석 기술은 다음과 같은 한계를 갖는다.

**① 완료된 거래만 데이터로 활용**: 기존의 모든 시장 분석 시스템(예: Zillow Zestimate, CoStar Analytics, R-ONE, KB부동산 시세)은 **거래가 최종 완료된 시점의 데이터**(실거래가, 계약서 정보)만을 수집·분석한다. 그러나 시장에서 발생하는 거래 시도(transaction attempt) 중 상당수(업종에 따라 50~90%)는 최종 계약에 이르지 못하고 중간 단계에서 종료된다. 이러한 **미완료 거래 데이터는 체계적으로 수집되지 않으며, 분석에 활용되지 않는다.**

**② 사후적(lagging) 지표의 한계**: 완료된 거래 기반의 시장 분석은 본질적으로 **후행 지표(lagging indicator)**이다. 거래가 완료되어 공시되기까지 통상 1~3개월이 소요되므로, 시장 참여자는 이미 과거가 된 데이터에 기반하여 의사결정을 내려야 한다. 시장의 수요 변화, 가격 저항선, 업종별 트렌드를 **실시간으로** 파악하는 수단이 존재하지 않는다.

**③ 미완료 사유의 구조화 부재**: 거래가 불성립되는 사유(예: 매수자 예산 초과, 매도자 가격 고집, 입지 부적합, 규제 장벽, 자금 조달 실패 등)는 **시장의 숨겨진 수요·공급 신호**를 담고 있다. 예를 들어, 특정 권역에서 매칭 S등급(최고 적합도) 거래가 연속적으로 "매수자 예산 초과"로 불전환되면, 이는 해당 권역의 **매도호가가 시장 적정가를 상회**한다는 강력한 선행 신호이다. 그러나 종래 기술에서는 이러한 미완료 사유를 구조화하여 수집·분석하는 시스템이 없다.

**④ 콜드스타트(Cold Start) 대응 부재**: 새로운 시장 또는 플랫폼 초기에는 축적된 거래 데이터가 부족하여, 통계적 학습 모델을 적용할 수 없다. 종래 기술은 일정 수준 이상의 데이터가 축적될 때까지 예측 기능을 제공하지 못하거나, 정밀도가 현저히 낮은 결과를 생성한다.

**⑤ 피드백 루프의 부재**: 시장 예측 결과가 중개인의 행동에 영향을 미치고, 그 행동이 새로운 데이터를 생성하여 예측 모델의 정밀도를 개선하는 **자기강화 피드백 루프(self-reinforcing feedback loop)**를 구현한 시스템이 존재하지 않는다.

따라서, 미완료 거래 데이터를 체계적으로 수집·구조화·분석하여, 시장 가격·수요·공급의 **선행 지표**를 실시간으로 산출하고, 예측 결과가 참여자 행동을 통해 모델 정밀도를 개선하는 자기강화 구조를 갖는 시스템이 요구된다.

---

## 해결하고자 하는 과제

본 발명은 다음을 목적으로 한다:

1. 고관여 거래 중개 플랫폼에서 발생하는 **거래 파이프라인의 모든 상태 전환 이벤트**(성사 및 미완료 포함)를 체계적으로 수집하는 기술을 제공한다.
2. 수집된 파이프라인 이벤트에서 **다차원 피처 벡터**를 자동 추출하는 기술을 제공한다.
3. 데이터 축적 수준에 따라 **휴리스틱 규칙에서 통계적 학습 모델로 자동 전환**되는 적응형 예측 모델을 제공하여 콜드스타트 문제를 해결한다.
4. 개별 거래의 **전환 확률**, 권역별 **수요 강도 지수**, **적정 시장 가격 밴드**, **흡수율** 등 시장 선행 지표를 산출하는 기술을 제공한다.
5. 예측 결과가 참여자 행동에 영향 → 새로운 데이터 생성 → 모델 갱신의 **자기강화 피드백 루프**를 구현한다.

### 1. 파이프라인 이벤트 수집기 모듈 (310)

파이프라인 이벤트 수집기 모듈(310)은 고관여 거래 중개 플랫폼에서 발생하는 **모든 거래 상태 전환 이벤트**를 실시간으로 수집한다. 상기 이벤트는 다음을 포함하되 이에 한정되지 않는다:

- 매물/매도 의향 등록 이벤트
- 마케팅 자료(블라인드 티저, IM 등) 생성 이벤트
- 매칭 실행 이벤트 및 매칭 등급(S/A/B/C) 결과
- 매수/수요측 관심 표명 이벤트
- 미팅/현장 방문 예약 이벤트
- 비밀유지확약(NDA) 서명 이벤트
- 매수 의향서(LOI) 제출 이벤트
- 계약 체결 이벤트
- **각 단계에서의 이탈(exit) 이벤트** — 이탈 사유(budget_exceeded, location_mismatch, price_gap, regulatory_issue, funding_failure, timing_mismatch, other)를 포함

각 이벤트에는 다음 메타데이터가 부여된다:
- 타임스탬프(timestamp)
- 이벤트 유형(event_type)
- 거래 고유 식별자(deal_id)
- 참여자 역할(actor_role: broker, buyer, seller, platform)
- 이전 단계(from_stage) 및 현재 단계(to_stage)

**핵심 기술적 특징**: 종래 시스템은 최종 "계약 체결" 이벤트만 기록한다. 본 발명은 **이탈 이벤트를 포함한 모든 중간 상태 전환**을 구조화된 형태로 수집하여, 미완료 거래의 패턴을 분석 가능한 데이터로 변환한다.

### 2. 피처 벡터 추출기 모듈 (320)

피처 벡터 추출기 모듈(320)은 수집된 파이프라인 이벤트로부터 각 거래에 대한 **다차원 피처 벡터(Deal Feature Vector)**를 자동으로 추출한다. 상기 피처 벡터는 바람직하게는 다음 차원을 포함한다:

| # | 피처 | 설명 | 산출 방법 |
|---|------|------|----------|
| 1 | totalHoldDays | 총 파이프라인 체류 일수 | (현재일 − 최초등록일) |
| 2 | currentStageOrd | 현재 도달 단계의 서수 | 단계 열거형의 인덱스 |
| 3 | bestMatchGrade | 최고 매칭 등급 | S=4, A=3, B=2, C=1 |
| 4 | sGradeCount | S등급 매칭 건수 | 매칭 이력 집계 |
| 5 | aGradeCount | A등급 매칭 건수 | 매칭 이력 집계 |
| 6 | totalMatchCount | 총 매칭 건수 | 매칭 이력 집계 |
| 7 | stageVelocity | 단계 전환 속도 | currentStageOrd / totalHoldDays |
| 8 | priceRevisionCount | 가격 조정 횟수 | 가격 변경 이벤트 수 |
| 9 | priceRevisionMagnitude | 가격 조정 총 변동폭 | |ΔPrice| 누적합 / 최초가격 |
| 10 | eventFrequency | 이벤트 발생 빈도 | 총 이벤트 수 / totalHoldDays |
| 11 | daysInCurrentStage | 현재 단계 체류 일수 | (현재일 − 현재단계 진입일) |
| 12 | interestExpressionCount | 관심 표명 건수 | 관심 이벤트 집계 |
| 13 | ndaSigned | NDA 서명 여부 | boolean → 0/1 |
| 14 | loiSubmitted | LOI 제출 여부 | boolean → 0/1 |
| 15 | exitStageOrd | 이탈 단계 서수 (미완료 시) | 이탈 이벤트의 from_stage 인덱스 |
| 16 | exitReasonCategory | 이탈 사유 카테고리 | 카테고리 원핫 인코딩 |
| 17 | marketSegment | 시장 세그먼트 | 권역 + 자산유형 조합 인덱스 |
| 18 | priceToMarketRatio | 호가 대비 시장가 비율 | 호가 / 해당 세그먼트 평균 거래가 |

상기 피처 벡터의 차원 수 및 각 피처의 구성은 도메인에 따라 변경 가능하며, 바람직하게는 10개 이상의 차원을 포함한다.

### 3. 적응형 예측 모델 엔진 모듈 (330)

적응형 예측 모델 엔진 모듈(330)은 **데이터 축적 수준에 따라 예측 모델을 자동으로 전환**하는 적응형 구조를 갖는다:

**(a) 제1 모드 — 휴리스틱 모드 (n < N₁)**

축적된 완료+미완료 거래 데이터가 제1 임계값(N₁, 바람직하게는 80건) 미만인 경우, 도메인 전문가 지식에 기반한 **규칙 기반 휴리스틱**으로 전환 확률을 예측한다:

```
P_conversion = Σ(w_i × f_i) / Σ(w_i × max_i)

여기서:
- f_i: i번째 피처 값 (정규화)
- w_i: i번째 피처의 도메인 전문가 가중치
- max_i: i번째 피처의 이론적 최대값
```

상기 가중치 w_i는 도메인 전문가가 사전에 설정한 값으로, 바람직하게는 다음과 같은 우선순위를 갖는다:
- 단계 진행도(currentStageOrd) × 0.25
- 최고 매칭 등급(bestMatchGrade) × 0.20
- 관심 표명 건수(interestExpressionCount) × 0.15
- 단계 전환 속도(stageVelocity) × 0.15
- 가격 조정 횟수(priceRevisionCount, 역방향) × 0.10
- 체류 일수(totalHoldDays, 역방향) × 0.10
- 이벤트 빈도(eventFrequency) × 0.05

**(b) 제2 모드 — 통계 학습 모드 (N₁ ≤ n < N₂)**

축적 데이터가 제1 임계값(N₁) 이상 제2 임계값(N₂, 바람직하게는 500건) 미만인 경우, **로지스틱 회귀 모델**로 자동 전환한다:

```
P_conversion = σ(β₀ + β₁×f₁ + β₂×f₂ + ... + β_k×f_k)
σ(x) = 1 / (1 + e^(-x))
```

모델 계수 β는 축적된 완료/미완료 거래 데이터에 대해 최대우도추정(MLE)으로 학습된다.

**자동 전환 트리거**: 모듈(330)은 제1 임계값 도달 시 휴리스틱 모드와 통계 학습 모드의 교차 검증(cross-validation) 성능을 비교하고, 통계 학습 모드의 AUC-ROC가 휴리스틱 모드를 소정의 마진(바람직하게는 5%p) 이상 상회하는 경우 자동 전환한다.

**(c) 제3 모드 — 앙상블 모드 (n ≥ N₂)**

축적 데이터가 제2 임계값(N₂) 이상인 경우, 복수의 학습 모델(로지스틱 회귀, 그래디언트 부스팅, 랜덤 포레스트 등)의 **앙상블**로 전환한다. 각 모델의 예측값을 가중 평균하여 최종 전환 확률을 산출하되, 각 모델의 가중치는 최근 검증 데이터에서의 성능에 비례한다.

### 4. 선행 지표 생성기 모듈 (340)

선행 지표 생성기 모듈(340)은 개별 거래의 전환 확률 및 누적된 파이프라인 데이터를 집계하여 **시장 수준의 선행 지표**를 생성한다:

**(a) 수요 강도 지수 (Demand Intensity Index)**

특정 시장 세그먼트(권역 × 자산유형)에서의 수요 강도를 산출한다:

```
DII(s, t) = Σ(관심 표명 이벤트 수) / Σ(활성 매물 수)
          × 시간 가중 계수(최근 이벤트에 높은 가중치)
```

상기 지수가 상승하면 해당 세그먼트의 수요가 증가하고 있음을 선행적으로 나타낸다.

**(b) 적정 시장 가격 밴드 (Market Price Band)**

미완료 거래 데이터에서 **이탈 사유가 "예산 초과(budget_exceeded)"인 건**의 패턴을 분석하여 적정 시장 가격을 역산한다:

```
적정 가격 상한 = median(예산 초과 이탈 건의 매수자 예산 상한)
적정 가격 하한 = median(예산 초과 이탈 건의 매수자 예산 하한)
시장 적정 밴드 = [하한, 상한]
```

**핵심 기술적 특징**: 종래의 시세 추정은 **완료된 거래**의 실거래가에 기반한다. 본 발명은 **"거래가 불발된 이유(예산 초과)"에서 시장 가격을 역산**하는 것으로, 이는 완료된 거래 이전에 시장 적정가를 파악할 수 있게 한다.

**(c) 흡수율 (Absorption Rate)**

특정 세그먼트에서 매물이 시장에서 소화되는 속도를 산출한다:

```
흡수율(s, t) = 기간 내 완료 거래 수 / 기간 초 활성 매물 수
보정 흡수율 = 흡수율 × (1 + 전환확률 가중 파이프라인 건수 / 활성 매물 수)
```

**보정 흡수율**은 현재 파이프라인에 있는 거래 중 전환 확률이 높은 건수를 반영하여, 종래의 흡수율보다 선행적인 지표를 제공한다.

**(d) 업종별 수요 트렌드 (Sector Demand Trend)**

임대차 거래의 경우, 임차인 의향서의 업종별 추이를 분석하여 업종 수요 트렌드를 산출한다:

```
업종_트렌드(업종, t) = 금주_의향서_수(업종) / 전월_주평균_의향서_수(업종) × 100
```

**(e) 시장 온도 (Market Temperature)**

시장 전반의 과열/냉각 정도를 나타내는 복합 지표:

```
시장_온도(t) = w₁ × (평균_계약_소요일의 변화율⁻¹) 
             + w₂ × (수요_강도_지수의 변화율)
             + w₃ × (가격_조정_횟수의 변화율⁻¹)
```

### 5. 피드백 루프 엔진 모듈 (350)

피드백 루프 엔진 모듈(350)은 선행 지표 생성기(340)의 산출물을 거래 참여자(중개인, 매수자 등)에게 제공하고, 참여자의 후속 행동이 새로운 파이프라인 이벤트를 생성하여 이벤트 수집기(310)로 환류되는 **자기강화 피드백 루프**를 구현한다:

```
[선행 지표 제공] → [참여자 행동 변화]
  "이 가격대는 시장 저항이 강합니다"  → 매도자가 호가 인하 → 가격 조정 이벤트 생성
  "성수동 50평대 수요 급증 중"       → 브로커가 해당 매물 우선 마케팅 → 매칭 이벤트 생성
  "이 거래 전환 확률 78%"           → 브로커가 집중 관리 → 관심 표명 이벤트 생성
                                       ↓
                              [새 이벤트 → 피처 벡터 갱신 → 모델 재학습]
                                       ↓
                              [더 정밀한 선행 지표 → ... 반복]
```

### 6. 시장 인텔리전스 집계기 모듈 (360)

시장 인텔리전스 집계기 모듈(360)은 복수의 거래에서 수집된 데이터를 **익명화(anonymization)** 및 **차분 프라이버시(differential privacy)** 처리를 거쳐 집계한다. 개별 거래나 참여자를 식별할 수 없는 형태로 시장 인텔리전스를 산출한다.

특정 거래의 선행 지표가 시장 평균 대비 유의미하게 높거나 낮은 경우(소정의 표준편차 이상), 해당 거래의 중개인에게 **벤치마크 알림**을 제공한다:

> "귀하의 매물은 열람→관심 전환율 6.4%로, 동일 세그먼트 시장 평균 3.2% 대비 2배 높습니다."

   - 예산 초과 이탈 2건의 매수자 예산 상한: [72억, 75억]
   - 적정 가격 밴드: **70~75억원** (현 호가 80억원은 시장 적정가 상회)

7. **피드백 루프 엔진(350)**이 중개인 A에게 알림: "S등급 매칭 2건이 모두 예산 초과로 이탈했습니다. 성수권역 동급 매물의 시장 적정가는 70~75억원으로 추정됩니다."

8. 중개인 A가 매도자를 설득하여 호가를 76억원으로 조정한다. 이 **가격 조정 이벤트**가 수집기(310)로 환류되어 모델이 갱신된다.

### 실시예 2: 상업용 임대차 — 강남 오피스 50평

1. 건물주가 강남 오피스 50평(보증금 5000만원/월세 250만원) 임대 매물을 등록한다.

2. AI 매칭 엔진이 임차인 의향서 DB에서 8건의 매칭을 생성한다: S등급 3건, A등급 3건, B등급 2건.

3. S등급 3건이 모두 **"월세 예산 200만원 상한으로 초과"**로 이탈한다.

4. **선행 지표 생성기(340)**가 산출:
   - 강남 오피스 50평대 적정 월세 밴드: **180~210만원** (현 호가 250만원은 상회)
   - 수요 강도 지수: 2.8 (높음 — S등급 3건이나 가격 저항이 강함)
   - 업종별 수요: IT/스타트업 60%, 전문서비스 25%, F&B 15%

5. 피드백 루프가 건물주에게 제공: "수요는 강하나 가격 저항이 있습니다. 월세 200~220만원으로 조정 시 S등급 매칭 전환 가능성이 높습니다."

### 실시예 3: 시장 인텔리전스 — 30일간 100개 딜 데이터 집계

1. 30일간 100개 딜의 파이프라인 데이터(완료 15건, 미완료 85건)를 익명 집계한다.

2. **시장 인텔리전스 집계기(360)** 산출:
   - 권역별 수요 강도: 성수/성동 2.8 (전월 대비 +35%), 강남 2.1 (전월 대비 -5%)
   - 가격대별 전환율: 50~70억 구간 18% (최고), 100억+ 구간 6% (최저)
   - 시장 온도: 0.72 (약간 과열 — 평균 계약 소요일 45일→32일)
   - 최다 이탈 사유: 예산 초과 42%, 입지 부적합 23%, 가격 격차 18%

3. 중개인 C의 매물(성수동 오피스)에 대해 벤치마크 알림: "귀하의 매물은 열람→관심 전환율 8.5%로, 동일 세그먼트 평균 3.2% 대비 2.7배 높습니다."

## 발명의 효과

1. **선행적 시장 분석**: 종래의 후행 지표(완료 거래 기반)를 보완하여, 거래 완료 이전에 시장 수요·가격·흡수율의 변화를 감지할 수 있다.

2. **미완료 데이터의 가치 전환**: 종래에는 폐기되던 미완료 거래 데이터(전체 거래 시도의 50~90%)를 구조화하여 시장 인사이트로 변환한다.

3. **콜드스타트 해결**: 적응형 모델 구조를 통해 데이터 부족 초기에도 휴리스틱 기반 예측을 제공하고, 데이터 축적에 따라 자동으로 고정밀 모델로 전환한다.

4. **자기강화 정밀도 향상**: 피드백 루프를 통해 예측 결과가 참여자 행동에 영향을 미치고, 그 행동이 새로운 데이터를 생성하여 모델 정밀도가 지속적으로 향상되는 데이터 플라이휠 효과를 달성한다.

5. **데이터 해자(Data Moat) 구축**: 시간이 지남에 따라 축적되는 미완료 거래 데이터는 후발 주자가 동일 기간을 투입하지 않고는 복제할 수 없는 자산으로, 지속 가능한 경쟁 우위를 제공한다.

6. **시장 투명성 제고**: 익명 집계된 시장 인텔리전스를 통해 시장 참여자들의 정보 비대칭을 줄이고, 데이터 기반 의사결정을 지원한다.

## 선행기술 회피 분석

| 선행기술 | 차이점 |
|----------|--------|
| **Zillow Zestimate** (US10475144B2) | 완료된 거래의 실거래가만 활용. 미완료 거래 데이터 수집·분석 없음. 피드백 루프 없음 |
| **CoStar Analytics** | 시장 동향 = 완료 거래 집계. 이탈 사유 구조화 없음. 적응형 모델 없음 |
| **Dealpath** | 포트폴리오 관리 도구. 파이프라인 데이터의 예측 분석 없음 |
| **Crexi / Buildout** | 마케팅 자동화 위주. 미완료 데이터 수집·분석 구조 없음 |
| **KB부동산 / R-ONE** | 공시 데이터 + 실거래가 기반 시세 추정. 미완료 데이터 및 선행 지표 개념 없음 |

**핵심 차별점**: 종래의 모든 부동산/거래 분석 시스템은 **"성사된 거래"**만 데이터로 취급한다. 본 발명은 **"성사되지 않은 거래의 패턴"에서 시장의 미래를 예측**하는 것으로, 데이터 소스 자체가 선행기술과 질적으로 다르다.

## 요약서

본 발명은 고관여 거래 중개 플랫폼에서 발생하는 모든 거래 상태 전환 이벤트를, 거래 성사(완료) 이벤트뿐 아니라 거래가 성사 없이 종료되거나 중단된 미완료 이벤트를 포함하여 체계적으로 수집하고, 각 거래에 대해 체류 기간, 도달 단계, 매칭 품질, 가격 변동 이력 등을 포함하는 다차원 피처 벡터를 자동 추출하며, 데이터 축적 수준에 따라 휴리스틱 규칙 모드에서 통계적 학습 모델로 자동 전환되는 적응형 예측 모델을 통해 개별 거래의 전환 확률, 시장 세그먼트별 수요 강도 지수, 미완료 거래의 이탈 사유 패턴에서 역산한 적정 시장 가격 밴드 등 시장 선행 지표를 산출하고, 산출된 선행 지표를 거래 참여자에게 제공하여 후속 행동을 유도하고 해당 행동으로 생성된 새로운 이벤트를 수집하여 예측 모델을 갱신하는 자기강화 피드백 루프를 구현하는 방법 및 시스템에 관한 것이다. 본 발명에 의하면 종래에 폐기되던 미완료 거래 데이터(전체 거래 시도의 50~90%)를 시장 인사이트로 변환하여, 완료된 거래 기반의 후행 지표를 보완하는 선행적 시장 분석이 가능하게 되는 효과가 있다.
