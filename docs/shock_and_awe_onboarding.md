# "Shock & Awe" 온보딩 설계

> **핵심 원칙**: 사용자가 **폼을 채우고 있다고 느끼면 안 된다.** "와, 이게 뭐야?" 연속 경험 속에서 자연스럽게 프로필이 완성되어야 한다.

---

## 기존 온보딩 vs Shock & Awe 온보딩

```
[기존 — 마찰의 연속]

회원가입 → 이름 입력 → 전화번호 → SMS 인증 → 사무실 주소 →
전문 분야 선택 → 경력 입력 → 프로필 사진 업로드 → 
"이제 시작하세요!" → ... 뭘 하지? → 이탈

소요시간: 5~8분
감정곡선: 귀찮음 → 짜증 → 포기
전환율: ~30%


[Shock & Awe — 감동의 연속]

프로필 사진 하나 찍기 → 🔥 AI가 나를 분석한다! → 
"당신은 Heritage-Authentic형" → 💥 명함이 30초만에! →
"관할 동네가 어디세요?" → ⚡ 내 동네 빌딩 데이터가! → 
"지금 매물 있으세요?" → 🎯 60초만에 딜카드! →
"이거 카톡에 보내보세요" → 📱 건물주한테 바로 전달!

소요시간: 4분 (체감: 1분)
감정곡선: 호기심 → 놀라움 → 탄성 → 공유 욕구
전환율: 목표 80%+
```

## 5단계 플로우 상세 설계

### 🎬 Stage 0: 진입 (10초)

```
┌─────────────────────────────────────┐
│                                     │
│     📸  사진 한 장이면 충분합니다     │
│                                     │
│   AI가 당신만의 프로필을 만들어       │
│   드립니다.                          │
│                                     │
│   ┌──────────────────────────┐      │
│   │                          │      │
│   │    📷 사진 찍기           │      │
│   │                          │      │
│   │    🖼️ 앨범에서 선택       │      │
│   │                          │      │
│   └──────────────────────────┘      │
│                                     │
│   카카오로 시작하기                   │
│                                     │
└─────────────────────────────────────┘
```

**설계 의도**:
- 회원가입 폼이 **보이지 않는다**
- CTA가 "회원가입" 이 아닌 **"사진 찍기"** — 행동 유도
- 카카오 로그인은 사진 선택 후 자연스럽게 (사진 먼저, 인증은 나중)
- "사진 한 장이면 충분합니다" — 진입 비용이 극히 낮음을 명시

**수집 데이터**: 없음 (사진 파일만)

```
│                                     │

│                                     │
│   ✓ 표정 분석 완료                   │
│   ✓ 색감 분석 완료                   │
│   ▸ 전문성 인상 분석 중...            │
│   ○ 신뢰도 벡터 산출 대기             │
│                                     │
└─────────────────────────────────────┘
```

**설계 의도**:
- 실제 Gemini Vision API 호출은 ~3초지만, **의도적으로 15~20초로 늘린다**
- 왜? → "빠르면 대충 한 것 같다". 분석 과정을 **스텝별로 보여주며** AI가 진지하게 분석 중이라는 인상
- 각 스텝이 완료될 때마다 ✓ 체크 → **기대감 점진적 상승**
- 이 30초 동안 사용자는 **화면을 떠나지 않는다** (호기심)

**기술 구현**:
```
실제 흐름:
  [사진 업로드] → [Gemini Vision 호출 (3초)] → [7D 벡터 수신]
  → [상보 벡터 계산 (즉시)] → [VTI 분류 (즉시)] → [프리셋 매칭 (즉시)]

연출 흐름 (프론트엔드):
  0s:  "표정 분석 중..." (실제로는 API 호출 중)
  3s:  "✓ 표정 분석 완료" (실제 결과 수신)
  5s:  "색감 분석 중..." (연출)
  8s:  "✓ 색감 분석 완료" (연출)
  10s: "전문성 인상 분석 중..." (연출 — 실제 상보 벡터 계산)
  14s: "✓ 전문성 인상 분석 완료"
  16s: "신뢰도 벡터 산출 중..." (연출 — 실제 VTI 분류)
  18s: "✓ 신뢰도 벡터 산출 완료"
  20s: "프로필 생성 중..." → 화면 전환
```

**수집 데이터**: 7D Vibe 벡터, VTI 유형 (모두 자동, 사용자 입력 없음)

### 💥 Stage 2: 결과 공개 — "나에 대한 AI 분석 결과" (45초)

이 단계가 **Shock & Awe의 핵심**이다.

```
│  ┌─── Vibe Card 완성 ────────────┐  │
│  │                               │  │
│  │  [상보적 배경 + 프로필 사진]    │  │

│  │                               │  │
```

**설계 의도 — 심리학적 근거**:

1. **바넘 효과(Barnum Effect)**: MBTI처럼 "나에 대한 분석 결과"는 **거부할 수 없는 관심**을 유발. 사람은 자신에 대한 이야기를 가장 주의 깊게 읽는다.

2. **시각적 충격**: 자기 사진이 AI가 선택한 전문적 배경과 합성되어 있다 → "이게 나야?" 반응 → **공유 욕구 폭발**

3. **숫자의 설득력**: Trust 82, Valence 71 같은 구체적 수치 → "어떻게 올리지?" 호기심 → 프로필 완성 동기

4. **결핍 시각화**: energy 0.35 ↑보강 표시 → "AI가 내 부족한 부분을 알고 보강해줬다" → **AI의 가치를 체감**

**카카오 공유 CTA가 여기에 있는 이유**:
- 프로필 완성 전에 공유 버튼을 보여준다
- "이름도 안 넣었는데 공유?" → 맞다. **불완전한 상태에서도 공유 가능**
- 공유 받은 사람: "이거 뭐야?" → 링크 클릭 → 본인도 시작 → **바이럴 루프**

**자연스러운 데이터 수집**:
```
"이름과 전문 분야를 추가하면 명함이 완성됩니다" 터치 시:

  ┌─────────────────────────┐
  │ 이름: [홍길동       ]    │  ← 카카오 프로필에서 자동 채움
  │ 연락처: [010-1234-5678] │  ← 카카오에서 자동 채움
  │                         │
  │ 전문 분야:              │

  │  ○ 오피스 임대          │
  │  ○ 상가 매매/임대       │
  │  ○ 지산/물류            │
  │  ○ 기타 [        ]     │
  │                         │
  │ 주 활동 권역:           │
  │  ● 성수/성동            │  ← 탭 한 번
  │  ○ 강남/서초            │
  │  ○ 여의도/마포          │

총 입력: 탭 2번 (전문 분야 + 권역)
나머지: 카카오에서 자동
체감: "폼 작성"이 아닌 "명함 커스터마이즈"
```

**수집 데이터**: 이름, 연락처, 전문 분야, 주 활동 권역 (총 4개, 입력 2탭)

## 바이럴 루프 구조

```
중개인 A: 사진 업로드
  ↓
Vibe Card 생성 (Trust 82)
  ↓
카톡으로 건물주 B에게 공유
  ↓
건물주 B: "이 명함 뭐지?" → 링크 클릭
  ↓
빌딩 레이더로 본인 건물 조회 → "내 건물이 이 정도야?"
  ↓
중개인 A에게 "매각 상담 한번 해볼까" 연락
  ↓
건물주 B가 다른 건물주 C에게 "이거 한번 해봐" 공유
  ↓ 
건물주 C: 빌딩 레이더 조회 → 매각 관심 시그널
  ↓
시스템에 매각 잠재 리드 축적

동시에:

중개인 A: 명함 카톡 공유
  ↓
동료 중개인 D: "나도 해볼래" → Stage 0 진입
  ↓
중개인 D: Vibe Card 생성 → 본인 네트워크에 공유
  ↓
... 자기 복제 ...
```

**핵심 지표**:
- K-Factor (바이럴 계수) = 초대 전송률 × 초대 전환율
- 목표: K ≥ 1.2 (자연 성장)
- 중개인 1명이 평균 3명에게 공유 × 전환율 40% = K = 1.2

## 이탈 방지 설계

### Stage 0에서 이탈 시
```
사진을 업로드했지만 카카오 로그인을 안 한 경우:
→ "분석 결과가 준비됐습니다! 로그인하면 바로 볼 수 있어요"
→ 사진 분석은 이미 완료 → 결과에 대한 호기심으로 로그인 유도
```

### Stage 2에서 이탈 시 (결과만 보고 떠남)
```
30분 후 카카오톡 알림:
"홍길동님의 Vibe 명함이 준비되어 있습니다. 
 이름만 추가하면 완성됩니다. (30초)"
→ 딥링크로 Stage 2 중단 지점으로 바로 이동
```

### Stage 3에서 이탈 시 (명함만 만들고 떠남)
```
다음 날 오전 9시 알림:
"성수/성동 권역 빌딩을 한번 분석해보세요.
 주소 하나면 건축물대장부터 시세까지 한 번에."
→ 빌딩 레이더로 바로 이동
```

### "나중에 하기" 선택 시
```
모든 Stage에 "나중에 하기" 존재
→ 이미 완료된 단계는 저장
→ 다음 로그인 시 중단 지점에서 재개
→ 절대 처음부터 다시 시작하게 하지 않음
```
