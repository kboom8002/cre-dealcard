## Segment A. 꼬마빌딩 전문 중개사 (JS부동산 소속)

### 현실 검증

| 시스템 기능 | 꼬마빌딩 업무 적합도 | 솔직한 평가 |
|-----------|----------------|-----------|
| 카톡 메모 → AI 딜카드 | ★★★★☆ | 유용하나, 숙련 브로커는 30분이면 직접 작성. "반나절→60초"는 과장 |
| 블라인드 매물장 자동 생성 | ★★★★★ | **핵심 업무와 정확히 일치.** 이것을 안 만들 수 없음 |
| 건축물대장 자동 조회 | ★★★★★ | **반복 노동 제거.** 정부 사이트 UX 고통을 직접 해결 |
| 실거래가 자동 조회 | ★★★★☆ | 유용하나 네이버·밸류맵으로도 가능. 차별화는 딜카드 내 자동 통합 |
| AI 매수자 매칭 | ★★☆☆☆ | **PoC 20명으로는 매수자 DB가 빈약.** 닭-달걀 문제 |
| 민감정보 자동 차단 | ★★★★☆ | 실수 방지 가치 높지만, 그 가치를 **사전에** 체감하기 어려움 |
| 카톡 문구 자동 생성 | ★★☆☆☆ | 브로커는 자기 말투에 자부심. AI 문구를 그대로 쓸 사람은 드묾 |
| 파이프라인 관리 | ★★★☆☆ | 엑셀/수첩 대비 좋지만, 학습 비용이 즉시 가치를 상쇄할 수 있음 |
| Agora (질문 게시판) | ★★☆☆☆ | 꼬마빌딩 전문 중개사는 이미 자기 네트워크가 있음. 온라인 QnA 수요 낮음 |
| Vendor 서비스 | ★★★☆☆ | 법무사/세무사 연결은 유용하나, 이미 알고 있는 사람이 있음 |

### 탑 5 소구력 순위

#### 🥇 1위: 건축물대장 + 실거래가 원클릭 조회

```
소구력: ★★★★★ | 동기 유형: 반복 고통 제거

· 매물마다 반드시 하는 작업 — 선택이 아닌 필수
· 정부 사이트(정부24, 세움터)의 끔찍한 UX
· 밸류맵/네이버로는 근생 건축물대장 상세(용도/구조/건폐율)를 못 봄
· 주소 한 번 입력으로 끝 → 체감 시간 절약이 즉각적

핵심 한마디:
"정부24 안 들어가도 됩니다. 주소 넣으면 바로 나옵니다."
```

#### 🥈 2위: 블라인드 딜카드 자동 생성 + 민감정보 자동 관리

```
소구력: ★★★★☆ | 동기 유형: 업무 품질 향상 + 리스크 제거

· 블라인드 매물장은 꼬마빌딩 거래의 첫 단추 — 이것 없이 시작 불가
· 매도인이 가장 싫어하는 것: "왜 내 주소가 돌아다녀?"
· AI가 "이건 숨겨야 합니다"를 자동 판단 → 실수 방지
· 다만 신규 도구 학습이 필요한 점은 허들

핵심 한마디:
"매물장 쓰다가 실수로 주소 풀릴 일 없습니다."
```

#### 🥉 3위: 무료 3개월 + JS부동산 소속감 + "20명 한정" 희소성

```
소구력: ★★★★☆ | 동기 유형: 관계적/심리적

· 솔직히 말하면 — 기능보다 이것이 실제 모집력의 절반
· "회장님이 직접 하시는 프로젝트" → 한국 조직 문화의 현실
· "돈 안 드니까 일단 해보자" → 진입 장벽 제로
· "20명만 뽑는다" → 특별대우 인식 + 탈락 회피 심리

핵심 한마디:
"회장님이 추진하시는 건데, 무료니까 일단 해보시죠."
```

#### 4위: 딜카드 기반 고객/파이프라인 관리

```
소구력: ★★★☆☆ | 동기 유형: 업무 체계화

· 매물-매수자-진행단계를 한눈에 보는 기능
· 엑셀/수첩보다 낫지만, 기존 습관을 바꿔야 하는 비용
· 장기적 가치는 높으나, 첫 한 달 안에 체감하기 어려움
· 매물이 10건 이상 쌓여야 관리 도구의 가치가 드러남

핵심 한마디:
"매물 5건 넘으면, 엑셀로는 한계가 옵니다."
```

#### 5위: AI 매수자 매칭 (조건부)

```
소구력: ★★☆☆☆ (현재) → ★★★★☆ (사용자 50명+ 시) | 동기 유형: 성사율 향상

· 현재: PoC 20명 → 매수 의향 등록이 극소 → 매칭이 제대로 안 됨
· 닭-달걀 문제: 매수자 DB가 없으면 매칭 불가, 매칭 안 되면 DB 안 쌓임
· 그러나 "향후 이렇게 된다"는 비전으로서 소구력은 있음
· PoC 기간 중 매칭 실적 1~2건만 나와도 폭발적 구전 효과

핵심 한마디:
"매수자 조건 넣어두면 맞는 매물 나올 때 자동으로 알려줍니다."
(단, 현재는 약속이지 경험이 아님 — 솔직하게 관리 필요)
```

## Segment B. 집합건물 전문 중개사 (용산구 등)

### "집합건물"이란?

구분소유가 적용되는 건물: **오피스텔, 상가(구분상가), 지식산업센터, 생활형숙박시설, 도시형생활주택** 등. 아파트만을 의미하지 않으며, **상업용 구분소유 물건도 포함**합니다.

### 현실 검증: 시스템과의 접점 재평가

| 집합건물 중개 업무 | 시스템 접점 | 평가 |
|----------------|----------|------|
| 오피스텔 매매·임대 | 딜카드 적합 (블라인드 필요한 고가 물건) | ★★★☆☆ |
| 구분상가 매매 | 딜카드 매우 적합 (근생 건축물대장 필수) | ★★★★☆ |
| 지식산업센터 임대 | 임대차 매칭 기능 적합 | ★★★☆☆ |
| 상가 공실 관리 | 임차인 의향 매칭 적합 | ★★★★☆ |
| 아파트 매매 | **불일치** — 네이버부동산/직방으로 충분 | ★☆☆☆☆ |
| 빌라/다세대 임대차 | **불일치** — 블라인드 불필요, 플랫폼 의존 | ★☆☆☆☆ |

> [!NOTE]
> **재발견**: 집합건물 중개사의 업무 중 **"구분상가 매매"와 "상가 공실 임차인 매칭"**은 시스템과 실질적 접점이 있습니다. 아파트/빌라만 취급하는 중개사와는 다릅니다.

### 탑 5 소구력 순위

#### 🥇 1위: 상가 공실 — 임차인 의향 매칭

```
소구력: ★★★★☆ | 동기 유형: 매출 직결

· 집합건물 중개사의 가장 큰 고민: "1층 상가가 6개월째 비었는데..."
· 건물주가 매일 "임차인 구했냐"고 전화
· 네이버부동산에 올려도 상가 임대는 반응이 느림
· 시스템의 tenant_intent(임차 의향) 등록 + AI 매칭은 여기서 작동
· 업종/예산/선호 위치 기반 매칭 → 중개사가 직접 발품 파는 것보다 효율적

핵심 한마디:
"건물주한테 '알아보고 있습니다'만 하지 말고,
 매칭 결과를 보여주세요."
```

#### 🥈 2위: 구분상가 매매 딜카드 + 건축물대장 자동 검증

```
소구력: ★★★★☆ | 동기 유형: 업무 효율

· 구분상가 매매 시 건축물대장 확인은 필수 (용도/면적/위반건축물 여부)
· 꼬마빌딩만큼은 아니지만, 고가 구분상가(₩5~30억)는 블라인드 매물장 필요
· 이 업무에 한해서 시스템이 정확히 작동함
· 다만 전체 업무의 20~30%에 불과

핵심 한마디:
"구분상가 매매 건이 들어오면, 건축물대장과 블라인드를 한 번에."
```

#### 🥉 3위: Agora (전문가 Q&A) — 세무/법률 질문 채널

```
소구력: ★★★☆☆ | 동기 유형: 정보 접근

· 집합건물 중개사는 세무/법률 질문이 자주 발생
  — "오피스텔 매매 시 부가세 환급이 되나요?"
  — "구분상가 전유부 면적 산정 기준이?"
  — "상가 임대차보호법 적용 범위가?"
· 현재 네이버 지식인이나 카페에서 검색 → 정확도 낮음
· Agora에 AI 1차 답변 + 전문가 후속 답변 구조 → 차별화
· 다만 PoC 초기에는 전문가 답변이 없어 AI 답변만 → 신뢰도 한계

핵심 한마디:
"오피스텔 세금 궁금한 거 물어보면, AI가 먼저 답하고
 세무사가 검증해줍니다."
```

#### 4위: Vendor 서비스 — 인테리어/법무사/세무사 연결

```
소구력: ★★★☆☆ | 동기 유형: 부가 서비스

· 집합건물 거래 후 인테리어, 법무사(등기), 세무사(양도세) 연결 빈번
· 현재는 개인 인맥으로 소개 → 체계적 플랫폼이 없긴 함
· 다만 "이미 아는 사람이 있다"는 브로커가 대다수
· Vendor 추천으로 리베이트/소개비를 받을 수 있다면 동기 ↑

핵심 한마디:
"고객한테 법무사 소개할 때, 검증된 곳을 추천하세요."
```

#### 5위: 무료 사용 + "새로운 기술 체험"

```
소구력: ★★☆☆☆ | 동기 유형: 호기심 + 비용 없음

· "공짜니까 해보자" 수준의 동기
· AI 도구를 써본 적 없는 중개사에게는 체험 자체가 가치
· 그러나 "자기 업무와 직접 관련 없으면" 1~2주 후 사용 중단
· 리텐션을 위해서는 상가/구분상가 거래가 있어야 함

핵심 한마디:
"무료입니다. AI가 어디까지 하는지 한번 보세요."
```

## Segment C. CRE 전문직 (변호사/회계사/법무사 등)

### 시스템이 제공하는 것

이들은 **브로커 도구를 쓰는 사람이 아닙니다.** 시스템에서 이들의 접점은:

| 기능 | 역할 | 수익 모델 |
|------|------|---------|
| **Vendor 입점** | 서비스 카드 등록, 리드 수신 | Basic(무료~₩5만/리드), Pro(₩29만/월), Premium(₩79만/월) |
| **Agora 전문가 답변** | 질문에 전문가 배지로 답변 | 브랜딩 + 리드 전환 |
| **딜카드 연계 의뢰** | 브로커가 거래 진행 시 자동 추천 | 직접 고객 유입 |

### 전문직별 현실 분석

| 직종 | 현재 고객 획득 채널 | CRE DealCard 대비 | 시스템 참여 동기 |
|------|----------------|----------------|-------------|
| **부동산 전문 변호사** | 지인 소개, 법률 플랫폼(로톡) | Agora에서 전문성 노출 → 새 채널 | ★★★☆☆ |
| **세무사/회계사** | 지인 소개, 세무 플랫폼(삼쩜삼) | CRE 특화 양도세/취득세 자문 리드 | ★★★☆☆ |
| **법무사** | 중개사 소개 (거래 연결) | Vendor로 등록 → 브로커 직접 연결 | ★★★★☆ |
| **감정평가사** | 은행/기관 의뢰 | CRE 딜카드 연계 감정평가 의뢰 | ★★★☆☆ |
| **보험 설계사** | 건물주 직접 영업 | 건물 데이터 기반 보험 제안 | ★★☆☆☆ |
| **인테리어 업체** | 네이버/지인 소개 | 상가 공실 임차인 인테리어 수요 연결 | ★★★☆☆ |
| **PF/금융 자문** | 증권사/은행 네트워크 | 딜카드 상의 매물 금융 구조화 자문 | ★★☆☆☆ |

### 탑 5 소구력 순위 (전문직 전체)

#### 🥇 1위: "자는 동안에도 리드가 들어온다" — 수동적 고객 획득 채널

```
소구력: ★★★★☆ | 대상: 전 직종 | 동기 유형: 매출

· 전문직의 가장 큰 고민: "일은 잘 하는데, 고객이 안 온다"
· 현재: 지인 소개 90% + 플랫폼(로톡/삼쩜삼) 10%
· CRE DealCard: Vendor 서비스 카드 등록 → 브로커가 거래 진행 중
  법무사/세무사 필요할 때 자동 추천 → 리드 수신
· Basic 무료 입점 → 리드 건당 ₩5만 → ROI 매우 높음
  (법무사 등기 수수료 ₩50~200만 → 리드비 ₩5만은 가성비 극상)

핵심 한마디:
"무료로 등록하세요. 근처에서 거래가 성사되면
 등기/세금 상담 의뢰가 자동으로 옵니다."
```

#### 🥈 2위: Agora 전문가 배지 — 브랜딩 + SEO

```
소구력: ★★★★☆ | 대상: 변호사, 세무사, 감정평가사 | 동기 유형: 마케팅

· Pro/Premium Vendor는 Agora에서 "전문가" 배지로 답변 가능
· 질문: "꼬마빌딩 양도세 비과세 요건이 뭔가요?"
  → AI가 1차 답변 → 세무사가 전문가 배지로 정정/보완
  → "이 세무사는 CRE를 잘 아네" → 신뢰 형성 → 상담 의뢰
· 로톡/삼쩜삼에서는 CRE 특화 전문가 포지셔닝이 어려움
· DealCard Agora = "CRE 전문" 니치에서 1등 가능

핵심 한마디:
"네이버 지식인에 답변 달지 마시고, CRE 전문가 배지를 다세요.
 진짜 거래하는 사람들이 보는 곳입니다."
```

#### 🥉 3위: CRE 특화 니치 시장 선점

소구력: ★★★☆☆ | 대상: 전 직종 | 동기 유형: 시장 포지셔닝

· 로톡 = 일반 법률, 삼쩜삼 = 일반 세무
· CRE DealCard = 상업용 부동산 전문

핵심 한마디:

### ⚠️ 전문직 PoC의 핵심 리스크

```
닭-달걀 문제:

브로커가 적다 → 거래가 적다 → 리드가 안 온다 → 전문직이 떠난다
전문직이 적다 → Agora가 빈다 → 브로커가 가치를 못 느낀다

해결: PoC 기간 중 전문직은 "Agora 답변자"로 먼저 참여시키고,
     리드 기반 Vendor 전환은 브로커 수 50명+ 이후에 추진
```

## 🔄 보정: "타깃별 문서 자동 생성 + 원클릭 공유" — 과소평가된 킬러 피처

### 이전 분석에서 빠진 것

실제 시스템의 딜카드 결과 화면에는 **6개의 문서 생성/공유 CTA**가 구현되어 있습니다:

```
┌─────────────────────────────────────────────┐
│  딜카드 결과 화면 — Sticky CTA 바            │
│                                             │
│  💬 카톡 문구 복사        → 매수 후보에게     │
│  📄 Full IM Studio       → 기관투자자에게     │
│  📱 모바일 투자설명서      → 개인 매수자에게   │
│  📊 건물주 리포트         → 매도인(건물주)에게 │
│  ✨ 전문가 코멘트 받기     → 내부 검증용       │
│  🎯 매수자 전체 보기      → 브로커 자신       │
└─────────────────────────────────────────────┘
```

**이것은 단순 "문서 생성"이 아닙니다.**  
**하나의 메모에서 수신자별로 다른 문서가 자동으로 만들어지는 것**입니다.

### 왜 과소평가되었나

이전 분석에서 "카톡 문구 자동 생성 ★★☆☆☆"로 평가한 것은 **카톡 문구 하나만 보았기 때문**입니다. 실제로는:

| 문서 유형 | 수신자 | 현재 수작업 | 시스템 자동화 | 소구력 |
|---------|--------|----------|-----------|--------|
| 블라인드 티저 | 매수 후보 1차 | 워드로 직접 작성 30분 | AI 60초 | ★★★★★ |
| 카톡 발송 문구 | 매수 후보 | 매번 직접 타이핑 | 복사 버튼 한 번 | ★★★☆☆ |
| **모바일 투자설명서** | **개인 매수자** | **PPT/한글로 2~4시간** | **AI 약 3분** | **★★★★★** |
| **Full IM (투자각서)** | **기관/법인 매수자** | **외주 또는 반나절~하루** | **Studio 핸드오프** | **★★★★★** |
| **건물주 리포트** | **매도인** | **별도 작성 (대부분 안 함)** | **자동 생성** | **★★★★☆** |
| 전문가 코멘트 | 내부 | 전화/카톡으로 물어봄 | 시스템 요청 | ★★★☆☆ |

### 보정된 평가

#### 세그먼트 A (꼬마빌딩) — 이것이 진짜 킬러 피처

```
[기존 순위에서의 위치]
· "카톡 문구 자동 생성 ★★☆☆☆"로 분류되어 있었음
· 모바일 IM, Full IM, 건물주 리포트는 아예 분석에서 누락

[보정 후]
· "타깃별 문서 자동 생성 파이프라인" 전체로 보면 ★★★★★
· 꼬마빌딩 중개사의 핵심 업무 흐름과 정확히 일치:

  매물 접수 → 블라인드 작성 → 관심 매수자에게 보내기
            → 관심 표하면 → 투자설명서(IM) 보내기
            → 진지하면 → Full IM + 현장 미팅

  이 전체 과정에서 문서를 4번 만들어야 하는데,
  현재는 4번 다 수작업. 시스템은 4번 다 자동.
```

> [!IMPORTANT]
> **보정된 소구력 순위 (꼬마빌딩)**:
>
> | 순위 | 기존 분석 | 보정 후 |
> |------|---------|--------|
> | 🥇 | 건축물대장/실거래가 자동 | **타깃별 문서 자동 생성 파이프라인** (블라인드→IM→Full IM→건물주 리포트) |
> | 🥈 | 블라인드 딜카드+민감정보 | 건축물대장/실거래가 자동 + 딜카드 내 교차검증 |
> | 🥉 | JS부동산 소속감+무료 | 블라인드 딜카드 + 민감정보 자동 관리 |
> | 4 | 파이프라인 관리 | JS부동산 소속감 + 무료 |
> | 5 | AI 매수자 매칭 | 파이프라인 관리 + AI 매수자 매칭 |

**왜 1위로 올라가는가:**

```
시간 절약 비교:

건축물대장 자동 조회:  30분 → 10초  (절약 ~30분/건)
문서 4종 자동 생성:   4~8시간 → 5분  (절약 ~4~8시간/건)
                    ─────────────────────────
                    → 문서 자동 생성의 절약 효과가 10배 이상 큼
```

꼬마빌딩 한 건의 거래 사이클에서 **문서 작성에 쓰는 총 시간이 가장 길기 때문**입니다. 건축물대장 조회는 30분이지만, 블라인드+IM+Full IM+건물주 리포트를 합치면 **하루 이상**입니다.

#### 세그먼트 B (집합건물) — 순위 변동

```
보정: 구분상가 매매 시에도 동일 문서 파이프라인이 작동
     → "구분상가 딜카드+건축물대장" (기존 2위)에
       "투자설명서 자동 생성"이 포함되어 가치 상승
     → 2위 유지하되 소구력 ★★★★☆ → ★★★★★로 상향
```

#### 세그먼트 C (전문직) — 순위 변동 없음

전문직은 문서를 만드는 사람이 아니라 **문서를 받는 쪽**이므로 직접 영향 없음. 다만 "건물주 리포트"에 Vendor 추천이 포함되면 간접적 리드 효과 ↑

### 소개서 수정 제안

기존 소개서의 프레이밍:
```
❌ "카톡 메모 한 장이면, 60초 만에 딜카드가 나옵니다"
```

보정된 프레이밍:
```
✅ "메모 한 장으로 — 블라인드, 투자설명서, 건물주 리포트까지
   거래의 모든 문서가 자동으로 만들어집니다"
```

**중개사가 가장 귀찮아하는 것은 "딜카드 1장 만들기"가 아닙니다.**  
**"매수자한테 보낼 거, 건물주한테 보낼 거, 기관한테 보낼 거를 각각 다르게 만들어야 하는 것"**입니다.  
이것을 원클릭으로 해결한다는 것이 진짜 소구점입니다.
