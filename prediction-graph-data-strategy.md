## 2. 데이터 수요 — 모듈별 현재 상태 & 활성화 조건

### P-D2 매수자 클러스터링

| 항목 | 내용 |
|---|---|
| **필요 테이블** | `buyer_intent_lite` |
| **최소 행 수** | 50건 |
| **현재 예상** | 0건 (시스템 첫 사용 시) |
| **Cold Start 동작** | K-Means 실패 시 오류 반환 + 안내 메시지 |
| **완전 활성화** | 50건+ → K-Means 자동 실행 |
| **자동 업데이트** | 매칭 실행마다 새 매수자 자동 분류 |

### P-X 딜 전환율 예측

| 항목 | 내용 |
|---|---|
| **필요 테이블** | `deal_pipeline_states`, `match_results`, `building_ssot_lite`, `activity_events` |
| **모드 전환** |  0-80건: 규칙 기반 휴리스틱 → 80건+: Logistic Regression |
| **지금 바로 사용 가능** | ✅ 규칙 기반으로 즉시 확률 제공 |
| **학습 데이터 소스** | `deal_conversion_features` 테이블 (파이프라인 종료 시 자동 스냅샷) |
| **학습 실행** | 별도 Python 스크립트 (오프라인) → 계수 저장 → DB `prediction_model_coefficients` |

### P-D 가격 추정

| 항목 | 내용 |
|---|---|
| **외부 데이터** | 국토부 실거래가 API (data.go.kr, 무료) |
| **최소 건수** | 지역당 5건+ (10억-300억 상업용) |
| **지금 바로** | ✅ ETL 실행 시 즉시 3,000건+ 수집 가능 |
| **API 키 필요** | `MOLIT_API_KEY` 환경변수 설정 |
| **정확도 개선** | 데이터 200건+ 시 XGBoost 학습 가능 |

### G-X 지식 그래프

| 항목 | 내용 |
|---|---|
| **필요 데이터** | 기존 테이블 (match_results, pipeline, spaces 등) |
| **즉시 시작** | ✅ 매칭 1건만 있어도 엣지 생성 시작 |
| **자동 생성** | 매칭 실행 → `matched_with` 엣지 자동 생성 (이미 연결됨) |
| **comparable_to** | 수동 트리거 또는 주간 배치 |

### G-D 딜 시맨틱 검색

| 항목 | 내용 |
|---|---|
| **필요 데이터** | `deal_casepacks` + `embedding` 컬럼 |
| **전환 조건** | 임베딩된 CasePack 30건 이상 |
| **지금 동작** | 텍스트 like 검색 (자동 fallback) |
| **pgvector 필요** | Supabase 대시보드에서 Extension 활성화 |
| **임베딩 자동화** | 매칭 API 호출 시 자동 생성 (이미 연결됨) |

### G-I IM 시맨틱 검색

| 항목 | 내용 |
|---|---|
| **필요 데이터** | `im_projects` + `embedding`, `outcome` 컬럼 |
| **전환 조건** | 임베딩된 IM 20건 이상 |
| **지금 동작** | SectionPlanner 주입 없이 일반 실행 |
| **연동 필요** | cre-fullim의 IM 생성 시 `generateImEmbedding()` 호출 추가 |

### G-S 매물 네트워크

| 항목 | 내용 |
|---|---|
| **필요 데이터** | `match_results` (매물 15건 + 매수자 10명 이상의 매칭) |
| **즉시 동작** | ✅ 매칭 결과만 있으면 buyer_overlap 자동 탐색 |
| **활성화 수준** | 매칭 30건+ 시 추천 품질↑ |

## 3. 데이터 확보 전략

### 단계 1: 즉시 실행 (Day 1)

```
① MOLIT ETL 실행
   POST /api/admin/etl/molit { "months": 36 }
   → 서울 상업용 실거래 ~9,000건 수집 (필터링 후 ~2,000건)
   → P-D 가격 추정 즉시 활성화

② G-X comparable_to 배치 실행
   knowledge-graph.ts buildComparableEdges()
   → 기존 매물 조합으로 유사 매물 엣지 생성
```

### 단계 2: 운영 중 자동 축적 (D+1 ~ D+30)

```
매칭 1건 실행 시 자동으로:
  → deal_casepacks 1건 추가 (딜 지식 축적)
  → knowledge_edges 2건 추가 (양방향 matched_with)
  → buyer_intent_lite.cluster_id 자동 갱신 (P-D2)
  → CasePack embedding 비동기 생성 (G-D 진도)

파이프라인 종료 시 자동으로:
  → deal_conversion_features 스냅샷 저장 (P-X 학습용)
```

### 단계 3: 중개인 수동 시드 (D+7까지)

```
과거 딜 30건 입력 가이드:
  - 성사 15건: 매물 기본정보 + 결과 = "closed"
  - 실패 15건: 매물 기본정보 + 결과 = "failed"
  - 각 10분 소요 → 총 5시간

매수자 조건 50건 입력:
  - 현재 보유 매수자 리스트 → buyer_intent_lite 입력
  - 각 5분 → 총 4시간
  → P-D2 클러스터링 즉시 활성화
```

### 단계 4: ML 모델 학습 (D+90)

```
deal_conversion_features.converted IS NOT NULL >= 80건 확인 후:
  Python 학습 스크립트 실행
  → Logistic Regression 계수 → prediction_model_coefficients 저장
  → P-X 모드 자동 전환 (heuristic → model)
```

## 4. 차별화 기능 — 중개인 관점 스토리

### 🎯 기능 1: "딜 성사 확률 AI"

**스토리**: 중개인 박 사장이 성수동 근생 80억 매물을 등록하고 매수자 3명과 매칭했습니다.
AI가 "이 딜 성사 확률 72%" 라고 보여주며 "최우선 매수자가 S등급 2명 확보됨 (+18%)"라고 설명합니다.
"매수자 미팅을 이번 주 안에 잡아보세요"라는 다음 행동을 제안합니다.

**차별화 포인트**:
- 경쟁사: 매물 등록 후 스스로 판단
- 이 시스템: AI가 성사 확률 수치 + 원인 + 다음 행동까지 제시

**데이터 조건**: 즉시 (규칙 기반) → 90일 후 ML 모델로 정확도↑

---

### 🔗 기능 2: "같은 매수자가 본 매물"

**스토리**: 합정 근생 55억 매물을 검색 중인 매수자 이 씨가 있습니다.
시스템이 "이 매수자가 성수동 근생 80억에도 관심을 보였습니다"라고 알려줍니다.
중개인은 두 매물을 연계해 "비교 제안"을 드릴 수 있습니다.

**차별화 포인트**:
- 경쟁사: 매물별 독립 관리
- 이 시스템: 매수자를 통해 매물들이 연결되는 **협업 필터링** 효과

**데이터 조건**: 매칭 30건+ (약 1개월)

---

### 🧠 기능 3: "유사 딜 자동 브리핑"

**스토리**: 강남 사옥 90억 매물이 등록되면, 시스템이 "이전에 서초 사옥 85억 딜에서는
'법인 매수자의 사옥 전환 가능성을 강조하는 것이 효과적'이었습니다"라고 알려줍니다.
중개인은 이 인사이트를 바탕으로 상담 전략을 세울 수 있습니다.

**차별화 포인트**:
- 경쟁사: 중개인 개인 경험에 의존
- 이 시스템: 과거 딜의 노하우가 AI 메모리로 자동 전달

**데이터 조건**: CasePack 10건(텍스트) → 30건(시맨틱 AI 검색)

---

### 💰 기능 4: "AI 가격 검증"

**스토리**: 건물주가 80억을 요구합니다. 중개인이 시스템에 입력하니
"성수동 같은 구역 최근 1년 실거래 23건 기준 AI 추정가: 72-88억 (참고용)"이라고 표시됩니다.
건물주 협상에서 "시장 데이터 기반"으로 이야기를 꺼낼 수 있습니다.

**차별화 포인트**:
- 경쟁사: 감각적 판단
- 이 시스템: 국토부 실거래 데이터 기반 구간 추정

**데이터 조건**: MOLIT ETL 즉시 실행 가능

---

### 🗂️ 기능 5: "매수자 자동 분류"

**스토리**: 매수자 메모를 입력하면 AI가 "법인 사옥 추구형"으로 자동 분류하고
"사옥 가중치 프로파일"을 적용해 매칭 정확도를 높입니다.
"이 유형의 매수자 5명이 현재 등록된 강남/서초 매물을 선호합니다"라고 알려줍니다.

**차별화 포인트**:
- 경쟁사: 목적 드롭다운 수동 선택
- 이 시스템: 메모에서 자동 추론 → 클러스터 배정 → 매칭 가중치 자동 조정

**데이터 조건**: 매수자 50건 (P-D2)

## 5. 데이터 플라이휠 효과

```
매물 등록
  → 딜카드 생성 → CasePack → G-D 딜 지식 축적
  → 매칭 실행 → G-X 그래프 엣지 → G-S 연관 매물 발견
  → 파이프라인 진행 → P-X 학습 데이터 스냅샷
  → 종료(성사/실패) → P-X 모델 개선 + G-I IM 패턴 학습

딜이 쌓일수록:
  P-D2 클러스터 정확도↑ → 매칭 정확도↑ → 성사율↑ → 더 많은 딜 데이터 →
  P-X 모델 정확도↑ → 더 나은 행동 제안 → 더 빠른 클로징 →
  (반복)
```

> **핵심**: 시스템이 사용될수록 모든 예측이 더 정확해지는 자기강화 루프
